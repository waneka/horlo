## Deviations from Plan

**None — plan executed exactly as written, with three minor incidental tightenings flagged here for completeness:**

1. The plan did not specify whether `BrowseModule.tsx` should keep `grid-cols-3` after the Genres tile deletion. I tightened it to `grid-cols-2` because two tiles in a three-column grid render with an empty third cell on every viewport. This matches the §F deletion intent ("layout reflows naturally") rather than mandating a specific grid width. No UI-SPEC contract is violated.

2. The plan said "Verify the existing test still passes after the source rewire" for `browse.test.ts`. I noticed the existing test comment said `(WHERE NOT NULL in query ensures this)` — I updated that comment to reflect the rewired predicate `(WHERE tag = ANY(ARRAY[...]))` so the test docs match the new SQL. Behavior unchanged.

3. The plan said the new `derives counts from style_tags` test should use `expect(callArg.toString()).toMatch(/unnest\(style_tags\)/)`. I used `JSON.stringify(callArg)` instead because Drizzle's `sql` template object's `.toString()` returns `'[object Object]'` under vitest (jsdom) — the JSON traversal of the internal `queryChunks` array is the cross-version safe fingerprint. This is mechanically equivalent for the substring assertion.

**Auth gates:** None encountered (DAL test mocks `db.execute`; component tests mock `next/link`; no live DB or auth path was touched).

## Patterns Established

- **Closed-vocab const survives the column drop:** `PRIMARY_ARCHETYPES` is retained in `src/lib/taste/vocab.ts` not for enricher use (that path is removed in other plans) but for the rewired SQL's ANY-array predicate. The const is the single source of truth for the 10 archetype keys — `ARCHETYPE_CONFIG` (display copy) and the SQL filter both consume it.
- **Drizzle `sql.join` over a const tuple:** `WHERE tag = ANY(ARRAY[${sql.join(consts.map((v) => sql\`${v}\`), sql\`, \`)}])` is the idiomatic parameter-bound multi-value match. Analog in `src/data/catalog.ts:611` (motifsSql). No string interpolation, no SQL injection surface even though the 10-value list is build-time const.
- **`'use cache' + cacheTag('explore', 'explore:browse')` scope is preserved across DAL rewires:** The rewired `getBrowseArchetypeCounts` keeps the same cache tag, so the existing `revalidateTag('explore', 'max')` calls in catalog mutation paths (`src/app/actions/watches.ts:294`, `src/app/api/extract-watch/route.ts`) continue to bust this cache on writes.
