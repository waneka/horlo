## 5. New Tech Debt Items

### DEBT-09 (proposed, HIGH severity) — surfaced by Phase 31 audit

`addWatch` and `editWatch` in `src/app/actions/watches.ts` do not persist the `notesPublic` field and do not call `revalidatePath('/u/{username}/{tab}')` after mutation. Phase 23 SUMMARY claimed both behaviors shipped via commit `4d362ff`, but `git merge-base --is-ancestor 4d362ff HEAD` returns exit 1 — that commit never reached `main`.

**Reproducible evidence (re-verified 2026-05-05T17:04Z by Phase 31 audit):**
- `git merge-base --is-ancestor 4d362ff HEAD` → `1` (not an ancestor)
- `grep -cnE "notesPublic|notes_public" src/app/actions/watches.ts` → `0`
- `grep -cE "revalidatePath\('/u/" src/app/actions/watches.ts` → `0`
- `npx vitest run tests/actions/watches.notesPublic.test.ts --reporter=basic` → `Test Files 1 failed (1) / Tests 4 failed (4)`

**User impact:** WatchForm sends `notesPublic` value, Zod silently strips it (no schema field), the value never reaches the DB. Cross-page caching also stale (no revalidate). Documented as `human_verification` carryover item in `23-VERIFICATION.md`.

**Remediation owner:** v4.2 patch or v5.0 carryover. **Not a v4.1 blocker.** DEBT-07/08 only required the audit, which Phase 31 delivered. Surfacing the regression IS the audit's value.
