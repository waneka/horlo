## v4.0 Requirements

### Catalog Foundation

Canonical `watches_catalog` table — silent infrastructure for v4.0. Unblocks /search Watches, /explore Trending, /search Collections by-watch-identity. Per-user `watches.catalog_id` FK is **nullable indefinitely** in v4.0.

- [x] **CAT-01**: System has a `watches_catalog` table with surrogate UUID PK and natural-key UNIQUE on normalized `(brand, model, reference)` trio — Phase 17 (Validated)
- [x] **CAT-02**: `watches_catalog` is public-readable via RLS; only the service-role Drizzle client can write — Phase 17 (Validated)
- [x] **CAT-03**: `watches_catalog` has `pg_trgm` GIN indexes on `brand` + `model` for sub-200ms search at production scale — Phase 17 (Validated)
- [x] **CAT-04**: `watches` table has a nullable `catalog_id` FK with `ON DELETE SET NULL` so catalog-row deletion doesn't break user collections — Phase 17 (Validated)
- [x] **CAT-05**: An idempotent batched backfill script links existing `watches` rows to catalog entries — Phase 17 (Validated)
- [x] **CAT-06**: System provides `upsertCatalogFromUserInput` helper using `ON CONFLICT … DO NOTHING` — Phase 17 (Validated)
- [x] **CAT-07**: System provides `upsertCatalogFromExtractedUrl` helper using `ON CONFLICT … DO UPDATE SET col = COALESCE(catalog.col, EXCLUDED.col)` per nullable spec column — Phase 17 (Validated)
- [x] **CAT-08**: `addWatch` Server Action and `/api/extract-watch` route handler both populate `watches_catalog` via the appropriate upsert helper — Phase 17 (Validated)
- [x] **CAT-09**: A `pg_cron` daily-batch function refreshes denormalized `owners_count` + `wishlist_count` on `watches_catalog` (prod-only; SECURITY DEFINER with REVOKE FROM PUBLIC/anon/authenticated) — Phase 17 (Validated)
- [x] **CAT-10**: A manual `npm run db:refresh-counts` script provides the same refresh path for local dev — Phase 17 (Validated)
- [x] **CAT-11**: Catalog is authoritative for SPEC fields at display time via `catalog_id` JOIN; per-user `watches` remains authoritative for OWNERSHIP/PROVENANCE fields — Phase 17 (Validated)
- [x] **CAT-12**: A `watches_catalog_daily_snapshots` table records `(catalog_id, date, owners_count, wishlist_count)` for 7-day Gaining Traction delta on /explore — Phase 17 (Validated)
