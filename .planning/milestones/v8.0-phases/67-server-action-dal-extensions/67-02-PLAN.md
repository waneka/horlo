## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → Server Action | Client passes `{ q, limit? }` payload; viewer identity sourced from server session |
| Server Action → DAL | DAL takes explicit `{ q, viewerId, limit }` typed params — viewerId derived server-side |
| DAL → Database | Drizzle parameterized SQL templates only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-67-02-01 | Tampering | `q` search input | mitigate | Drizzle parameterized `sql` templates (matches `searchCatalogWatches` T-19-01-01 mitigation); no string interpolation in WHERE/ORDER BY |
| T-67-02-02 | Information Disclosure | DAL error message leakage to client | mitigate | T-19-02-04 lock — outer catch returns generic `"Couldn't run search."`; DAL error detail logged with `[searchCatalogForAddFlow]` prefix, NEVER returned to client |
