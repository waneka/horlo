    Create `src/lib/searchEntry/parseSearchQuery.ts` (NEW directory + file). NO 'use client' directive. Pure function — no React imports. Export named function `parseSearchQuery(query: string, catalogBrands: string[]): { brand: string; model: string; reference: string }`.

    Algorithm per D-12 (CONTEXT.md, MUST follow verbatim):
    1. Normalize the input query: trim, collapse internal runs of whitespace to single spaces, lowercase. Call this `normalizedQuery`.
    2. If `normalizedQuery` is empty, return `{ brand: '', model: '', reference: '' }`.
    3. Build a list of `(originalBrand, normalizedBrand)` tuples from `catalogBrands` using the same normalization (trim + collapse + lower). Sort this list by `normalizedBrand.length` DESCENDING so multi-word brands match before their prefixes (Tag Heuer before Tag).
    4. Iterate the sorted tuples; find the first one whose `normalizedBrand` is a whitespace-bounded prefix of `normalizedQuery`. "Whitespace-bounded" means the character immediately AFTER the matched prefix is either a space or end-of-string (so "rolex" does NOT match a query starting with "rolexX").
    5. On hit: set `brand` = `originalBrand` (catalog casing preserved). Compute `remainder` = the substring of `normalizedQuery` after the matched prefix, trimmed. Within `remainder`, split on whitespace; the LAST token containing at least one digit (`/\d/`) becomes `reference`; the remaining tokens (joined with a single space) become `model`. If no token contains a digit, `reference` is `''` and `model` is the full remainder.
    6. On miss (no brand prefix matched): naive fallback. Split `normalizedQuery` on whitespace into tokens. If empty, return all empty. Otherwise: `brand` = first token (PRESERVE user input casing — use the corresponding token from the ORIGINAL trimmed-collapsed query, not the lowercased copy); the LAST token containing a digit becomes `reference`; tokens between first and reference (exclusive) join into `model`. If only one token, `model` is `''` and `reference` is `''` (unless that single token contains a digit — then it stays as `brand` with `reference: ''`; see Test 6 vs Test 5).

    Edge case clarification: For Test 5 ("cartier 4329xx" → brand="cartier", reference="4329xx", model=""), the naive split treats "cartier" as brand and "4329xx" as the last digit-bearing token → reference. Nothing between → model = "". For Test 6 ("speedmaster" → brand="speedmaster"), the single token has no digit, so brand="speedmaster", reference="", model="".

    Casing preservation on naive miss: the algorithm normalizes to lowercase for prefix matching, but the FALLBACK preserves user input as typed. Track the original-case tokens by splitting the trim-and-collapse (but NOT lower) form of the query in parallel.
