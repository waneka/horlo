<objective>
Swap the recommender read path from free-text (brand|model) string comparison to FK-keyed (brandId|familyId) identity comparison so the home rail's exclusion set and multi-brand-match scoring no longer surface a user's own watch or drift-branded catalog rows. Widen topBrandOf to key on brandId + return canonical brandName; restructure RationaleContext to carry a pre-computed viewerTopBrand; INNER JOIN brands + watch_families in topUpFromCatalogPopularity so synthetic top-up rows carry canonical display strings.

Purpose: Closes RECO-01 (Brut Date vs Brut Datejust / Héron vs Héron Watches drift no longer surfaces owner's own family), RECO-02 (Hamilton + Hamilton Watch both trigger the +100 boost against canonical Hamilton), RECO-03 (topBrandOf counts by canonical brand id, excluding legacy catalogId=null rows from stale-string totals), and RECO-04 (rationale renders canonical brand strings from JOINed brands.name). Depends on Plan 01's Watch.brandId?/familyId? projection + brands/watch_families schema alignment.

Output: Recommender exclusion + scoring keys on brand_id/family_id; synthetic top-up carries canonical strings + FK identity; unit tests cover the new key shape, legacy-row exclusion, and Pitfall 5 self-in-own-rail identity check; npm run build exits 0; existing DAL recommendations tests still pass.
</objective>
