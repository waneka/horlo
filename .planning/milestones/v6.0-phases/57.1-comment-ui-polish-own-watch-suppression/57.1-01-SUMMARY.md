# Phase 57.1 Plan 01: Comment UI Polish — SC-2 + SC-3 + D-04 Summary

**One-liner:** Restructured CommentCompose to side-by-side Post button (SC-2) and moved CommentItem's edit/delete icons and delete confirm inline onto the meta row (SC-3 + D-04), fixing the "controls hanging below" UAT complaint.

---

## What Was Built

### Task 1 — SC-2: Inline Post Button (CommentCompose)

**Commit:** `14762b5`
**File:** `src/components/comment/CommentCompose.tsx`

Replaced the stacked `flex flex-col gap-2` compose layout with a side-by-side `flex items-end gap-2` layout. The textarea is wrapped in a `flex-1 min-w-0 flex flex-col gap-1` column that takes all available horizontal space. The char counter sits below the textarea inside that column (text-right). The Post button is a `flex-shrink-0 self-end` sibling of the textarea column — always inline to its right. Added `min-h-[44px] min-w-[44px]` touch-target minimum to the Post button. Removed the old `flex items-center justify-between` counter/button row entirely.

Char-counter thresholds preserved verbatim: appears at `body.length >= 450`, turns `text-destructive` at `body.length >= 480`.

### Task 2 — SC-3 + D-04: Inline Controls in CommentItem

**Commit:** `ecc623e`
**File:** `src/components/comment/CommentItem.tsx`

Wrapped the meta `<p>` (username · timeAgo · [edited]) in a `flex items-center gap-1 flex-wrap` row container with `flex-1 min-w-0` on the `<p>`. Inside that same flex row:

- **SC-3:** When `isAuthor && !isEditing && !isDeleting`, Pencil + Trash2 buttons render inline as a `flex-shrink-0` sibling. Icon sizes reduced from `size-4` to `size-3.5` for proportionality on 12px meta text. All existing button classes (`min-h-[44px] min-w-[44px]`, aria-labels, handlers) preserved.
- **D-04:** When `isDeleting && !isEditing`, the "Delete? · Cancel" confirm renders inline in the same meta row (as a `flex-shrink-0` sibling), replacing the icons. The `flex-wrap` on the row allows graceful wrapping on narrow phone widths.

Deleted the old `flex items-center gap-2 mt-1` icon block (was below body) and the old `flex items-center gap-1 mt-1` delete confirm block (was below body). The edit-mode Save/Cancel textarea block is unchanged.
