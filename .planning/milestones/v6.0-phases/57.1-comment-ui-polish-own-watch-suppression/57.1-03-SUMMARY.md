# Phase 57.1 Plan 03: SC-6 Own-Watch Compose Suppression Summary

SC-6 display gate threaded through CommentList/CommentThread/page.tsx: owner sees no compose box on their own watch, count badge preserved, DAL unchanged.

## What Was Built

Added `suppressCompose?: boolean` prop to `CommentList` and `CommentThread`, and a `canCommentDisplay` derived boolean to `/watch/[id]/page.tsx`. When an owner visits their own watch:

- `canCommentDisplay = isOwner ? false : canComment` → false for owner
- `suppressCompose={isOwner}` → true for owner
- `CommentList` compose slot: `{!suppressCompose && (canComment ? <CommentCompose> : <CommentGateLocked>)}` — renders null for owner (D-02 silent suppression)
- The existing comment list below the compose slot is unaffected — owner still sees all comments
- The count read at `~line 80` keeps the raw `canComment` (true for owner via GATE-04) so the count badge still shows (CMNT-09)

Non-owners are completely unaffected: `WearCommentHost` never passes `suppressCompose`; the gate locked state (GATE-03) still renders for gated non-owners.
