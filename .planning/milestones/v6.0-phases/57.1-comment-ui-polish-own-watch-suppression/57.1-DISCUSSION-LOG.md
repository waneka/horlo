## /watch count update (SC-5)

| Option | Description | Selected |
|--------|-------------|----------|
| router.refresh() after post | After a successful post on /watch/[id], call router.refresh() to re-fetch the RSC — count updates within a server roundtrip, no manual refresh. Cheap, no restructuring. | ✓ |
| Accept static (spec default) | Count updates only on next manual refresh/navigation. Zero effort, but the count appears not to update after posting. | |
| Full optimistic (restructure) | Lift the count into the client island for instant increment. Most work; risks RSC/cache fragility the spec avoided. | |

**User's choice:** router.refresh() after post
**Notes:** Refines the UI-SPEC SC-5 "leave it static" default. Planner must verify against the Next 16
Router Cache behavior in project memory (stale client-component instance, proxy/Router-Cache poisoning);
fall back to static rather than restructuring if router.refresh() proves unsafe. Fires only on the
success branch, only on /watch/[id].

## Count drift / reconcile (SC-5)

| Option | Description | Selected |
|--------|-------------|----------|
| Accept eventual consistency | Optimistic local count only; self-corrects on next full page load/navigation. Simplest; ships the required behavior. | ✓ |
| Reconcile on drawer reopen | Re-sync count from a fresh server read when the bottom-sheet reopens. More correct; inline /wear variant still drifts. | |
| Refetch after each post | router.refresh()-style re-sync after every post on the wears-lane too. Most consistent; partly defeats SC-5's instant feedback. | |

**User's choice:** Accept eventual consistency
**Notes:** Confirms the UI-SPEC default. No reconcile logic on wears-lane / /wear. Drift requires a
concurrent commenter on the same target while the drawer is open — accepted rare edge for a personal
collector app.

## Delete-confirm placement (SC-3)

| Option | Description | Selected |
|--------|-------------|----------|
| Inline on meta row | "Delete? · Cancel" replaces the icons in place on the meta row (mutually exclusive states). Fully resolves the UAT "controls hanging below body" feel. Minor wrap risk on narrow widths. | ✓ |
| Keep below body (spec default) | Icons inline, but the confirm shows below the body as the UI-SPEC specifies. Zero wrap risk; re-introduces a below-body control element. | |

**User's choice:** Inline on meta row
**Notes:** Refines the UI-SPEC SC-3. The confirm and the edit/delete icons are mutually exclusive states
of the author controls (icons when isAuthor && !isEditing && !isDeleting; confirm when isDeleting), so the
confirm naturally occupies the same meta-row slot. Allow the meta row to wrap gracefully on small phones.
