# Milestone v7.0: Watch Photos & Detail Redesign

**Status:** ✅ SHIPPED 2026-05-28
**Phases:** 59–65 (7 phases; no decimal/inserted phases — Phase 65 was a planted follow-on to v7.0 surfaced during the Phase 64 UAT, not an insertion)
**Total Plans:** 29 (Phase 59: 3, Phase 60: 4, Phase 61: 6, Phase 62: 5, Phase 63: 3, Phase 64: 5, Phase 65: 3)
**Timeline:** 4 days (2026-05-25 → 2026-05-28)
**Sources:** SEED-013 (multi-photo + carousel + wear-pic surfacing), SEED-015 (inline grid engagement), SEED-016 (`/w/[ref]` detail redesign), Phase 50 architecture spike (Variant C verdict).

## Overview

Give every watch real, owned photography on a redesigned detail page — built once on a unified `/w/[ref]` route.

Variant C was tackled **first** as a hard cutover (no redirect) so the carousel and all downstream photo / wear-pic / engagement / IA surfaces landed exactly once on the canonical URL. The schema change to `watches_catalog` was an in-place ALTER (not a wipe — `watches_catalog` is not wipeable per `project_db_wipeable_2026_05_09` memory). Public wear pics surface to watch detail automatically (public = consent), with an owner per-pic hide control. Inline like + comment from profile grid cards lowers the friction to engage with a watch without leaving the profile. The detail page itself was recomposed into an intentional information hierarchy (carousel-forward, elevated verdict, deliberate comment placement), preserving the Phase 51/52 Cache Components structure (CommentThread stays an uncached Suspense sibling; `unstable_instant = false` on `/u/[username]/[tab]` is permanent and untouched). Phase 65 added a follow-scoped "people you follow who own this" module to the hero right column — surfaced during the Phase 64 UAT and captured as a follow-on rather than expanding Phase 64's recompose-only scope.

## Milestone Summary

**Decimal / Inserted Phases:** None. Phase 65 was a planted follow-on (not an insertion); all other phases were originally scoped.

**Key Decisions:**

- **Variant C as hard cutover, not redirect** (operator decision 2026-05-25) — legacy `/watch/[id]` + `/catalog/[catalogId]` removed; un-migrated links fail loudly; CI guard (ROUTE-03) is the completeness guarantee.
- **`watches_catalog` is NOT wipeable** — in-place ALTER + UPDATE migration only; data migrations keyed by `(brand, model, reference)`, not `id` (ids diverge local/prod).
- **Phase 65 added as a follow-on, not Phase 64 expansion** — kept the IA redesign scope clean; let the follow-scoped module ship with its own DAL/component/integration trio.
- **`unstable_instant = false` on `/u/[username]/[tab]` is PERMANENT** — Phase 52 lesson reaffirmed; Phase 64 IA redesign preserved CommentThread as uncached Suspense sibling.
- **`connection()` static-shell opt-out is the React #419 soft-nav fix** (Phase 61 Plan 06) — `await connection()` above the page/layout Suspense opts out of the prerendered static shell that aborts on soft-nav resume; resolves the family of #419+404 regressions.
- **`workflow.use_worktrees = false` remains permanent** — this project is build-gated + DB-touching; `.env.local` unavailable in worktrees.

**Issues Resolved (during the milestone):**

- React #419 soft-nav 404 on `/w/[ref]` and `/wear/[id]` (Phase 61 Plan 06; `connection()` opt-out + admin client for cover signing).
- React #418 date-TZ hydration mismatch in WatchDetail/WornTimeline/WornCalendar (Phase 61 Plan 06; pinned `timeZone: 'UTC'` + `'en-US'`).
- Catalog-photo placeholder gap on cards (Phase 61 inline; migration `20260526120000` backfilled 31 prod `watch_photos` rows from catalog covers).
- IDOR CR-02: storage path must prefix `{userId}/` (Phase 61 Plan 01).
- Add-watch flow gap #9: `WatchForm` onWatchCreated suppresses success toast so Sonner action-button cannot navigate away from the photos-pending step (Phase 61 Plan 06).
- Mobile brand+model below the fold on `/w/[ref]` (Phase 64 Plan 05; `lg:hidden` / `hidden lg:block` JSX dup, not CSS `order-`).
- `revalidateTag('viewer:…:counts')` D-12 gap in `toggleLikeAction` + `addCommentAction` (Phase 63 Plan 01).

**Issues Deferred (to future milestones):**
