<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL → hook state | Initial facet values come from `searchParams.get(...)` — untrusted input on first render. Validated downstream by Zod on every Server Action call. |
| Hook → Server Action | `searchWatchesAction` invocation passes movement / size / style values — Zod `.strict()` + `.enum(...)` rejects malformed payloads (Plan 40-01 mitigation). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-11 | Tampering | URL param injection (user pastes URL with `?movement=DROP TABLE`) | mitigate | Zod `.enum(['auto','manual','quartz','spring_drive'])` rejects unknown values; the hook does not pre-validate. If Zod rejects, `searchWatchesAction` returns `{ success: false, error: 'Invalid request' }`, the hook flips `setWatchesHasError(true)`, UI shows error banner. Defense-in-depth: chip components in 40-05 only ever emit known-good values. |
| T-40-12 | Information Disclosure | URL bookmark sharing | accept | Facet params are catalog filters (no PII). Sharing a URL with `?movement=auto&style=tool` exposes nothing private. |
| T-40-13 | Tampering | comma-separated style values longer than Zod cap | mitigate | Zod `style: z.string().max(500)` rejects payloads > 500 chars. UI never produces more than 8 style chips × ~20 chars = ~160 chars; legitimate clients are well under the cap. |
</threat_model>
