  <behavior>
    Function signature: `export function computeDeltaPhrase(candidate: CatalogTasteAttributes, owned: CatalogTasteAttributes): string`

    Algorithm (D-16, 5 steps):

    1. **Compute per-dimension deltas:**
       - formalityDelta = `(candidate.formality !== null && owned.formality !== null) ? |c - o| : null`
       - sportinessDelta = same pattern
       - heritageDelta = same pattern (over `heritageScore`)
       - archetypeDelta = `candidate.primaryArchetype === owned.primaryArchetype ? 0 : 1` (treat null===null as match=0; null vs non-null as mismatch=1)
       - eraDelta = same pattern (over `eraSignal`)
       - motifJaccard = `jaccardSimilarity(candidate.designMotifs ?? [], owned.designMotifs ?? [])`
       - motifDelta = `1 - motifJaccard`

    2. **Fallback test:** If `[formalityDelta, sportinessDelta, heritageDelta].every(d => d === null || d < 0.1)` AND `archetypeDelta === 0` AND `eraDelta === 0` AND `motifJaccard >= 0.8` → return `'Very similar across all taste dimensions'`.

    3. **Find the winning dimension:** Build a candidate list of `{ name, delta }` pairs, excluding any with `delta === null`. The scalars contribute their raw absolute delta; archetype and era contribute `1` if different else are excluded; motif contributes `motifDelta`. Pick the entry with the maximum `delta`. Ties are broken by deterministic dimension priority: `formality > sportiness > heritageScore > primaryArchetype > eraSignal > designMotifs` (the order in `CatalogTasteAttributes`). Implementation-wise: iterate in that fixed order; track `best` only when `current > best`, so ties keep the earlier entry.

    4. **Emit phrase** per the 10 copy templates above. For scalars, use the `candidate > owned` / `candidate < owned` comparison to choose direction. For archetype/era diff, use `displayEnum(value)` to format the enum (replace `_` with space, capitalize each word) — handles `null` by rendering `'—'`.

    5. **Edge case:** If the fallback test fails AND every dimension delta is `null` (all scalars null, archetype/era null on both sides, motifs both empty), return the fallback string (degenerate case; safest user-facing default).

    Helper: `jaccardSimilarity(a: string[], b: string[]): number`:
    - Both empty → return `1` (identical sets)
    - Otherwise: `intersection.size / union.size`; if `union.size === 0`, return `1`

    Helper: `displayEnum(val: string | null): string` — `val ?? '—'`, then `.replace(/_/g, ' ').replace(/\b\w/g, c => c.toUpperCase())`. Re-implemented locally (not imported from elsewhere) — only ~3 lines.

    Test scenarios (8 minimum in `tests/unit/lib/verdict/fit-delta.test.ts`):

    1. **Identical taste profiles** → "Very similar across all taste dimensions"
    2. **Formality-dominant scalar (candidate higher)** — candidate.formality=0.9, owned.formality=0.3, all else equal → "This is more formal"
    3. **Sportiness-dominant scalar (candidate lower)** — candidate.sportiness=0.2, owned.sportiness=0.8, all else equal → "This is less sport"
    4. **Heritage-dominant (candidate higher)** — heritageScore delta 0.7 dominates, all else within thresholds → "More heritage-leaning"
    5. **Archetype mismatch** — candidate.primaryArchetype='dive', owned.primaryArchetype='dress', all scalars equal, era matches, motifs identical → "Different archetype: Dive vs Dress"
    6. **Era mismatch** — candidate.eraSignal='vintage-leaning', owned.eraSignal='modern', all else within thresholds → "Different era: Vintage Leaning vs Modern"
    7. **Motif mismatch** — candidate.designMotifs=['brushed','polished'], owned.designMotifs=['matte','satin'] (jaccard=0), all else identical → "Different design motifs"
    8. **Null scalar gracefully excluded** — candidate.formality=null, owned.formality=0.5; sportiness delta=0.2 (the only non-null winner) → "This is more sport" or "This is less sport" depending on direction. Test asserts NO crash, NO NaN.
