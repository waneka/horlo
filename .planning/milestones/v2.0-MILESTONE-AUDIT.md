### E2E Flow Verification

**Flow A — New user signup → profile auto-created → 5-section home (complete)**

1. Supabase Auth creates `auth.users` row → existing `auth_users_sync` trigger creates `public.users` row
2. `on_public_user_created` trigger (Phase 7) fires → `handle_new_public_user()` inserts `profiles` + `profile_settings` rows (defaults: all 4 visibility flags = true)
3. User signs in → `proxy.ts` (Phase 4) enforces auth at edge
4. Lands on `/` → `src/app/page.tsx` calls `getCurrentUser()` + `Promise.all([getWatchesByUser, getWearRailForViewer])`
5. Renders 5 sections in L-01 order: WYWT rail → Collectors Like You → Network Activity → Personal Insights → Suggested Collectors

Status: **WIRED end-to-end**; Phase 7 HUMAN-UAT #1 confirms trigger behavior is still pending verification in prod (TD-08).

**Flow B — User A follows User B → User B logs wear → User A sees it in WYWT + Network Activity (complete)**

1. User A opens `/u/userB/collection` → `FollowButton` calls `followUser` Server Action → writes to `follows` table + `revalidatePath('/u/[username]', 'layout')`
2. User B visits `/u/userB/worn`, clicks `+ Log Today's Wear` → `WatchPickerDialog` → `markAsWorn` Server Action:
   - Writes `wear_events` row (Phase 7)
   - Fire-and-forget `logActivity('watch_worn', ...)` (Phase 7) — row in `activities`
3. User A loads `/`:
   - **WYWT rail**: `getWearRailForViewer(userA.id)` JOINs `wear_events × profile_settings × follows`; includes User B's tile (48h window, follow gate, `profile_public=true AND worn_public=true` per WR-02 hardening in plan 10-09)
   - **Network Activity feed**: `getFeedForUser(userA.id, null, 20)` JOINs `activities × follows × profile_settings`; RLS policy `activities_select_own_or_followed` + DAL per-event privacy gate both fire. Row shows `{userB} wore {watch}` with `timeAgo`

Status: **WIRED end-to-end LOCALLY**; the feed JOIN depends on `activities_select_own_or_followed` RLS policy which is **NOT YET IN PROD** (TD-01). **Until `supabase db push --linked --include-all` runs, Flow B's Network Activity panel returns empty in prod.**

**Flow C — User A opens watch from User B's public collection → detail page; private → 404 (complete)**

1. Public: `/u/userB/collection` shows owned watches → link to `/watch/{id}`
2. `src/app/watch/[id]/page.tsx` → `getWatchByIdForViewer(userA.id, watchId)`:
   - If User B's `profile_public=false` OR `collection_public=false` (for owned) OR `wishlist_public=false` (for wishlist) → returns `null` → `notFound()` → 404
   - Otherwise returns `{watch, isOwner: false}` → `WatchDetail` rendered with `viewerCanEdit=false` and `lastWornDate=null` (non-owner, T-RDB-03)

Status: **WIRED** (just landed in quick-task 260421-rdb, commit `0604e09`). Unit test `f34b50d` pins the privacy matrix.
