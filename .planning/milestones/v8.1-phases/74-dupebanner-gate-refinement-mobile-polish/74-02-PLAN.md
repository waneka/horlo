<objective>
Close MOB-01: tapping any input/textarea/select with computed font-size < 16px on iOS Safari triggers auto-zoom (per the v8.0 prod UAT report in 70-UAT.md Gaps + REQUIREMENTS.md verbatim guidance). Three known offenders ship raw native elements with `text-sm` (14px) overrides — CommentCompose textarea, CommentItem edit-comment textarea, and SearchEntry Combobox.Input.

Per CONTEXT.md D-06 + D-07 + D-08 + D-11 + D-12 the fix is THREE layers:

(1) **Global floor (D-06):** add an `@layer base { input, textarea, select { font-size: 1rem } }` block to src/app/globals.css. Catches any raw native element without an explicit Tailwind override (defense in depth). Specificity 0,0,1 — beaten by any utility class — so components that legitimately want a different size CAN still set it, but DOM-defaults get the safe 16px floor.

(2) **Override audit + rewrite (D-07):** the three user-facing offenders flip `text-sm` → `text-base md:text-sm` (16px mobile, 14px desktop — matches shadcn Input + Textarea primitives at src/components/ui/input.tsx:12 + src/components/ui/textarea.tsx:10). Admin-only files (ListEditorClient.tsx, CmsCoverUploader.tsx, MarkdownEditor.tsx) are explicitly out of scope per CONTEXT — admin tooling is not part of the user-facing mobile path. shadcn primitives are UNTOUCHED.

(3) **Static guards (D-11 + D-12):** two new fs-walking guards in tests/static/ that fire CI tripwires against regression. D-11 defends src/app/layout.tsx:29 viewport export against future easy-fix additions of `maximum-scale=1` / `user-scalable=no` (an a11y regression that defeats pinch-zoom — ROADMAP SC#3 explicitly forbids). D-12 defends the 3 user-facing files against className reverts that would re-introduce bare `text-sm` on a native form control. Both files MUST declare `// @vitest-environment node` on the first line per project_vitest_static_node_env (without the pragma jsdom externalizes node:fs and the guard silently passes on Vercel prebuild — cost a failed Phase 59 prod deploy, retrofitted in Phase 71).

Per CONTEXT.md D-08 + D-09 the viewport meta is NOT touched (the current `{ viewportFit: 'cover' }` is correct; the static guard just defends the explicit anti-properties). Pinch-zoom + auto-zoom verification is prod UAT only per feedback_mobile_ui_verify_on_prod — local jsdom cannot meaningfully test Safari runtime behavior.

Per CONTEXT.md D-15 this plan ships in the bundled v8.1 prod push alongside Phases 72 + 73 + the Plan 01 DUPE-04 fix — single Vercel deploy, single UAT walk covering all 6 items.

Output: 1 CSS edit + 3 className rewrites + 2 new static guard test files, across 6 files. Total surface ~30 LOC.

Purpose: User can focus any input field on iOS Safari without the page auto-zooming, and intentional pinch-zoom still works.
</objective>
