## Color

Pre-populated from `src/app/globals.css` CSS custom properties.

| Role | Token | Light value | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `--background` | `oklch(0.985 0.003 75)` | Page surface, sheet background |
| Secondary (30%) | `--card` / `--popover` | `oklch(1 0 0)` | Card surface, sheet `bg-popover` |
| Accent (10%) | `--accent` | `oklch(0.76 0.12 75)` | Liked ♥ chip fill (active state only) — see reserved-for list below |
| Destructive | `--destructive` | `oklch(0.55 0.22 27)` | `CommentCompose` char counter at ≥480 chars |

Accent reserved for:
- Active/liked state of the ♥ chip icon fill (`fill: currentColor` + `text-destructive` mirrors `LikeButton` — note `LikeButton` uses `text-destructive` for liked, matching the red heart semantic; the accent token (warm gold) is used for the wear badge; ♥ active state stays `text-destructive` for semantic heart-red consistency).
- `--ring` (`oklch(0.76 0.12 75)`) is used for focus rings on chip buttons and the `CommentCompose` textarea.

**Chip scrim — CSS chain assertion (MEMORY `feedback_ui_spec_css_chain_blind_spot`):**

The scrim is a `<div>` (or `::before`) with class `bg-black/55` positioned `absolute inset-x-0 bottom-0 h-12` behind the chips, inside the `relative aspect-square` image wrapper. The full chain is:

```
.relative.aspect-square.bg-muted          ← image container (overflow clipped)
  └── <Image fill className="object-cover" />  ← photo fills container
  └── <div className="absolute inset-x-0 bottom-0 h-12 bg-black/55" />  ← scrim layer
  └── <div className="absolute bottom-2 left-2 flex gap-2" />            ← chips container
       └── chip buttons (white text+icon on black/55 scrim)
```

Contrast chain: white (`#fff`) on `rgba(0,0,0,0.55)` = effective background luminance ≈ 0.05 → contrast ratio ≈ 18:1. Passes WCAG AA (4.5:1) and AAA (7:1) for the 12px tabular-nums count text. Legibility is photo-independent because the scrim is opaque at 55%.

**Dark mode:** Chip scrim is `bg-black/55` in both modes (it overlays the image, not the theme surface). Chip icon+count text is `text-white` in both modes. No dark-mode override needed.

## Component Inventory

New component introduced by this phase:

### `GridLikeChip` (inline in `ProfileWatchCard` or extracted)

An overlay pill button rendering ♥ + count. Mirrors `LikeButton` optimistic pattern but as a scrimmed pill over the image.

- Visual: `rounded-full bg-black/30 px-2 py-1 flex items-center gap-1 text-white text-xs tabular-nums`
  - Note: the scrim div behind the chips handles the dark background; the individual chip button background is transparent or `bg-black/30` for the pill shape.
- Tap target: button `min-h-[44px] min-w-[44px]` with icon sized `size-4` (16px).
- Liked state: `Heart` with `fill="currentColor" className="text-red-400"` (warm red, matches `LikeButton`'s `text-destructive` adapted for white-on-dark context).
- Unliked state: `Heart` with `fill="none" className="text-white/90"`.
- Pending state: `opacity-50 cursor-wait` (mirrors `LikeButton`).
- Count visibility: render count span only when `count > 0 || liked` (mirrors `LikeButton:114`).
- Event handling: `onClick` calls `e.preventDefault(); e.stopPropagation()` BEFORE any state change (D-02).
- Owner gate: rendered only when `!isOwner` (D-03). Owner cards keep existing static count line at lines 117–136 of `ProfileWatchCard.tsx` — unchanged.

### `GridCommentChip` (inline in `ProfileWatchCard` or extracted)

An overlay pill button rendering 💬 + count. Triggers the compose sheet.

- Visual: same pill treatment as `GridLikeChip` — `rounded-full bg-black/30 px-2 py-1 flex items-center gap-1 text-white text-xs tabular-nums`.
- Icon: `MessageCircle size-4 text-white/90`.
- Count visibility: render count span only when `commentCount > 0`.
- Visibility gate: rendered only when `canComment === true` (D-09). For gated viewers on foreign wishlist cards, chip is hidden entirely — no locked/teaser affordance.
- Event handling: `onClick` calls `e.preventDefault(); e.stopPropagation(); setSheetOpen(true)` (D-02/D-06).
- Owner gate: same as `GridLikeChip` — only non-owner viewers see this chip (D-03).

### `WatchCommentSheet` (new file: `src/components/watch/WatchCommentSheet.tsx`)

A compose-only bottom sheet for posting a comment on a watch from the grid. Does NOT render `CommentList` or `CommentThread` (GRID-04).

Discretion decision (from `Claude's Discretion` in CONTEXT.md): **Option A** — new lightweight `WatchCommentSheet` using `Sheet` + `SheetContent` + `CommentCompose` directly. Rationale: GRID-04 requires compose-only; `WearCommentHost` hardcodes `type: 'wear'` and renders a full `CommentList`; a new wrapper is cleaner and avoids props surgery on an unrelated component. Aligned with RESEARCH.md Pattern 4 recommendation.

Sheet structure:

```
<Sheet open={open} onOpenChange={onOpenChange}>
  <SheetContent side="bottom">
    <SheetHeader>
      <div className="flex items-center gap-3">
        {/* watch thumbnail — 48×48 rounded-md */}
        <div className="relative w-12 h-12 rounded-md overflow-hidden bg-muted flex-shrink-0">
          <Image src={thumbnailUrl} alt={...} fill className="object-cover" />
        </div>
        <div>
          <p className="text-sm text-muted-foreground truncate">{brand}</p>
          <p className="text-base font-semibold leading-tight truncate">{model}</p>
        </div>
      </div>
    </SheetHeader>
    <div className="px-4 pb-4">
      <CommentCompose
        key={composeKey}        {/* re-mount on success to clear textarea (Pitfall 7) */}
        viewerId={viewerId}
        pending={pending}
        onSubmit={handleSubmit}
      />
    </div>
  </SheetContent>
</Sheet>
```

Sheet identity header: watch thumbnail (48×48px, `rounded-md`, `object-cover`) + brand (14px/400) + model (16px/600 `leading-tight`). No count, no thread. Sheet title is sr-only `"Add a comment"` for accessibility.

Post success behavior: close sheet (`onOpenChange(false)`) + increment `composeKey` (clear textarea) + bump local `commentCount` optimistically + fire `toast('Comment posted')` (D-07).
Post failure behavior: roll back optimistic count bump + keep typed text (do NOT increment `composeKey`) + fire `toast.error('Failed to post comment. Please try again.')` (D-08).

### Chip container placement within `ProfileWatchCard`

The `relative aspect-square bg-muted` image div gains:
1. A scrim layer: `<div className="absolute inset-x-0 bottom-0 h-12 bg-black/55 pointer-events-none" />` — positioned below the chips in the DOM so chips receive pointer events.
2. A chip row: `<div className="absolute bottom-2 left-2 flex gap-2 z-10">` containing `GridLikeChip` + `GridCommentChip`.

Both are rendered only when `!isOwner` (D-03). When `isOwner`, the image div is unchanged (no scrim, no chips).

The existing `wear` badge at `absolute top-2 left-2` is for owned/worn watches only (`!isWishlistLike && (isWornToday || isStale)`). Chips are at `bottom-2 left-2` — no collision at any watch status combination.

**Link restructuring decision (from `Claude's Discretion` in CONTEXT.md):** Use `e.preventDefault() + e.stopPropagation()` on chip click handlers rather than restructuring the `<Link>` wrapper. Rationale: the Link restructure would require extracting the card content from the link and managing two separate interactive regions, which is higher complexity and risk. The `preventDefault`/`stopPropagation` approach is already validated by the `LikeButton` pattern on the detail page where the button sits inside anchor-adjacent context.

## Interaction Contract

### Chip Tap — ♥ Like (D-04/D-05)

1. Tap ♥ chip → `e.preventDefault(); e.stopPropagation()`.
2. Optimistic: flip `liked`, increment/decrement `count`. Button becomes `disabled` (`pending=true`).
3. Fire `toggleLikeAction({ type: 'watch', id: watch.id })`.
4. On success: reconcile `liked` + `count` from server response.
5. On failure: roll back `liked` + `count` silently (no toast). Log `console.error`.
6. Card does NOT navigate. The rest of the card remains a `<Link>` to `/w/${watch.id}`.

### Chip Tap — 💬 Comment (D-06/D-07/D-08)

1. Tap 💬 chip → `e.preventDefault(); e.stopPropagation()`.
2. Set `sheetOpen = true`. `WatchCommentSheet` renders with `open={true}`.
3. User types in `CommentCompose` textarea (max 500 chars; char counter appears at ≥450).
4. Tap "Post" → `handleSubmit` fires `addCommentAction`.
5. Optimistic: bump local `commentCount` by 1 before action resolves.
6. On success: close sheet, increment `composeKey` (clear textarea), fire `toast('Comment posted')`.
7. On failure: roll back `commentCount` bump, keep textarea text, fire `toast.error('Failed to post comment. Please try again.')`.
8. Sheet close (X button or backdrop tap): do NOT clear textarea if post was not submitted — preserve any in-progress draft.

### State Reset on Navigation (MEMORY `project_router_cache_stale_instance`)

`sheetOpen` is controlled state (`onOpenChange` closes it) — no special reset needed; closing the sheet sets it to false. The optimistic `liked`/`count` state is seeded from RSC-resolved props on fresh navigation. Cache-tag bust (`viewer:{userId}:counts`) ensures fresh `liked` values on return navigation (D-12).

### Owner Cards (D-03)

Owner sees NO chips, NO scrim. The existing static count line (lines 117–136 of `ProfileWatchCard.tsx`) remains unchanged for owner cards. The `isOwner` prop gates all chip rendering.

### Gated Wishlist Cards (D-09)

Non-mutual viewer on a foreign wishlist card: `canComment = false` from batched counts. 💬 chip is NOT rendered. ♥ chip is rendered and interactive (likes are ungated — GATE-02). No locked/teaser state for the hidden comment chip.

| Failure toast | **Failed to post comment. Please try again.** — problem + action (D-08) |
| Like failure | Silent rollback — no toast (D-05, mirrors `LikeButton` behavior) |
| Empty comment post blocked | Post button disabled when `body.trim().length === 0` — no error message needed (button is disabled) |
| Char counter | **{N}/500** — appears at ≥450 chars; `text-muted-foreground` below 480, `text-destructive` at ≥480 (matches `CommentCompose` existing behavior) |

No destructive actions in this phase. No confirmation dialogs. No empty-state surfaces (chips only appear when the viewer can engage; no empty grid state is introduced).

## Accessibility Contract

- Chip buttons: `aria-pressed={liked}` on ♥ chip (mirrors `LikeButton`); `aria-busy={pending}` when pending; `aria-label` per table above.
- Touch targets: chip buttons `min-h-[44px] min-w-[44px]` — meets WCAG 2.5.5.
- Sheet: `SheetPrimitive.Title` (sr-only "Add a comment") for screen reader announcement on open.
- Keyboard: Sheet dismisses on Escape (handled by `@base-ui/react Dialog`).
- Focus: on sheet open, focus moves to the textarea inside `CommentCompose` (standard `Dialog` auto-focus behavior from `@base-ui/react`).
- Optimistic transitions: `aria-busy={pending}` on chip button during Server Action; `disabled={pending}` prevents double-tap.

## CSS Chain Assertions

Per MEMORY `feedback_ui_spec_css_chain_blind_spot`: the following chains are explicitly asserted, not just declared.

### Chip scrim legibility chain

```
ProfileWatchCard image container:
  className="relative aspect-square bg-muted"        ← sets positioning context + fallback bg

  <Image fill className="object-cover" />             ← photo fills the square at 100%

  <div className="absolute inset-x-0 bottom-0
                  h-12 bg-black/55
                  pointer-events-none" />             ← scrim: full-width, 48px tall, 55% opacity black
                                                        overlaid on photo pixels

  <div className="absolute bottom-2 left-2
                  z-10 flex gap-2" />                 ← chip row above scrim (z-10 > default z-index)
    └── <button className="rounded-full bg-black/30
                            px-2 py-1 ...
                            text-white text-xs" />    ← pill on scrim; white text on ~rgba(0,0,0,0.69)
                                                        effective (0.55 scrim + 0.30 pill = additive)
```

Effective contrast: white on rgba(0,0,0,0.55+) background. Even at the lower scrim-only value (0.55), white on #000 at 55% opacity over a mid-gray photo = WCAG AA passes.

### Sheet bottom-side slide chain

```
SheetContent side="bottom":
  data-[side=bottom]:inset-x-0 data-[side=bottom]:bottom-0 data-[side=bottom]:h-auto
  data-[side=bottom]:border-t
  data-starting-style:translate-y-[2.5rem] data-ending-style:translate-y-[2.5rem]   ← slide in/out
  bg-popover                                                                          ← sheet background
  text-popover-foreground                                                             ← text color
```

Sheet uses `h-auto` (content-height driven), not full viewport height. On mobile the sheet does not cover the full screen — it shows the identity header + CommentCompose. This is intentional (compose-only; no scrollable thread to accommodate).
