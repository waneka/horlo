<objective>
Close the single cosmetic UAT gap from Phase 62 (Test 4, WPIC-06 discoverability). Relocate the wear-pic like + comment-count controls from the standalone stacked row below the carousel INTO the per-slide wear-pic map as an absolute overlay anchored bottom-right of the photo. This is a JSX-position-only move within ONE file — no data, props, server actions, or event handlers change.

Purpose: A user reported "I didn't notice the like/comment icons at first... it's not obvious they even apply to the wear pic you're seeing." The controls currently render two elements removed from the photo (below the carousel viewport AND below the position indicator), breaking the visual association. Overlaying them on the photo's bottom-right corner ties them to the wear pic on screen. This reverses a deliberate 62-UI-SPEC Surface-2 "beneath the slide" discretion call that UAT overruled.
Output: WatchPhotoSection.tsx with a per-slide bottom-right social overlay; standalone row deleted; build exits 0; 54 unit tests green; final visual confirmation deferred to prod human re-check of UAT Test 4.
</objective>
