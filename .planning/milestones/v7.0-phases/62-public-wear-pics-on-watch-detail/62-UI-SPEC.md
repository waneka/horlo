## Surface 1: "Worn · [date]" Badge

**Source:** D-05, D-07, CONTEXT.md. Appearance and placement are Claude's discretion.

**Placement:** Bottom-left corner of the carousel slide, above the filmstrip. Overlaid on the photo.

**Visual treatment:**
- `absolute bottom-2 left-2` positioning within the slide's `relative` container
- Background: `bg-background/80 backdrop-blur-sm` (matches the existing arrow-button overlay style for visual consistency)
- Text: `text-xs font-semibold text-foreground` (12px/600 — matches the "Cover" badge in SortablePhotoThumb)
- Padding: `px-2 py-0.5` (8px horizontal, 2px vertical)
- Border radius: `rounded` (radius-sm = 0.375rem)
- Content: "Worn · May 26" format (abbreviated month + day, no year unless cross-year display)
- Date format: `new Date(wornDate + 'T00:00:00Z').toLocaleDateString('en-US', { timeZone: 'UTC', month: 'short', day: 'numeric' })` — MANDATORY per D-07 React #418 guard
- Owner upload slides: no badge (blank)
- Catalog fallback slide: no badge (blank)

**Contrast requirement:** `text-foreground` on `bg-background/80` over a photo. The semi-opaque white/dark background ensures minimum 4.5:1 contrast against the text in both light and dark modes.

**Filmstrip reflection:** The filmstrip thumbnail for a wear-pic slide does not carry its own date badge (too small at 64px). The badge is on the full-size carousel slide only.

## Surface 2: Inline Like Toggle + Comment Count on Active Wear-Pic Slide

**Source:** D-14. Placement is Claude's discretion.

**Placement decision:** BENEATH the active slide, not overlaid ON the slide. Rationale: overlaying on the photo would compete visually with the "Worn · [date]" badge and obscure the photo content. A row just beneath the carousel (between the carousel viewport and the position indicator) is cleaner and mirrors existing WearCard social row patterns.

**Layout:** A single flex row, `flex items-center gap-2 px-0` aligned to the left edge, sitting between the carousel viewport and the position indicator `(N / total)`. This row is only rendered when `selectedIndex` points to a wear-pic slide; it is invisible (rendered as `null`) on owner-upload slides and the catalog fallback slide.

**Like button:**
- Reuse `LikeButton` component as-is, `target={{ type: 'wear', id: wearEventId }}`
- `min-h-[44px] min-w-[44px]` touch target (already built into LikeButton)
- Liked state: Heart filled, `text-destructive` (existing LikeButton contract)
- Unliked state: Heart outline, `text-muted-foreground`
- Count: `text-sm tabular-nums`, `text-destructive` when liked, `text-muted-foreground` when not
- Count hidden when count === 0 and !liked (existing LikeButton rule)

**Comment count button:**
- A `<button>` element with `min-h-[44px] min-w-[44px] px-2 rounded-md` (matches LikeButton outer shell)
- Icon: `MessageCircle` from lucide-react, `size-5 text-muted-foreground`
- Count: `text-sm tabular-nums text-muted-foreground` shown when count > 0
- Count hidden when count === 0
- `aria-label`: "View comments (N)" when count > 0, "Add a comment" when 0
- On tap/click: opens the wear-pic comment bottom sheet (Surface 3)

**Social row visibility rule:** The entire social row (like + comment) mounts and unmounts based on `isWearPicSlide(selectedIndex)`. No transition animation needed — the row simply appears/disappears as the carousel scrolls between owner and wear-pic slides.

**Non-owner viewers:** Social row is fully visible and interactive for non-owners (this is the point of WPIC-06). `viewerCanEdit` does not gate the social row.

## Surface 3: Wear-Pic Comment Bottom Sheet

**Source:** D-15. Sheet primitive is Claude's discretion.

**Primitive:** `WearCommentHost` with `variant="bottom-sheet"` — reuse as-is. No new sheet component. The existing Sheet is `SheetContent side="bottom"` with `bg-background max-h-[60vh] overflow-y-auto z-50` and `pb-[calc(1.5rem+env(safe-area-inset-bottom))]`. Do NOT change this CSS chain.

**Open trigger:** Tapping the comment count button on the active wear-pic slide (Surface 2).

**Dismiss gestures:**
- Tap the overlay/scrim behind the sheet
- Swipe down on the sheet (Sheet component built-in behavior)
- No explicit close button required (shadcn Sheet provides scrim dismiss)

**Cache Components contract:** The WearCommentHost is a client component ('use client'). Initial comments are pre-fetched server-side in the page RSC and passed as `initialComments` props per Option A in RESEARCH.md. No 'use cache' directive anywhere in the comment path. The `unstable_instant = false` on `/w/[ref]/page.tsx` must not be disturbed.

**Sheet title:** "Comments" (existing SheetTitle in WearCommentHost — no change).

**Count sync:** Use `onCountChange` callback on `WearCommentHost` to propagate comment delta back to `WatchPhotoSection` state, matching the existing WearCard.tsx pattern. This keeps the comment count badge on the carousel social row in sync after the user posts without a page reload.

## Surface 4: Per-Pic Hide Control in Edit Mode

**Source:** D-08, D-10. Eye icon, hidden treatment, and toggle affordance are Claude's discretion.

**Edit mode activation:** Unchanged from Phase 61 — the "Edit photos" toggle button.

**Filmstrip rendering in Edit mode (wear-pic thumbnails):**

Wear-pic thumbnails in the filmstrip use a parallel non-sortable wrapper (not `SortablePhotoThumb` — wear pics are not owner uploads and cannot be reordered). They render at the same 64×64px size with the same `rounded-md overflow-hidden bg-muted` base.

**Visible (not hidden) wear-pic thumbnail in Edit mode:**
- Full-opacity image (`opacity-100`)
- Eye-slash icon button in the top-right corner (where owner thumbnails have the × delete button)
- Icon: `EyeOff` from lucide-react, `size-4`
- Button: `absolute top-1 right-1 size-6 bg-background/80 rounded-full flex items-center justify-center`
- `aria-label`: "Hide from this page"
- On tap: calls `hideWearPicAction`, optimistically greys the thumbnail

**Hidden wear-pic thumbnail in Edit mode:**
- `opacity-50` on the image (greyed)
- `bg-muted` base visible through the opacity wash
- "Hidden" text label: `absolute bottom-0 left-0 right-0 text-center text-[10px] font-semibold text-foreground bg-background/70 py-0.5` (full-width label strip at the bottom of the thumbnail)
- Eye icon button (top-right corner, same size/position as EyeOff) changes to `Eye` (visible/restore) icon
- `aria-label`: "Show on this page"
- On tap: calls `unhideWearPicAction`, optimistically restores opacity

**Wear-pic thumbnails in Edit mode have NO:**
- Drag handle (GripVertical) — wear pics are not reorderable
- Delete × button — wear pics cannot be deleted from the carousel (only hidden)
- "Cover" badge — never

**Owner upload thumbnails in Edit mode:** Unchanged from Phase 61. No eye/hide affordance on owner uploads.

**Optimistic update pattern:** `useOptimistic` (same pattern as Phase 61 delete). Flip `hiddenFromDetail` locally before server action resolves. On failure: revert + `toast.error("Couldn't update.")`.

**Touch target:** The EyeOff/Eye button at `size-6` (24px) is small. Expand the effective tap target using `p-1` padding or a wrapping `size-[36px]` element — minimum 36px. At 64px thumbnail size, 36px is the maximum the overlay can occupy without covering the photo entirely.

## Surface 5: Wears Tab Thumbnail (WPIC-03)

**Source:** D-16. Visual treatment is a minor change to WornTimeline/WornCalendar.

**Current behavior:** All wear events use the watch cover (`watch.imageUrl`) as the thumbnail.

**Phase 62 behavior:** Prefer `event.photoUrl` (signed wear-photo URL) when present; fall back to `watch.imageUrl` (catalog/cover) when the wear event has no photo.

**Visual treatment:** No change to the thumbnail rendering itself — same `<Image>` or `getSafeImageUrl` pattern. The only change is the source priority.

**Fallback visual:** When `event.photoUrl` is null or signing fails (returns null), the existing catalog/cover image is shown. No new placeholder or empty-state needed.

**No caption or badge needed:** The thumbnail swap is invisible to the user as a design change — it just shows the right photo.

## Copywriting Contract

| Element | Copy |
|---------|------|
| "Worn · [date]" badge | "Worn · [abbreviated date]" — e.g. "Worn · May 26" |
| Hide from page (optimistic toast — success) | "Hidden from this page" (no undo — reversible in Edit mode) |
| Unhide (optimistic toast — success) | "Shown on this page" |
| Hide action fail | "Couldn't update. Try again." |
| Unhide action fail | "Couldn't update. Try again." |
| Eye button (visible state) aria-label | "Hide from this page" |
| Eye button (hidden state) aria-label | "Show on this page" |
| Comment count button aria-label (count > 0) | "View {N} comment{s}" |
| Comment count button aria-label (count = 0) | "Add a comment" |
| Like button aria-label (liked) | "Unlike" (existing LikeButton) |
| Like button aria-label (unliked) | "Like" (existing LikeButton) |
| Sheet title | "Comments" (existing WearCommentHost, unchanged) |
| "Hidden" thumbnail label | "Hidden" |
| Empty wear-pic comment state | "No comments yet." (existing CommentList empty state) |
| Wear-pic like error | Silent rollback, no toast (existing LikeButton contract) |

**No new destructive actions.** Hide is not destructive — it is reversible in the same Edit mode without a confirmation dialog. The existing photo delete confirmation pattern (undo toast) does not apply here.

**No new empty states.** If a watch has no public wear pics, the carousel simply shows owner photos only (or the catalog fallback), with no "no wear pics" message. The page does not surface the absence of wear pics to viewers.

## CSS Chain Assertions

These must hold after implementation (lesson from GSD UI-SPEC CSS chain blind spot — MEMORY feedback_ui_spec_css_chain_blind_spot):

1. **"Worn · [date]" badge** — `bg-background/80` must produce a semi-opaque overlay (not transparent, not solid). `text-foreground` on `bg-background/80` must not result in near-zero contrast. Assert the badge is legible over a dark and a light photo.

2. **Wear-pic comment sheet** — `SheetContent` must retain `bg-background` (SOLID, not `/80` opacity). Existing class `bg-background max-h-[60vh] overflow-y-auto z-50` must not be modified. CSS chain: Sheet → SheetContent side="bottom" → inner div with max-w-[640px] px-4 pb-[calc(1.5rem+env(safe-area-inset-bottom))].

3. **Hidden thumbnail** — `opacity-50` on the `<Image>` element (not the container) so the `bg-muted` base shows through as a grey wash. If `opacity-50` is on the container, the "Hidden" text label also becomes 50% opaque — apply to image only.

4. **Social row visibility** — The like/comment row must not leave a blank gap when not rendered. Use conditional render (`isWearPicSlide ? <SocialRow /> : null`), not `visibility: hidden` or `opacity: 0`, so the layout does not reserve space for it.

5. **Carousel position indicator** — The `N / total` counter must count ALL slides (owner + wear-pic merged array). Verify the total reflects the union, not just owner photo count.
