<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anon/authenticated Supabase client → watch_photos | Untrusted; RLS gates direct reads/writes |
| anon/authenticated Supabase client → storage.objects (watch-photos bucket) | Untrusted; folder RLS gates uploads to the caller's own `{uid}/` prefix |
| local DB (RLS OFF on pre-existing tables) → prod DB (RLS ON) | Cross-table RLS behavior differs; verify on prod (Plan 04) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-60-01 | Tampering | backfill→DROP ordering in migration | mitigate | Backfill INSERT + DO $$ lossless assert run BEFORE `DROP COLUMN`; transaction-wrapped; idempotent via `ON CONFLICT DO NOTHING` + `DROP COLUMN IF EXISTS` (D-07) |
| T-60-02 | Elevation of Privilege | storage.objects watch-photos bucket | mitigate | Folder RLS `(storage.foldername(name))[1] = (SELECT auth.uid())::text` — caller can only write/read their own `{uid}/` prefix (Phase 11 analog) |
| T-60-03 | Information Disclosure | watch_photos cross-tenant read via direct client | accept (defense-in-depth via service-role DAL) | RLS SELECT policy subqueries owner-only `watches` → fails closed for non-owners (project_rls_subquery_caller_rls). Service-role DAL is the real read gate; this RLS is anon-block backstop. Local RLS OFF → not testable locally; verify on prod (Plan 04). |
| T-60-04 | Spoofing | watch_photos INSERT for another user's watch via direct client | mitigate | INSERT RLS `WITH CHECK EXISTS(... w.user_id = auth.uid())`; DAL ownership check is the primary gate (Plan 03) |
</threat_model>
