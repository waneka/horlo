| user-supplied image bytes → stripAndResize | Untrusted EXIF/dimensions; re-encode strips metadata + caps size |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-60-TRAVERSAL | Elevation of Privilege | buildWatchPhotoPath | mitigate | `photoId` validated against `UUID_RE` before path construction; a crafted `../` photoId fails the UUID test and throws. Storage folder RLS (`(storage.foldername(name))[1] = auth.uid()`) is the DB-side backstop (Plan 01). |
| T-60-EXIF | Information Disclosure | uploaded photo EXIF (GPS, device) | mitigate | `stripAndResize` re-encodes via canvas.toBlob — all EXIF (incl. GPS) is dropped before upload; SC4 test pins this invariant (D-16) |
| T-60-UPSERT | Tampering | uploadWatchPhoto overwriting another object | mitigate | `upsert: false` — each photo gets a fresh UUID-stamped path; no overwrite of existing objects (wearPhotos precedent) |
</threat_model>
