<!-- The Chip primitive's contract — Plan 03 (Wave 2) implements against this. -->

The primitive MUST export:
- `Chip` — React component (named export); renders <button type="button">; accepts standard ButtonHTMLAttributes<HTMLButtonElement> plus chip-specific props.
- `chipVariants` — the cva() function (named export); enables unit-test introspection and ad-hoc reuse.
- `ChipVariants` — `VariantProps<typeof chipVariants>` type (named export).

Public prop surface (the planner picks the API shape — these are the contracts callers in Plan 03 will use):
- `variant: 'toggle' | 'removable'` (default: 'toggle')
- `selected?: boolean` (only meaningful for variant='toggle'; drives the selected vs unselected class set)
- `removeLabel?: string` (only meaningful for variant='removable'; becomes the sr-only "Remove X filter" text)
- `className?: string` (merged via cn() — caller overrides)
- All standard <button> props pass through (onClick, aria-pressed, type, disabled, etc.)
- `children: React.ReactNode` (the chip label)

The primitive is STATELESS — it does NOT track selection. Callers (BrandChips, StyleChips, etc.) own selection logic:
- BrandChips: `selected={selected === entry.slug}` (single-select null toggle)
- StyleChips: `selected={selected.includes(tag)}` (multi-select array membership)
- SearchPageClient removable chips: no `selected` prop; rendered as always-active.

CSS contract — from UI-SPEC.md §"Chip Primitive Visual Contract" + PATTERNS.md (concrete class strings, MUST match):

Shared base (both variants):
  rounded-full border px-3 py-1 text-sm transition-colors
  focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-1

toggle unselected:
  bg-secondary text-secondary-foreground border-border hover:bg-muted font-normal

toggle selected (applied when selected={true}):
  bg-accent text-accent-foreground border-accent font-semibold

removable (BUG-02 fix lives here):
  gap-1 bg-accent/10 border-accent text-foreground font-semibold hover:bg-accent/20
  // ^^^ text-foreground replaces the broken text-accent-foreground (D-05)

Internal markup when variant='removable':
  Renders {children} followed by <X className="size-3" aria-hidden /> followed by <span className="sr-only">{removeLabel}</span>.
  X is imported from 'lucide-react' (already a project dependency).
  When removeLabel is undefined or empty string, the sr-only span is omitted (no empty announcement).

Pattern source (project convention — copy exactly):
- src/components/ui/badge.tsx — the CVA + cn() pattern. Note: badge.tsx ALSO imports `mergeProps` and `useRender` from `@base-ui/react` for its polymorphic span; the Chip primitive does NOT need these — it renders a plain <button>. Copy ONLY the `cva` and `cn` imports.
- src/components/search/BrandChips.tsx (current implementation) — the inline className strings being consolidated.

Required directive: `'use client'` on line 1 — the primitive is consumed by FilterDrawer chips and SearchPageClient.tsx (both client components) and renders an interactive button. badge.tsx omits 'use client' because it is a display-only span; this primitive cannot.
