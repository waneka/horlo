<objective>
Re-gate `/u/*` to authenticated viewers (Branch B):
1. Remove the `&& !isProfile` clause from the auth-gate predicate in `src/proxy.ts` (revert recurrence-2 fix commit `5def872`).
2. Remove the `isProfile` local variable and the `isProfilePath` import.
3. Add `Cache-Control: no-store` header to the `NextResponse.redirect()` to `/login` — this prevents Next 16's Router Cache from storing the 307 response on RSC prefetches.
4. Delete the now-unused `isProfilePath` function from `src/lib/constants/public-paths.ts`.

**Hard prerequisite:** Plan 51-04 (cookie-only proxy refactor) MUST have shipped first. Without 51-04, this plan re-introduces the recurrence-2 root cause (`getUser()` network round-trip → transient null → 307 → cached → 404).

Purpose: Restore the pre-recurrence-2 invariant — anon viewers cannot reach `/u/*` — while ensuring no 307 response from this path can poison the Router Cache (the `Cache-Control: no-store` header is the safety net beyond cookie-only auth).

**Note on `LockedProfileState` for anon viewers:** After Branch B, `viewerId === null` becomes unreachable for `/u/*` page renders (anon viewers are redirected at the proxy before reaching the page). The `viewerId === null` branch inside `ProfileGate` and `[tab]/page.tsx` is dead from the page-render perspective. HOWEVER, we keep the null-handling code in place as defense in depth:
- Build-time prerendering may still invoke gate code without a session (the layout has no session lookup; the page calls `getCurrentUser().catch(() => null)`).
- Internal callers (RSC internals, test fixtures) may invoke gate code with `viewerId=null`.
- Removing the null branch invites a future regression if the proxy ever ungates again.

**No code removal is performed in this plan** beyond `isProfilePath` and `isProfile`. Audit-driven removal of `LockedProfileState` and `viewerId === null` branches is OUT OF SCOPE — see "Out of Scope" in 51-PLAN.md.

Output: Two-file edit. The Phase 51 test suite continues to pass. After this plan, the prod contract check (plan 51-06) must verify REQ-51-07 (anon → 307 with no-store).
</objective>
