## Spacing Scale

Standard 8-point scale. No new exceptions introduced by this phase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, avatar-to-text gap in notification row |
| sm | 8px | Intra-row element spacing, toggle padding |
| md | 16px | Row horizontal padding (`px-4`), section content padding |
| lg | 24px | Section vertical spacing (`mb-6` on `SettingsSection`) |
| xl | 32px | Page-level vertical padding |
| 2xl | 48px | Target row height for `NotificationRow` (min-h-12 = 48px) |
| 3xl | 64px | Major page breaks |

Exceptions:
- Notification row minimum height: 48px (`min-h-12`) — matches existing `PrivacyToggleRow` and `ActivityRow` patterns exactly.
- Bell unread dot: 8px diameter (`size-2`), positioned `top-0 right-0` relative to bell icon container.
- Touch target for the bell icon wrapper: 44px (`min-h-11 min-w-11`) to meet iOS tap-target guidelines.

## Color

All values are semantic tokens resolved from `src/app/globals.css`. Both light and dark modes are covered by the existing theme.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` | Page background, `/notifications` page surface |
| Secondary (30%) | `bg-card` | Notification row background (read state), settings section container (`rounded-xl border bg-card`) |
| Accent (10%) | `bg-accent` / `border-accent` | Unread row left border only; toggle ON state; bell unread dot |
| Muted surface | `bg-muted/40` | Row hover/focus-within state (matches `ActivityRow` pattern) |
| Muted foreground | `text-muted-foreground` | Relative time stamp, group subheader labels, empty state body copy, settings description text |
| Destructive | `text-destructive` / `bg-destructive` | Not applicable to Phase 13 — no destructive actions in this phase |

Accent reserved for:
1. Unread notification row left border (`border-l-2 border-l-accent`)
2. Bell unread dot fill (`bg-accent`)
3. Settings toggle switch ON state (existing `PrivacyToggleRow` uses `bg-accent` — reused as-is)

Accent is NOT used for: links, read rows, group headers, the "Mark all read" button, or the empty state icon.

Unread row treatment (D-14):
```
border-l-2 border-l-accent bg-card
```
Read row treatment:
```
bg-card (no left border)
```
Both states show `hover:bg-muted/40` on pointer hover and `focus-within:bg-muted/40` on keyboard focus.

## NotificationRow Anatomy

Source: CONTEXT.md D-12, D-13, D-14, D-15.

```
[AvatarDisplay size=40]  [Actor name]  verb  object  ·  relative-time
```

Full-row click semantics (D-08): entire row is a Client Component that calls a Server Action (mark-one-read) optimistically then calls `router.push(target)`. The actor avatar and name are visually presented as a link but resolve to the same target as the full-row click.

Structural classes for the row container:
```
group relative flex items-center gap-3 min-h-12 bg-card px-4 py-2 transition-colors hover:bg-muted/40 focus-within:bg-muted/40
```

Unread state adds:
```
border-l-2 border-l-accent
```

Actor name span:
- Read: `font-medium text-foreground`
- Unread: `font-semibold text-foreground`

Relative time:
```
text-sm text-muted-foreground
```

Separator between object and time: `·` (middle dot, U+00B7), rendered as a text node with surrounding spaces.

### Type-specific copy (locked by CONTEXT.md D-20)

| Type | Copy pattern |
|------|-------------|
| `follow` | `**{actorName}** started following you · {timeAgo}` |
| `watch_overlap` (actor_count = 1) | `**{actorName}** also owns your **{watchModel}** · {timeAgo}` |
| `watch_overlap` (actor_count > 1) | `**{actorName}** + {N} others also own your **{watchModel}** · {timeAgo}` |
| `price_drop` (stub) | `Your **{watchModel}** wishlist watch dropped to **{newPrice}** · {timeAgo}` |
| `trending` (stub) | `**{actorCount} collectors** in your taste cluster added a **{watchModel}** this week · {timeAgo}` |

N for watch_overlap = `actor_count - 1`.

Unknown `type` values render `null` (silent no-op, not a broken card) per ROADMAP pitfall B-8.

### Click-through targets (D-09)

| Type | Target |
|------|--------|
| `follow` | `/u/{actorUsername}` |
| `watch_overlap` | `/u/{actorUsername}?focusWatch={watchId}` |
| `price_drop` | TBD — stub renders row but click target deferred to wiring phase |
| `trending` | TBD — same |

## `/notifications` Page Layout

Route: `src/app/notifications/page.tsx` — Server Component.

Structure:
1. Page heading: `<h1>Notifications</h1>` — `text-xl font-semibold` — left-aligned, top of page body.
2. "Mark all read" button — right-aligned in the heading row. Plain text button style: `text-sm text-muted-foreground hover:text-foreground`. No destructive color. Submits a Server Action (`markAllNotificationsRead`).
3. Inbox list — wrapped in a `<Suspense>` boundary. Shows last 50 rows, newest-first.
4. Group subheaders — "Today", "Yesterday", "Earlier" — sticky within their group. Empty buckets are omitted entirely. Style: `text-xs font-normal uppercase tracking-wide text-muted-foreground py-2 px-4 sticky top-0 bg-background z-10`.
5. Empty state — shown when zero notification rows exist (replaces the entire list).

The `NotificationBell` component is built and exported by this phase but placed in the nav by Phase 14. Phase 13 adds a temporary placement in the existing header (below the existing nav items) for UAT visibility only — clearly marked with a `{/* TEMP: UAT placement — Phase 14 will move this to nav */}` comment and removed when Phase 14 ships.

## NotificationBell Component

File: `src/components/notifications/NotificationBell.tsx`

Visual spec:
- Icon: `Bell` from lucide-react, `size-5` (20px)
- Unread dot: `absolute top-0 right-0 size-2 rounded-full bg-accent` — visible when `hasUnread === true`, absent otherwise
- Wrapper: `relative inline-flex items-center justify-center min-h-11 min-w-11` (44px touch target)
- The bell icon itself links to `/notifications`

DAL contract: `getNotificationsUnreadState(viewerId: string): Promise<{ hasUnread: boolean }>`. Returns only what the bell renderer needs — no `lastSeenAt` exposed to UI layer.

Cache: `'use cache'` with `cacheTag(['notifications', `viewer:${viewerId}`])` + `cacheLife({ revalidate: 30 })`. Source: CONTEXT.md D-25, D-26.

## Settings — Notifications Section

Section ordering on `/settings` after Phase 13:
1. Profile (existing)
2. Privacy Controls (existing)
3. **Notifications** (Phase 13 — replaces the current "Coming soon" stub)
4. Appearance (existing)
5. Data Preferences (existing)
6. Account (existing)

Section title: `"Notifications"` — passed as `title` prop to `<SettingsSection>`.

Content: two `PrivacyToggleRow` instances separated by `divide-y divide-border` (matching Privacy Controls pattern).

| Toggle | `label` prop | `description` prop | `field` prop |
|--------|-------------|-------------------|--------------|
| Follow notifications | `"New Followers"` | `"Get notified when someone starts following you."` | `"notifyOnFollow"` |
| Watch overlap notifications | `"Watch Overlaps"` | `"Get notified when another collector owns a watch you own."` | `"notifyOnWatchOverlap"` |

Both default ON (`DEFAULT true` in `profile_settings`). Opt-out is checked at write time in `logNotification` (CONTEXT.md D-18).

The `VisibilityField` type (currently `field` prop constraint on `PrivacyToggleRow`) must be extended to include `"notifyOnFollow"` and `"notifyOnWatchOverlap"`. The Server Action `updateProfileSettings` must handle these new field names by updating the corresponding `profile_settings` columns.

## Interaction States

### NotificationRow
| State | Visual |
|-------|--------|
| Unread | `border-l-2 border-l-accent bg-card` + actor name `font-semibold` |
| Read | `bg-card` (no left border) + actor name `font-medium` |
| Hover (pointer) | `hover:bg-muted/40` |
| Focus (keyboard) | `focus-within:bg-muted/40` + `focus-visible:ring-2 focus-visible:ring-ring` on full-row link |
| Pending (optimistic click) | Row immediately switches to read visual; no spinner shown |
| Unknown type | Renders `null` — no error UI, no broken card |

### NotificationBell
| State | Visual |
|-------|--------|
| Has unread | Bell icon + `size-2 bg-accent` dot |
| No unread | Bell icon only |
| Loading (Suspense fallback) | Bell icon without dot (safe default — never shows false positive) |

### "Mark all read" button
| State | Visual |
|-------|--------|
| Default | `text-sm text-muted-foreground` |
| Hover | `hover:text-foreground` |
| Pending (transition) | `opacity-60 pointer-events-none` |
| Success | Button remains; rows transition to read visual (revalidation updates server state) |
| Error | Inline `role="alert"` error message below the button |

### Settings toggles
Inherit all states from existing `PrivacyToggleRow`: optimistic toggle with snap-back on failure, `disabled:opacity-60` during pending transition.

## Accessibility

- Full-row link on `NotificationRow`: `aria-label="{actorName} notification"` on the overlay anchor.
- Bell unread dot: `aria-label="Unread notifications"` on the bell wrapper; dot itself `aria-hidden`.
- "Mark all read" button: standard `<button>` with visible label; `aria-disabled` when pending.
- Group subheaders: rendered as `<h2>` (not `<p>` or `<div>`) for screen-reader landmark traversal.
- Empty state: `<p>` with `role="status"` so screen readers announce when inbox transitions to empty.
- Settings toggles: inherit `role="switch"` + `aria-checked` from existing `PrivacyToggleRow`.
