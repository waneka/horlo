### Auto-fixed issues

**1. [Rule 3 - Blocking] Worktree absolute-path mistake on first Edit calls**

- **Found during:** Task 1 ‚Äî after running the failing CollectionFitCompareTable test and seeing it still report "expected 6 to be 5" despite the visible source change.
- **Issue:** The first two Edit calls used absolute paths of the form `/Users/tylerwaneka/Documents/horlo/src/components/insights/...`. Because the worktree lives at `/Users/tylerwaneka/Documents/horlo/.claude/worktrees/agent-a54a3d5af19fc7457/`, those absolute paths resolved to the MAIN REPO instead of the worktree ‚Äî the edits silently landed on `main`'s working tree, not on the per-agent branch. The worktree's copy of the files was unchanged, so the test (run from the worktree) continued to see the 6-row table.
- **Fix:** Reverted the misplaced main-repo edits via `cd /Users/tylerwaneka/Documents/horlo && git checkout -- src/components/insights/CollectionFitCompareTable.tsx src/components/insights/ReferenceIdentityCard.tsx tests/components/insights/ReferenceIdentityCard.test.tsx`. Then re-applied the same edits using the worktree-rooted absolute path `/Users/tylerwaneka/Documents/horlo/.claude/worktrees/agent-a54a3d5af19fc7457/src/...`. The system reminder confirmed the main repo was clean; the worktree-rooted edits then landed correctly and tests went GREEN.
- **Files modified:** No additional production-source changes ‚Äî purely a tooling-path recovery.
- **Commit:** N/A (recovered before the first Task 1 commit; both committed states reflect the correct worktree-rooted edits).
- **Prevention going forward:** Per `<absolute_path_safety>` guidance (#3099), derive absolute paths from `git rev-parse --show-toplevel` inside the worktree before passing to Edit/Write ‚Äî never reconstruct from a remembered `pwd`. Better: use relative paths from cwd (the worktree) for in-worktree files.
