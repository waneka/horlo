### D-39b-04 identical-rendering lock (REASSERTED)

The era-only headline MUST render identically on both `/watch/{id}` and `/catalog/{id}`. Both routes consume the same `ReferenceIdentityCard` component with the same `CatalogTasteAttributes` shape. The catalog route builds its `taste` literal via manual projection in `src/app/catalog/[catalogId]/page.tsx:90-99`; `primaryArchetype` is dropped from that projection in lockstep with the type definition in `src/lib/types.ts:208,224`. **Both projection sites change atomically** — verified by Phase 39b's `tests/static/ReferenceIdentityCard.no-engine.test.ts` and `tests/components/insights/ReferenceIdentityCard.test.tsx`.

### Edge cases (graceful degradation)

| Pre-phase state | Post-phase rendering |
|-----------------|----------------------|
| era set, archetype set | era only (archetype half deleted) |
| era set, archetype null | era only (UNCHANGED) |
| era null, archetype set | **headline disappears** — `hasHeadline = Boolean(eraLabel || null) = false` → block does not render. Card still renders scale-bars + motifs. |
| era null, archetype null | headline does not render (UNCHANGED) |
| `taste === null` or confidence `< 0.5` | entire card returns `null` (UNCHANGED — D-39b-03 confidence gate) |

The era-null + archetype-set case is a NEW gap behavior: today, archetype carries the headline; post-phase, the headline silently disappears. Per CONTEXT.md D-SCOPE-01a, this is acceptable: "accept truncation" is the locked decision and no surrogate is introduced.

## CSS Chain Assertions (per `feedback_ui_spec_css_chain_blind_spot` memory)

The 6-pillar checker validates declared tokens but not whether the CSS chain renders correctly. Phase 30 shipped 6/6 PASS while shipping a regressed black-bar bug. To prevent recurrence, the following CSS chains MUST be verified during implementation against a live dev server (NOT jsdom — see `project_turbopack_next_cache_stale_css`: clear `.next/` before verifying):

### Chain 1: era-only ReferenceIdentityCard headline (§A)

- DOM: `<p class="text-base font-semibold text-foreground"><span class="truncate">Modern era</span></p>`
- Resolved CSS: `font-size: 1rem; font-weight: 600; color: var(--foreground); white-space: nowrap; overflow: hidden; text-overflow: ellipsis;`
- Pass criteria: text reads "Modern era" (or analogous era label) at 16px semibold, single line, no trailing separator (` · `), no archetype tail. Card height adjusts naturally; scale-bars + motifs remain visible below.
