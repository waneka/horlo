## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 regular | 1.5 (`leading-normal`) |
| Label | 12px (`text-xs`) | 600 semibold | 1.25 (`leading-tight`) |
| Heading | 24px (`text-2xl`) | 600 semibold (`font-semibold`) | 1.2 (`leading-tight`) |
| Display | 36px (`text-4xl`) | 400 regular | 1.2 |

Two font weights only: **400 regular** and **600 semibold**. No weight-700 (`font-bold`) reference anywhere in this phase. Heading hierarchy is established by size differential (24px heading vs 14px body), not weight escalation. The heading Tailwind class is `font-semibold` — confirm this resolves to `font-weight: 600`, not `font-bold` / `font-weight: 700`.

**LikeButton-specific typography:**
- Count digit: `text-sm` (14px) at weight 400 — same visual weight as the icon, reads as a supporting stat, not a headline.
- Wear photo overlay text (top-left "tyler · 2d"): `text-sm` (14px) at weight 600 semibold — legible at small size on a scrimmed surface.
- Wear photo overlay text (bottom-left brand/model): brand in `text-sm` (14px) weight 600 semibold; model in `text-sm` (14px) weight 400 regular.

Source: existing `WearDetailMetadata.tsx` uses `text-sm font-semibold` for collector name and `text-xs text-muted-foreground` for timestamp. This contract preserves those sizes on the overlay.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` `oklch(0.985 0.003 75)` light / `oklch(0.14 0.005 75)` dark | Page background, wear footer action row background |
| Secondary (30%) | `--muted` `oklch(0.95 0.005 75)` light / `oklch(0.24 0.005 75)` dark | Photo placeholder background, skeleton, reserved comment slot background |
| Accent (10%) | `--accent` `oklch(0.76 0.12 75)` light / `oklch(0.78 0.13 75)` dark | Focus ring on LikeButton (`focus-visible:ring-ring`); existing CTA elements only |
| Destructive | `--destructive` `oklch(0.55 0.22 27)` light / `oklch(0.65 0.2 27)` dark | **Filled heart in liked state** (see token rationale below) |

Accent reserved for: focus rings, primary CTA buttons (FollowButton primary variant), chart highlights. The liked-state heart is NOT accent — it uses `--destructive` (red family) which is the universally understood "liked/loved" signal.

**Token rationale — liked-state heart color:**

`--accent` (oklch hue 75) is amber/warm-gold — the existing CTA and ring color. Using it for a heart would be semantically confusing because it is already the "primary action" signal on every interactive button.

`--destructive` (oklch hue 27, orange-red family, light: `0.55 0.22 27`) is the standard red-family semantic token. It is used only for destructive action buttons (Delete) today. For a filled heart it acts as an emotional positive signal — exactly as every social platform uses a red heart. The token is available in both light and dark themes and already adapts correctly. No new token is needed.

**LikeButton state colors, explicit:**

| State | Icon | Count |
|-------|------|-------|
| Idle / not liked | `text-muted-foreground` (outline Heart, no fill) | `text-muted-foreground` |
| Liked | `text-destructive` (filled Heart, `fill="currentColor"`) | `text-destructive` |
| Pending (transition) | opacity-50 on the whole button (`opacity-50`) | same as pre-click state |
| Hover (not liked) | `hover:text-foreground` | `hover:text-foreground` |
| Hover (liked) | `hover:opacity-80` on the whole button (keep red, don't change color) | same |

**Photo overlay scrims — exact values:**
- Top gradient: `background: linear-gradient(to bottom, rgba(0,0,0,0.45) 0%, transparent 40%)` — covers avatar+timestamp row, fades to transparent.
- Bottom gradient: `background: linear-gradient(to top, rgba(0,0,0,0.45) 0%, transparent 40%)` — covers brand/model row, fades to transparent.
- Overlay text color: `text-white` (`color: white`) — always white on both photo and no-photo fallback (see D-08 chain below).
- No-photo fallback: the `bg-muted` placeholder is a medium-lightness surface (oklch ~0.95 light, ~0.24 dark). Text must be legible on muted. Use `text-foreground` for the overlay text on the fallback path (not white, which may not contrast on light muted). Implementation detail: the overlay component detects the no-photo path via a `hasPhoto: boolean` prop and swaps between `text-white` (photo) and `text-foreground` (fallback).

## Component Contracts

### LikeButton (`src/components/shared/LikeButton.tsx` — new file)

Structural clone of `FollowButton.tsx`. Props:

```ts
interface LikeButtonProps {
  viewerId: string | null        // null → anon, click bounces to /login?next=...
  target: { type: 'watch' | 'wear'; id: string }
  initialLiked: boolean
  initialCount: number
}
```

**Layout:**
- Outer element: `<button type="button">` with `className="inline-flex items-center gap-1 min-h-[44px] min-w-[44px] px-2 rounded-md transition-colors focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring disabled:cursor-wait"`
- Heart icon: `<Heart className="size-5" />` — 20px, lucide-react. Liked state: `fill="currentColor"`. Not-liked: no fill prop (or `fill="none"`). Icon `className` toggles between `text-muted-foreground` (idle) and `text-destructive` (liked).
- Count: `<span className="text-sm tabular-nums">` — renders the number as a bare integer. Hidden via conditional render when count is 0 and not liked (i.e. `{(liked || count > 0) && <span>…</span>}`). Color matches icon state.
- Pending: `disabled={pending}` + `aria-busy={pending}` + wrap everything in `className={cn(…, pending && 'opacity-50')}`.

**Interaction states (explicit):**

| State | Visual |
|-------|--------|
| Idle, not liked | Heart outline `text-muted-foreground`, no count shown if 0 |
| Idle, not liked, hover | Heart outline `hover:text-foreground` |
| Liked | Heart filled `text-destructive`, count shown |
| Liked, hover | `hover:opacity-80` on the button — red stays red |
| Pending (optimistic flip in flight) | `opacity-50`, `disabled`, `aria-busy=true` |
| Anon (viewerId null) | Heart outline shown, count shown if >0; click → `/login?next=encodeURIComponent(pathname)` via `router.push` |

**Optimistic logic (mirror of FollowButton):**
1. On click: optimistically set `liked = !liked`, `count = liked ? count + 1 : count - 1`.
2. Start `useTransition`. Call `toggleLikeAction({ type, id })`.
3. On success: reconcile `liked` and `count` to server-confirmed values (`result.data.liked`, `result.data.count`). Do NOT trust the optimistic increment as final (CONTEXT.md locked decision).
4. On failure: roll back to pre-click `liked` and `count`. Log error. Do not surface error toast (idempotent re-like must not show an error — LIKE-04/SC#4).

**Accessibility:**
- `aria-pressed={liked}` — screen reader announces toggle state.
- `aria-busy={pending}` — announced during transition.
- `aria-label`: `liked ? 'Unlike' : 'Like'` (no target name needed; context is the page).

### Wear page redesign and footer action row (D-04/05/06/07/08)

#### Photo container CSS chain (CRITICAL — D-08 blind spot)

The photo container is the existing `w-full aspect-[4/5] overflow-hidden bg-muted md:rounded-lg md:max-w-[600px] md:mx-auto` element in both `WearDetailHero.tsx` and `WearPhotoClient.tsx`.

**Mandatory chain:**

```
outer div:
  position: relative           ← `relative` class (must be on the photo container)
  width: 100%                  ← `w-full`
  aspect-ratio: 4/5            ← `aspect-[4/5]`
  overflow: hidden             ← `overflow-hidden`

img / fallback div (child):
  width: 100%                  ← `w-full`
  height: 100%                 ← `h-full`
  object-fit: cover            ← `object-cover` (img only; fallback is a div)
  position: static             ← default, fills the aspect-ratio container

overlay divs (absolute children of the outer div):
  position: absolute           ← `absolute`
  left: 0; right: 0           ← `inset-x-0`
  z-index: 10                  ← `z-10`
  pointer-events: none         ← `pointer-events-none` (overlays don't intercept clicks)
  content (links inside): pointer-events-auto ← on the clickable username link only
```

**The `relative` class is mandatory on the photo container div. There are exactly 4 callsites that need it — executor must patch all 4 and not stop early:**

| File | Approx line | Condition | Action |
|------|-------------|-----------|--------|
| `WearPhotoClient.tsx` | ~L94 | signed-URL happy path | **ALREADY HAS `relative`** — leave as-is |
| `WearPhotoClient.tsx` | ~L68 | `status==='failed'` + `watchImageUrl` fallback | **MISSING `relative`** — add it |
| `WearPhotoClient.tsx` | ~L81 | `status==='failed'` + no-photo fallback | **MISSING `relative`** — add it |
| `WearDetailHero.tsx` | ~L33 | `watchImageUrl` path | **MISSING `relative`** — add it |
| `WearDetailHero.tsx` | ~L46 | no-photo fallback | **MISSING `relative`** — add it |

That is 4 additions across 2 files (L94 is already patched and must not be double-applied). The class string `"w-full aspect-[4/5] overflow-hidden bg-muted md:rounded-lg md:max-w-[600px] md:mx-auto"` becomes `"relative w-full aspect-[4/5] overflow-hidden bg-muted md:rounded-lg md:max-w-[600px] md:mx-auto"` at each of the 4 missing callsites.

**Top overlay (avatar + username + timestamp):**
```
  position: absolute; inset-x-0; top: 0; z-index: 10
  background: linear-gradient(to bottom, rgba(0,0,0,0.45) 0%, transparent 40%)
  padding: 12px (p-3)
  display: flex; align-items: center; gap: 8px (gap-2)
```
Content: `[Avatar 32px] [username · timeAgo]` all `text-white` on photo path, `text-foreground` on no-photo fallback path.

**Bottom overlay (brand / model):**
```
  position: absolute; inset-x-0; bottom: 0; z-index: 10
  background: linear-gradient(to top, rgba(0,0,0,0.45) 0%, transparent 40%)
  padding: 12px (p-3)
  display: flex; flex-direction: column
```
Content: `[brand: text-sm font-semibold] [model: text-sm font-normal]` all `text-white` on photo path, `text-foreground` on fallback.

**No-photo fallback correctness (D-08):**
The fallback renders `bg-muted` with `items-center justify-center`. The existing centered `{brand} {model}` text must be REMOVED from the fallback's interior — brand/model moves to the bottom overlay. The fallback becomes a plain `bg-muted` block. The overlay divs are rendered on top regardless of photo presence. Text color: `text-foreground` (not white) on the muted fallback since `--muted` is light in light mode and dark text has adequate contrast.

**Assert:** After clearing `.next/` cache (`rm -rf .next`), the photo renders at 4:5 aspect ratio with overlays positioned correctly at top and bottom edges. The muted fallback shows overlays, not centered text.

#### Wear footer action row (D-04)

Sits below the photo container and below the note caption. Replaces nothing from Phase 55 — it is new markup.

```
outer div:
  display: flex; align-items: center
  padding: 16px horizontal (px-4), 12px vertical (py-3)
  border-top: 1px solid var(--border)  → `border-t border-border`
  max-width: 600px (md:max-w-[600px] md:mx-auto)

left slot (reserved for Phase 57 comment input):
  flex: 1 1 0%  → `flex-1`
  min-height: 44px  → `min-h-[44px]`
  background: transparent
  [EMPTY in Phase 56 — no placeholder text, no border, no visible content]
  Implementation: <div className="flex-1 min-h-[44px]" aria-hidden />

right slot (LikeButton):
  flex: 0 0 auto  → `flex-none`
  LikeButton aligned to flex-end
```

The left slot is sized `flex-1 min-h-[44px]` so Phase 57 drops a `<textarea>` or input into it without any re-layout. The overall row height is determined by `min-h-[44px]` on both slots. The `border-t` creates a visual separator between caption and action area.
