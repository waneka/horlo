## Component Inventory

### New components (`src/components/comment/`)

All new; none have shadcn registry equivalents. Build from scratch using existing patterns.

#### `CommentThread` (Server Component — NOT 'use cache')

**What:** Async Server Component that fetches the comment thread and gate state, passes `initialComments` + gate props to `CommentList`. Wrapped in `<Suspense>` by the host page/component.

**Props:**
```
viewerId: string | null
target: { type: 'watch' | 'wear'; id: string }
canComment: boolean           // from canViewerCommentOnTarget
ownerFollowsViewer: boolean   // from isFollowing(ownerId, viewerId); false for wear targets
ownerUserId: string           // for FollowButton in GATE-03
ownerUsername: string         // for GATE-03 copy
```

**CSS chain (no 'use cache' assertion):** This component MUST NOT carry a `'use cache'` directive. The absence of caching is the privacy guarantee. Verified path: `src/data/comments.ts` line 2 PRIVACY LAYER NOTE.

**Skeleton fallback (for Suspense):** A `<CommentThreadSkeleton>` rendering 2–3 placeholder bars at `h-4 bg-muted animate-pulse rounded` with `mb-3` spacing. Matches the loading state visual of other Suspense skeletons in the app.

#### `CommentList` ('use client')

**What:** Client island that receives `initialComments` and manages the optimistic list. Contains `CommentCompose` (always at top when `canComment = true`) or `CommentGateLocked` (when `canComment = false`), then the list of `CommentItem` below.

**Layout:** `flex flex-col gap-4` container. Compose box or locked state above. Items below in newest-first order.

**Empty state:** When `comments.length === 0` and compose is visible: no empty state message — the compose box alone is sufficient. The host section heading ("Comments") already signals the context. When `comments.length === 0` and viewer is gated: the locked state component fills the space.

#### `CommentItem` ('use client' — handles edit/delete transitions)

**What:** Single comment row. Always-visible pencil + trash on the author's own comments; nothing for non-authors.

**Layout:**
```
[avatar 32px] [right block: flex-1]
               ├── [meta line: {username · timeAgo [· edited]}]
               └── [body text]
                   [edit/delete controls row — author only, always visible]
```

**Avatar:** `size-8` (32px) circle. Uses `AvatarDisplay` from `src/components/profile/AvatarDisplay` — same size as feed rows (40px avatar in `ActivityRow`; comment items are visually subordinate so 32px is appropriate). Linked to `/u/{username}/collection`.

**Username:** `text-sm font-semibold text-foreground`; linked to `/u/{username}/collection` (`relative z-10` to escape any parent link).

**Timestamp:** `text-xs text-muted-foreground`; uses existing `timeAgo()` from `src/lib/timeAgo`.

**`[edited]` indicator:** Appended to the META line as a muted suffix — `text-xs text-muted-foreground` — following the `timeAgo` value: `{username} · {timeAgo} · edited`. NOT placed after the body text (D-07, locked).

**Body text:** `text-sm font-normal text-foreground leading-relaxed`. No markdown rendering. Max visible height: unconstrained (flat thread, no collapse).

**Edit/delete controls (author only, always visible):**
- Layout: `flex items-center gap-2 mt-1` row below the body.
- Pencil button: `<button aria-label="Edit comment">` with `<Pencil className="size-4 text-muted-foreground hover:text-foreground" aria-hidden />`. Touch target: `min-h-[44px] min-w-[44px] inline-flex items-center justify-center`.
- Trash button: `<button aria-label="Delete comment">` with `<Trash2 className="size-4 text-muted-foreground hover:text-destructive" aria-hidden />`. Same touch target class.
- Both: `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring rounded-md`.
- `disabled={pending}` on both while any transition is in progress.

**Edit in-place state (when editing):**
- Body swaps to `<textarea>` using the same styling as `CommentCompose` textarea.
- Save button: `<button className="h-8 px-3 bg-primary text-primary-foreground text-sm rounded-md hover:opacity-90 disabled:cursor-wait">Save</button>`.
- Cancel button: `<button className="h-8 px-3 border border-border text-sm rounded-md hover:bg-muted disabled:cursor-wait">Cancel</button>`.
- Row layout: `flex items-center gap-2 mt-1`.

**Delete confirm row (inline, NO AlertDialog — D-06, locked):**
- Replaces the pencil/trash row when trash is clicked.
- Copy: `Delete? · Cancel` — two adjacent inline text-buttons.
- Delete? button: `<button className="text-sm text-destructive hover:underline">Delete?</button>`.
- Cancel button: `<button className="text-sm text-muted-foreground hover:underline">Cancel</button>`.
- Separator: `·` in `text-muted-foreground mx-1`.
- Row layout: `flex items-center gap-1 mt-1`.
- The confirm row MUST fit within the bottom-sheet overlay without overflow. Maximum width is constrained by the sheet's `max-h-[60vh]` container.

#### `CommentCompose` ('use client')

**What:** Compose box: textarea + char counter + submit button.

**Layout:** `flex flex-col gap-2` container.

**Textarea:**
- `<textarea>` (not shadcn Textarea — the built-in element for maxLength enforcement).
- `maxLength={500}` — enforced at input (CMNT-04 first layer).
- `rows={3}` default; resizes vertically (`resize-y`) but not horizontally (`resize-none` on horizontal).
- Class: `w-full rounded-md border border-input bg-background px-3 py-2 text-sm text-foreground placeholder:text-muted-foreground focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring disabled:cursor-not-allowed disabled:opacity-50 resize-none`.
- Placeholder: `"Add a comment…"`.
- `disabled={pending}`.

**Char counter:**
- Visible only when `body.length >= 450` (threshold: 450/500 — 90% of max).
- Rendered as: `<span className="text-xs tabular-nums text-muted-foreground self-end">{body.length}/500</span>`.
- When `body.length >= 480`: class changes to `text-destructive` (warning approaching limit).
- Threshold 450 chosen: large enough that casual commenters never see it; close enough to the limit to catch approaching overflow before the submit attempt.

**Submit button:**
- Text: `"Post"`.
- Class: `self-end h-8 px-4 bg-primary text-primary-foreground text-sm font-semibold rounded-md hover:opacity-90 focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring disabled:cursor-wait disabled:opacity-50`.
- `disabled={pending || body.trim().length === 0 || body.length > 500}`.

**Row layout:** `flex items-center justify-between` for [char-counter (left)] [Post button (right)] below the textarea. When counter is hidden, Post button still right-aligns.

#### `CommentGateLocked` (can be Client or Server Component — receives all data as props)

**What:** Two-state locked container replacing the compose box when `canComment === false` on a gated wishlist watch.

**Container:** `rounded-md bg-muted px-4 py-4 flex flex-col gap-2 border border-border`.

**State 1 — pre-follow (`viewerIsFollowing === false`):**
- Heading: `<p className="text-sm font-semibold text-foreground">Follow {ownerUsername} to comment</p>`.
- Sub-copy: none.
- Follow button: inline `<FollowButton variant="inline" ... />` — reuse the existing component. The button sits below the heading in the same container, `self-start`.

**State 2 — followed but not yet mutual (`viewerIsFollowing === true`, `ownerFollowsViewer === false`):**
- Heading: `<p className="text-sm font-semibold text-foreground">{ownerUsername} needs to follow you back before you can comment</p>`.
- Sub-copy: none.
- No Follow button (the viewer's half is done; the ball is in the owner's court).

**State 3 — mutual (compose appears):** `CommentGateLocked` is not rendered; `CommentCompose` takes its place.

**Over-photo constraint:** Both states must be readable and functional inside the wears-lane bottom sheet (over-photo). The `rounded-md bg-muted` background provides sufficient contrast against the photo backdrop. No absolute positioning inside the locked container.

The `WearCommentHost` bottom-sheet uses shadcn `Sheet` with `SheetContent side="bottom"`. The full CSS chain:


## Interaction Contracts

### Optimistic comment add

1. User types in compose box. Submit enabled when `body.trim().length > 0`.
2. Char counter appears at `body.length >= 450`. Counter text turns `text-destructive` at `>= 480`.
3. On submit: optimistic comment inserted at the TOP of the list (`setComments([optimistic, ...comments])`). Submit button `disabled`. Optimistic item rendered with the same `CommentItem` layout; no visual "pending" spinner — just the `opacity-50` on the compose area (mirrors `LikeButton` `pending && 'opacity-50'`).
4. On success: temp ID replaced with server-confirmed `{ id, createdAt }`. Compose textarea cleared.
5. On failure: optimistic item removed (rollback). Compose textarea content preserved (user can retry). If `result.code === 'gate'`: transition to locked state silently (the gate enforcer at the action layer is the race backstop).

### Edit in-place

1. Author taps pencil icon. Body text swaps to prefilled textarea. Save/Cancel appear.
2. Save disabled when `body.trim().length === 0` or unchanged from original.
3. Char counter same threshold as compose.
4. On save: optimistic update of body + set `editedAt: new Date().toISOString()` (temp). Reconcile with server-confirmed `editedAt`.
5. On cancel or failure: revert to original body; rollback `editedAt`.

### Delete inline confirm

1. Author taps trash icon. "Delete? · Cancel" row appears in place of pencil/trash.
2. Tapping "Delete?": optimistic removal of the comment from the list. Confirmed with server.
3. On failure: rollback (comment reappears at its original position).
4. Tapping "Cancel": reverts to pencil/trash row immediately (no server call).

### GATE-03 transitions

1. Initial render with `canComment = false`, `viewerIsFollowing = false`: locked state State 1 (Follow CTA).
2. After viewer taps Follow (`FollowButton` fires `followUser`): `router.refresh()` (existing `FollowButton` behavior). Page re-renders with updated `canComment` and `ownerFollowsViewer` from server.
3. If `canComment` is still false but `viewerIsFollowing` is now true: locked state State 2 (needs to follow back).
4. If `canComment` is now true: compose box renders, locked state disappears.

All copy locked (from CONTEXT.md D-01, D-02, D-11).

| Element | Copy |
|---------|------|

Error state (delete fails): Silent rollback; comment reappears.
