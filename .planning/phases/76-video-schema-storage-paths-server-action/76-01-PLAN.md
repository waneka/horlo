<objective>
Extend the Drizzle schema and Supabase migration to add the `media_type` enum and three new wear_events columns (`media_type`, `media_path`, `poster_path`); add the DB CHECK constraint that gates video rows; extend the `wear-photos` bucket MIME allowlist to accept `video/mp4`. Push the schema locally via `drizzle-kit push` (interactive — autonomous:false), then run an integration test that proves VID-11 (existing photo rows unaffected) and VID-12 (CHECK rejects video row with NULL paths).

Purpose: this is the schema/storage foundation Plan 03's Server Action depends on. Without these columns in the local DB, the DAL helper `logWearEventWithVideo` will not type-check (Drizzle generates the row type from the schema definition). Without the bucket MIME UPDATE, Phase 77's client-direct upload returns 422 from Supabase Storage. The migration is strictly additive (no UPDATE of `photo_url`, no RENAME, no DROP) so the existing photo path through `logWearWithPhoto` is provably unchanged (VID-15 foundation, formally tested in Plan 04 verification).
