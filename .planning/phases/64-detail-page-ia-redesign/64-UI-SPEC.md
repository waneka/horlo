## Information Architecture Contract

This section is the primary deliverable for this phase. It is the "design contract" for the downstream planner and executor.

### Canonical Vertical Order (D-02, locked)

All three branches must follow this top-to-bottom DOM order. No CSS-order tricks permitted (D-07).

```
[PAGE TOP]
1. Hero block
   — Desktop: 2-column grid (carousel left · verdict+title+like+spec-strip right)
   — Mobile:  single-column stack
2. Comments (Suspense-wrapped CommentThread RSC sibling)
3. Full spec cards (WatchDetailTrailing RSC)
4. Discovery rails (SameFamilyRail, LineageRail)
5. Footer / owner actions (fresh-account CTAs if applicable)
[PAGE BOTTOM]
```

### Hero Composition (D-01, D-03, D-04, D-09)

**Desktop layout:** `grid gap-8 lg:grid-cols-[3fr_2fr]`
- Left column (3fr): `WatchPhotoSection` carousel + filmstrip (full width of column)
- Right column (2fr): brand heading → model → reference → status badge → `SpecsSublabel` strip → `CollectionFitCard` (verdict) → `LikeButton` + comment-count jump link → last-worn line (owner only) → flag-deal checkbox (owner only) → owner action buttons (owner only)

**Mobile layout:** single column. `WatchPhotoSection` full-width → title block → verdict → like → actions (standard block flow).

**Carousel:** Reuse `WatchPhotoSection` as-is (D-04). Do not re-engineer the carousel, filmstrip, or wear-pic social overlay.

**Grid ratio rationale:** `lg:grid-cols-[3fr_2fr]` (60%/40%) gives the carousel dominant real estate without making the verdict column too narrow for `CollectionFitCard`. RESEARCH.md Open Question 4 resolves here. Source: RESEARCH.md recommendation + D-01 intent.

**Breakpoint:** `lg` (1024px). Same as the existing `WatchDetail.tsx:173` breakpoint. Do not change.

### Verdict Placement (D-09)

`CollectionFitCard` renders in the **hero right column**, directly below the watch title/reference/spec-strip block. This is an elevation from its current position (`WatchDetail.tsx:547`, below all spec cards).

**Empty verdict state (D-10 — Claude's Discretion, resolved here):**
- `verdict === null` + `collection.length === 0` + `catalogTaste.confidence >= 0.5` → render `ReferenceIdentityCard` in the hero right column, replacing the verdict slot
- `verdict === null` + `collection.length === 0` + confidence below threshold → render the caption `"Add a few watches to see how this one fits your collection."` as `text-sm text-muted-foreground` in the hero right column
- The fresh-account 3-CTA block (Add to Wishlist / Add to Collection / Skip) remains at the **bottom** of the page, after rails — it is an action footer, not hero content

### Gap-Fill Placement (D-11 — Claude's Discretion, resolved here)

The wishlist/grail gap-fill callout (`computeGapFill`) stays in `WatchDetailTrailing` (below comments, before Notes). It is reference material like the spec cards, not a verdict-level insight. Do not elevate it into the hero.

### Owner Actions Placement (D-15 — Claude's Discretion, resolved here)

Mark-as-Worn, Edit, Delete, Flag-deal checkbox remain in the **hero right column** of `WatchDetailHero` (below the like/comment row). They require `'use client'` (handlers + useTransition + Dialog) and are contextually tied to the hero. A separate bottom footer would require additional island plumbing. Keep hidden for non-owners via `viewerCanEdit` prop (unchanged from today).

### Jump-to-Comments Anchor (D-06)

The comment-count display at `WatchDetail.tsx:237-243` (currently a non-interactive `<span>`) becomes an `<a href="#comments">` anchor.

**Visual treatment:**
```
<a href="#comments" className="inline-flex items-center gap-1 text-sm tabular-nums text-muted-foreground px-2 min-h-[44px] hover:text-foreground transition-colors">
  <MessageCircle className="size-5" aria-hidden />
  {commentCount}
</a>
```

- Hidden at zero (`commentCount === 0` renders nothing — same as today)
- `min-h-[44px]` touch target (unchanged)
- `hover:text-foreground` on hover — no accent color
- `transition-colors` for smooth state change
- Keyboard: native `<a>` is keyboard-focusable by default; no additional JS needed
- Focus ring: inherits `outline-ring/50` from `globals.css @layer base` global

**Anchor target:** `CommentThread`'s wrapping `<section>` receives `id="comments"`. Change `CommentThread.tsx:66` from `<section className="mt-6">` to `<section id="comments" className="mt-6">`.

**Smooth scroll:** `scroll-behavior: smooth` is already set on `html` in `globals.css:145`. Verified. The `<a href="#comments">` anchor produces smooth scroll without any JS.

**Accessibility:** The `<a>` tag is natively focusable and reachable by keyboard tab in visual order. `aria-hidden` on the `MessageCircle` icon (decorative). `tabIndex` not needed. Screen-reader reads the link as "N [icon label suppressed]" — add `aria-label="Jump to comments"` or wrap count in a visually-hidden label if the number alone is insufficient for SR context.

### CommentThread Section (D-05, D-07, D-08)

CommentThread renders as a **Suspense-wrapped RSC sibling** of `WatchDetailHero`, positioned immediately after the hero and before `WatchDetailTrailing`. No `'use client'`, no `'use cache'`. The section element has `id="comments"`.

**Desktop width:** Full content width (same as the page `max-w-4xl` container). No narrower reading column. The page container already constrains width appropriately. Source: Claude's Discretion, resolved here for simplicity and parity with existing section widths.

**Skeleton:** `<CommentThreadSkeleton />` is already used as the Suspense fallback (Branch 1 at `page.tsx:360`). Preserve unchanged.

### Branch 3 — Catalog Shell-Only Variant (D-12, D-13, D-14)

The generic catalog view (Branch 3, `page.tsx:656-725`) gets the same visual IA shell:

```
[catalog hero: flex items-start gap-4 → 96px thumbnail + title/ref/SpecsSublabel]
[CollectionFitCard — verdict, if non-null]
[ReferenceIdentityCard or caption — empty-collection state]
[OtherOwnersRoster — MOVED UP (D-13)]
[CatalogPageActions — MOVED UP (D-13)]
[SameFamilyRail]
[LineageRail]
```

- **No comments** (no per-user `watches.id` target exists — out of scope)
- **No multi-photo carousel** (catalog entries carry one image — out of scope)
- **Container class:** Align to `space-y-8` (currently `space-y-6` at `page.tsx:657`). Change to match Branches 1 & 2 for visual parity (D-14).
- **OtherOwnersRoster + CatalogPageActions** move to immediately after the verdict / empty-state block — above rails. This resolves the TODO comments at `page.tsx:704-706` and `page.tsx:715-716`.

### WatchPageSkeleton Update

The existing skeleton (`page.tsx:103-114`) mirrors the old layout (square carousel, then two lines, then a tall block). Update to mirror the new IA:

```
container mx-auto px-4 py-8 max-w-4xl space-y-8 (aria-hidden)
  — Hero grid (lg:grid-cols-[3fr_2fr] gap-8):
    Left: Skeleton aspect-[3/4] w-full rounded-lg  ← carousel shape
    Right: space-y-4
      Skeleton h-7 w-3/4    ← brand heading
      Skeleton h-5 w-1/2    ← model
      Skeleton h-4 w-1/3    ← spec strip
      Skeleton h-40 w-full rounded-lg  ← verdict card
      Skeleton h-9 w-24     ← like button row
  — Skeleton h-32 w-full rounded-lg  ← comment section
  — Skeleton h-40 w-full rounded-lg  ← spec cards block
```

The skeleton is pure JSX with `<Skeleton>` primitives — no dynamic API, no hooks. Must remain compatible with the PPR opt-out structure (it is the Suspense fallback, not a prerendered static shell).

## Accessibility Contract

- **Tab order must match visual order** (D-07). No CSS-order tricks. DOM order = visual order = keyboard order.
- **Hero left-to-right / top-to-bottom tab sequence (desktop):** `WatchPhotoSection` interactive controls (arrows, filmstrip) → (right column) brand heading (non-interactive) → `LikeButton` → comment-count jump link (`<a>`) → owner action buttons (if viewerCanEdit) → then on to the comments section below.
- **Jump link** is a native `<a href="#comments">` — keyboard-focusable, skip-link friendly.
- **Comment count icon** has `aria-hidden` (decorative).
- **All interactive elements** maintain `min-h-[44px]` touch targets (existing pattern — preserve).
- **`WatchPageSkeleton`** has `aria-hidden` (already has it; confirm when updating the skeleton).
- **Reduced motion:** `globals.css:147-149` sets `scroll-behavior: auto` under `prefers-reduced-motion: reduce`. The `<a href="#comments">` anchor respects this automatically.
