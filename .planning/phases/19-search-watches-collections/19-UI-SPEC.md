## Spacing Scale

Declared values (multiples of 4 only). Source: Tailwind 4 default scale, confirmed by existing class usage in `PeopleSearchRow.tsx` and `SearchPageClient.tsx`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline label padding |
| sm | 8px | Row internal gaps (`gap-2`), pill internal padding |
| md | 16px | Row horizontal padding (`px-4`), section gap |
| lg | 24px | Section vertical spacing (`space-y-6`), page top padding |
| xl | 32px | Page bottom padding (`py-8`) |
| 2xl | 48px | — (reserved for major section breaks; not used in search shell) |
| 3xl | 64px | — (page-level; not used in search shell) |

Exceptions:
- Touch target minimum 44px enforced by `min-h-16 md:min-h-20` on result rows (existing pattern from `PeopleSearchRow` — carry forward to `WatchSearchRow` and `CollectionSearchRow`).
- Watch thumbnail circles: `size-10` (40px) on mobile, `size-12` (48px) on md+ (existing pattern from `PeopleSearchRow` shared-watch cluster — carry forward).

## Color

Source: `src/app/globals.css` CSS custom properties; confirmed present in both `:root` (light) and `.dark` themes.

| Role | CSS Token | Light Value | Dark Value | Usage |
|------|-----------|-------------|------------|-------|
| Dominant (60%) | `--background` | `oklch(0.985 0.003 75)` | `oklch(0.14 0.005 75)` | Page background, tab content area |
| Secondary (30%) | `--card` | `oklch(1 0 0)` | `oklch(0.19 0.005 75)` | Result row background (`bg-card`), skeleton rows |
| Muted surface | `--muted` | `oklch(0.95 0.005 75)` | `oklch(0.24 0.005 75)` | Search input background (`bg-muted/50`), thumbnail placeholder bg |
| Accent (10%) | `--accent` | `oklch(0.76 0.12 75)` | `oklch(0.78 0.13 75)` | Reserved for specific elements only (see below) |
| Destructive | `--destructive` | `oklch(0.55 0.22 27)` | `oklch(0.65 0.2 27)` | Error alert border + text |
| Muted foreground | `--muted-foreground` | `oklch(0.48 0.01 75)` | `oklch(0.68 0.01 75)` | Sub-labels (overlap %, match count), placeholder text |

Accent (`--accent` / `--ring`) is reserved for:
- Focus ring on the search input
- Focus ring on result rows (`:focus-visible` ring on the absolute-inset `<Link>`)
- The `'Evaluate'` button hover/focus state
- The `'Owned'` pill background (use `bg-primary text-primary-foreground` for owned, consistent with how "owned" is surfaced elsewhere in the codebase; use `bg-muted text-muted-foreground` for "Wishlist" — lower visual weight than owned)

Accent is NOT used for: section headings, row hover states (use `hover:bg-muted/40` per existing `PeopleSearchRow` pattern), tab triggers, or skeleton shimmer.

Hover state on rows: `hover:bg-muted/40` — inherited from `PeopleSearchRow` pattern. Apply identically to `WatchSearchRow` and `CollectionSearchRow`.

## Interaction Contracts

### Search Input

- Input: shadcn `<Input>` component. Existing `className` from `SearchPageClient` carries forward. Placeholder updates per active tab: `"Search collectors…"` (people), `"Search watches…"` (watches tab active), `"Search collections…"` (collections tab active), `"Search everything…"` (all tab). The input is shared across all tabs.
- Debounce: 250ms (carry-forward from Phase 16).
- Min chars: 2 client-side gate (carry-forward from Phase 16).
- URL sync: `router.replace({scroll: false})` (carry-forward from Phase 16).

### Result Rows — Click Targets

- Whole-row absolute-inset `<Link>`: destination is `/evaluate?catalogId={uuid}` for watch rows; `/u/{username}/collection` for collection rows.
- Inline CTA button raised with `relative z-10` to prevent bubble (pattern from `PeopleSearchRow` + `SuggestedCollectorRow`).
- Focus management: `focus-visible:ring-2 focus-visible:ring-ring` on the absolute-inset Link (matches existing rows).

### Loading States

- Skeleton rows appear immediately when a fetch dispatches (no delay).
- Per-section skeleton in All tab: each section shows its own skeleton independently (D-15).
- Skeleton row count: 4 rows for Watches tab; 4 rows for Collections tab; 3 rows per section in All tab.

### AbortController Gating

- One AbortController per `[debouncedQ, tab]` dependency pair.
- Switching tabs while a fetch is in flight aborts the prior controller.
- Stale-result guard: `signal.aborted` checked after each `await` before dispatching to state.

### Scroll

- No infinite scroll, no pagination (project constraint: <500 watches, cap at 20 results per tab).
- Results list is non-scrollable within the tab; the page scrolls normally.
