## Spacing Scale

Declared values (must be multiples of 4). Mirrors the existing Horlo rhythm in `src/app/page.tsx` (home composition) and `src/components/home/SuggestedCollectorRow.tsx` (rail row).

| Token | Tailwind | Value | Usage |
|-------|----------|-------|-------|
| xs | `gap-1` / `p-1` | 4px | Icon-to-label gap inside BottomNav column; tight inline label clusters |
| sm | `gap-2` / `space-y-2` | 8px | Inter-row gap inside Popular Collectors list; row-internal element separation |
| md | `gap-3 md:gap-4` / `px-4 py-3` | 12 / 16px | Horizontal rail card gap; row internal padding |
| lg | `space-y-4` / `gap-6` | 16 / 24px | Inter-rail-section vertical rhythm inside a rail (header → list/strip) |
| xl | `space-y-8 md:space-y-12` / `py-8` | 32 / 48px | Inter-section vertical rhythm on `/explore` page (hero → rail → rail → rail) |
| 2xl | `py-12` / `py-16` | 48 / 64px | Hero internal vertical padding (sparse-network welcome breathing room) |

**Container:** `container mx-auto px-4 md:px-8 py-8 space-y-8 md:space-y-12 max-w-6xl` — exactly mirrors home page `<main>` for visual continuity (both surfaces are 5-rail-style stacks).

**Rail strip:** `flex gap-3 md:gap-4 overflow-x-auto snap-x snap-mandatory scroll-smooth pb-2` — exact mirror of `src/components/home/WywtRail.tsx`. Cards use `class="snap-start"`.

**BottomNav:** unchanged — `h-[calc(80px+env(safe-area-inset-bottom))]`, `pb-[env(safe-area-inset-bottom)]`, `px-2`, column `gap-1 pb-4` (Phase 14 contract preserved per D-04).

**Exceptions:** none — Phase 18 introduces no new spacing tokens. The `gap-3` (12px) → `gap-4` (16px) responsive jump on rail cards is the established Horlo idiom and is not classified as an "exception" because both values are 4-multiples.

## Typography

Three sizes + one display variant, two weights. Mirrors prior phase rhythm (`text-xl font-semibold` section headings; `text-sm` body; serif display only on hero h1).

| Role | Size (px / Tailwind) | Weight | Line Height | Font | Usage |
|------|----------------------|--------|-------------|------|-------|
| Body | 14 / `text-sm` | 400 (regular) | 1.5 | IBM Plex Sans | Sublabels (`{N} collectors`, `↑ +12 this week`), See-all link, Gaining Traction empty-state copy, "Showing top 50" footer, hero supporting paragraph |
| Label | 14 / `text-sm` | 600 (semibold) | 1.4 | IBM Plex Sans | Collector row primary line (displayName / username), card brand+model line |
| Heading | 20 / `text-xl` | 600 (semibold) | 1.25 | IBM Plex Sans | Rail section `<h2>` ("Popular collectors", "Trending", "Gaining traction") |
| Display | 30 / `text-3xl` md:36 / `md:text-4xl` | 400 (regular) | 1.1 | Instrument Serif (`font-serif`) | Hero `<h1>` only — sparse-network welcome headline |

**Mono / monospace:** not used in Phase 18.

**Bottom-nav label:** `text-[12px] leading-[16px] font-semibold` — Phase 14 NAV contract preserved verbatim (D-01 leaves BottomNav typography unchanged; only slot order changes).

**Rationale (single page-level display use):** Per Horlo convention, Instrument Serif is reserved for marketing/welcome moments. Rails and rows stay sans-only to keep visual hierarchy crisp. The hero's serif `<h1>` is the ONLY page element using `font-serif`.

| Foreground | `--foreground` / `text-foreground` | `oklch(0.18 0.01 75)` → `oklch(0.96 0.005 75)` | Primary copy: row displayName, rail headings, hero h1, brand+model on cards |
| Border | `--border` / `border-border` | `oklch(0.9 0.005 75)` → `oklch(1 0 0 / 10%)` | BottomNav top border (unchanged) |

**Accent reserved for (explicit list — Phase 18):**
1. Hero icon glyph (`<Compass />`) inside the hero icon-circle (`text-accent` on `bg-accent/10` 48×48 rounded-full backdrop) — mirrors v3.0 stub treatment for continuity, swapping `Sparkles` → `Compass` to signal "discovery, not coming-soon."
2. Hero primary CTA button — `<Button>` from `@/components/ui/button` with default variant; the shadcn `base-nova` `--primary` token is the canonical button bg, NOT the accent token — the button is dominant-foreground-on-primary, not accent. The accent appears only as the hero ICON treatment, NOT as the CTA button color.
3. BottomNav active-state icon + label color (`text-accent` with `strokeWidth=2.5`) — Phase 14 contract preserved unchanged.
4. `<Flame />` icon on the Trending rail header (`text-accent`).
5. `<TrendingUp />` icon on the Gaining Traction rail header (`text-accent`).
6. Focus ring (`focus-visible:ring-ring` resolves to `--ring` which equals `--accent` in this preset) — automatic via shadcn primitives; no explicit ring class declarations needed in Phase 18 components.

**Accent NOT used for:** rail headings (use `text-foreground`), See-all links (use `text-muted-foreground` per established pattern), card hover state (use `bg-muted/40`), follower / collector counts (use `text-muted-foreground`).

**Dark-mode parity:** all six accent uses above resolve through CSS variables; Phase 18 components must not hardcode hex/oklch — only token names. The `oklch(0.78 0.13 75)` dark accent slightly desaturates to remain readable on the deep neutral dark surface (Horlo dark theme convention).

## Interaction & Motion

### Hover / focus

- Collector rows (`PopularCollectorRow`) — `hover:bg-muted/40 transition-colors` (mirrors `SuggestedCollectorRow`).
- Watch cards (`DiscoveryWatchCard`) — `hover:bg-muted/40 transition-colors` ONLY if the card becomes clickable. v4.0 Phase 18 ships non-clickable per Discretion default → no hover state. Phase 20 will add the hover when it adds the `/evaluate?catalogId=` link.
- See-all link — `hover:text-foreground transition-colors` (mirrors `/notifications` page "Mark all read" affordance).
- Hero CTA — shadcn `Button` default variant carries its own hover state; no override.
- Focus rings — `focus-visible:ring-2 focus-visible:ring-ring` automatic on all shadcn primitives + `<Link>` overlays (already in `SuggestedCollectorRow`).

### Scroll-snap (rails)

- Strip: `overflow-x-auto snap-x snap-mandatory scroll-smooth pb-2`.
- Cards: `class="snap-start"` on each child.
- Mobile: card snaps to left edge of strip on tap-and-flick.
- Desktop: card scroll smooth; trackpad horizontal scroll honored; no scrollbar styling override.

### No motion / animation in Phase 18

- No reveal animations on hero (the hero is a server-render gate; once rendered, it's static until follow/wear count crosses threshold on next page render).
- No skeleton states for cached rails (the cached `'use cache'` Server Components return instantly on hit; first-load fallback is the layout-level Suspense fallback already in `src/app/layout.tsx`).
- No infinite scroll on See-all routes (D-10 cap-only; matches PROJECT MVP constraint).

### Keyboard / accessibility

- All `<Link>` and `<Button>` are tab-reachable in DOM order.
- Rail headings are `<h2>` so a screen reader user can navigate by heading level.
- Lucide icons all carry `aria-hidden` (decorative); accent color signals via icon are paired with the heading text label.
- Rail strips: native horizontal scroll honors arrow-key scroll on focus; no custom keyboard handler needed.
- BottomNav links carry `aria-current="page"` when active (Phase 14 contract preserved).
- Hero CTA: the `<Link>` wraps the `<Button>`; the `<Link>` carries the `aria-label` if needed for clarity, but the button label `Browse popular collectors` is descriptive enough — no extra aria-label.

| `<768px` (mobile) | BottomNav visible. Container `px-4 py-8 space-y-8`. Rail card width `w-44` (176px). Hero h1 `text-3xl`. Page max-width = viewport. |
