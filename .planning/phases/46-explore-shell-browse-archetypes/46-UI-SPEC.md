## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, chip internal gap, badge padding |
| sm | 8px | Chip rail inter-chip gap, compact list row internal spacing |
| md | 16px | Module internal padding, card padding, page horizontal padding |
| lg | 24px | Module-to-module gap on mobile (within the shell `space-y-6`) |
| xl | 32px | Module-to-module gap on desktop grid |
| 2xl | 48px | Major section breaks between `/explore` shell header and first module |
| 3xl | 64px | Page-level bottom safe area |

Exceptions:
- Archetype chip touch target: minimum 44px height (accessibility tap target). Use `py-2.5 px-4` which produces ~44px on 16px body text.
- Browse tile minimum tap target: 48px height. Use `min-h-12` on each tile.
- A–Z jump nav sticky offset: 0px top (sticks to top of scroll container on `/explore/brands`). Not a spacing exception — use `sticky top-0 z-10 bg-background`.

Source: standard 8-point scale; exceptions derived from WCAG 2.5.5 touch target guidance and CONTEXT.md D-07.

## Typography

Two weights only: 400 (regular) and 600 (semibold). Matching the established
pattern from Phase 45 and the existing explore/search surfaces.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Brand list item names, era list item names, count values, chip badge text, filter chip labels |
| Label | 14px (text-sm) | 600 (semibold) | 1.4 | Module subgroup headings, filter section labels in FilterDrawer, archetype editorial description |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 | Module headings (e.g. "Browse the Catalog", "Collector Archetypes"); index page h1 (e.g. "Brands"); archetype editorial header on `/search` |
| Subtext | 12px (text-xs) | 400 (regular) | 1.4 | Count badges on archetype chips, brand/era/genre count on index list items, "N watches" result count in archetype header |

Notes:
- Module headings use `text-xl font-semibold leading-tight text-foreground` — matches the existing `TrendingWatches` and `SearchPageClient` section heading pattern.
- Archetype chip display names use `text-sm font-semibold` inside the chip button.
- Count text (e.g. "42 watches") on index page items uses `text-xs text-muted-foreground`.
- A–Z letter group headings on `/explore/brands` use `text-sm font-semibold text-muted-foreground uppercase tracking-wide` — distinguishes structural grouping from content.
- Archetype editorial description in the `/search` header uses `text-sm text-muted-foreground` at weight 400. Archetype name uses `text-xl font-semibold`.
- Result count under the archetype header uses `text-xs text-muted-foreground`.

Source: pattern-matched from `src/components/explore/TrendingWatches.tsx`, `src/components/search/SearchPageClient.tsx`, `src/components/search/MovementChips.tsx`, `src/components/ui/badge.tsx`.

## Interaction Contracts

### Archetype chip rail (10 chips)

- Chips are navigation buttons (`<button type="button">`), not toggles. Tapping navigates immediately to `/search?tab=watches&archetype={value}`. No selected state on the explore page.
- No multi-select on this surface. Each chip is an independent entry point.
- On mobile, chips wrap to 2–3 rows (`flex-wrap`). Do not force horizontal scroll — the chip count (10) is known and manageable at wrap.
- Focus order: left-to-right, top row first (natural DOM order).

### Browse entry tiles

- 3 tiles in a 3-column grid (`grid-cols-3`). Each is a `<Link>` navigating to its index page.
- Hover state: `hover:bg-muted` (same as brand row pattern).
- No loading state — tiles are static links.

### A–Z jump nav (brands index)

- Tapping a letter scrolls to the corresponding letter section via native anchor (`href="#letter-A"`).
- Letters with no brands are rendered but non-interactive (`pointer-events-none opacity-30`).
- The nav is sticky with `top-0 z-10`. On mobile, the nav bar itself scrolls horizontally if all 26 letters don't fit — use `overflow-x-auto scrollbar-none`.
- EXPL-04 CSS chain: the letter group `<section id="letter-A">` must have `scroll-margin-top` equal to the sticky nav height to prevent the heading from hiding under the nav after jump. Use `scroll-mt-12` (48px) on each letter section as a safe offset.

### Removable inline facet chips on `/search`

- Single chip rendered above watch results when a brand/era/genre/archetype facet is active.
- Tapping the dismiss `X` calls the appropriate setter (e.g. `setArchetype(null)`), clears the URL param, and re-runs the search without that facet.
- No confirmation — facet removal is reversible via the browser back button or re-navigation from Explore.
- The chip is rendered client-side by `SearchPageClient` based on state from `useSearchState`. It appears immediately on navigation (URL param → state hydration on mount).

### Archetype editorial header on `/search`

- Rendered synchronously — header copy is hardcoded in `ARCHETYPE_CONFIG` and requires no async fetch.
- Result count updates as results stream in. If count is not yet available (loading state), show skeleton or omit count until loaded.
- The header is part of the WatchesPanel render, not a separate server fetch.

### Empty groupings

- EXPL-02: any module with no content returns `null` (no empty container). In Phase 46 this applies to:
  - `CollectorArchetypes`: returns null if `getBrowseArchetypeCounts()` returns empty array (theoretical — Phase 44 verified ≥1 result per archetype)
  - `BrowseModule`: returns null if all three count queries return empty (theoretical — catalog has ≥1 brand, era, and genre)
- On index pages: brand/era/genre rows with count 0 are filtered out in the DAL (the GROUP BY only returns rows with COUNT ≥ 1).

## CSS Chain Assertions

Per memory note `feedback_ui_spec_css_chain_blind_spot`: assert the following
chains explicitly — the checker validates declared tokens, not whether the CSS
chain produces the visual contract.

1. **Archetype chip 44px touch target:** `<button className="... py-2.5 px-4 text-sm ...">` — `py-2.5` = 10px top + 10px bottom + 20px line height for `text-sm` = 40px minimum. Add `min-h-[44px]` explicitly: `className="... min-h-[44px] py-2.5 px-4 ..."`. Assert `min-h-[44px]` is present.

2. **Browse tile 48px touch target:** `<div className="... min-h-12 p-4 ...">` — `min-h-12` = 48px. Assert `min-h-12` is present on the tile container, not only on the wrapping `<Link>`.

3. **A–Z scroll-margin offset:** Every `<section id="letter-{X}">` must include `className="scroll-mt-12"` (or equivalent `scroll-mt-[{navHeight}]`) to prevent jump-nav targets from hiding under the sticky nav bar. Assert `scroll-mt-*` is present on each letter section.

4. **Inline removable chip `bg-accent/10`:** Tailwind 4 opacity modifier syntax — verify `bg-accent/10` is recognized (Tailwind 4 uses `oklch` values natively; `/10` opacity works on CSS custom properties via the color-mix approach). If `bg-accent/10` renders as transparent, fall back to `bg-accent` with `opacity-10` on the background only (not the text). Assert the chip is visually distinguishable from the unselected chip (`bg-secondary`) on both light and dark themes.

5. **`flex-wrap` chip rail overflow:** The 10-chip archetype rail uses `flex flex-wrap gap-2`. On the narrowest supported viewport (360px wide), verify that chips wrap to multiple rows rather than overflowing the container. Do NOT add `overflow-x-auto` to the chip rail — wrapping is the intended behavior on this surface.
