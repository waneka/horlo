<objective>
Fix G5 — the major Phase 46 UAT gap. Selecting a new archetype chip from
`/explore` does not apply the new facet on `/search` when navigation is
client-side (soft nav); only a full page refresh works. This breaks the
phase's headline deep-link flow (EXPL-05).

Root cause (see `.planning/debug/g5-soft-nav-facet-not-applied.md`): two
compounding faults in `useSearchState`.
1. Facet state (brand/era/genre/archetype) is seeded from URL params ONLY at
   first mount via `useState` initializers. App Router soft nav does not
   remount `/search`, so a soft nav to `/search?archetype=B` never re-seeds
   state — the stale archetype stays applied (Fault 1).
2. The URL-sync effect rebuilds the URL from stale in-memory state with a dep
   array that excludes `searchParams`, so after a soft nav it fires and
   `router.replace`s the stale state — stripping the just-arrived param
   (Fault 2).

Purpose: restore the EXPL-05 deep-link flow so Browse/Archetypes entry points
work on every navigation, not just hard refreshes.
Output: a reconciled `useSearchState` with a URL→state sync, plus a regression
test in the existing hook test file.
</objective>
