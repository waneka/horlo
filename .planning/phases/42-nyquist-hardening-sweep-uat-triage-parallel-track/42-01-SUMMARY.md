## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] CSS import in setup file required explicit relative path**
- **Found during:** Task 3 verification (first smoke test run)
- **Issue:** `tests/browser/setup.browser.ts` imported `@/app/globals.css` using the `@/*` alias. The browser project's `resolve.alias` config was not active during setupFile processing — Vite reported `Failed to resolve import "@/app/globals.css"`.
- **Fix:** Changed import to `'../../src/app/globals.css'` (relative path from `tests/browser/`). This is stable and resolves correctly.
- **Files modified:** `tests/browser/setup.browser.ts`
- **Commit:** fa9c720

**2. [Rule 3 - Blocking] Verification script regex flagged comments containing "playwright()"**
- **Found during:** Task 2 verification
- **Issue:** The plan's automated verification regex `/playwright\(\)/` was intended to catch the forbidden `provider: playwright()` function-call pattern. The workspace file's comments included "playwright() function" text which triggered a false positive.
- **Fix:** Reworded comments to say "playwright function-call" instead of "playwright()" so the regex only fires on actual code usage.
- **Files modified:** `vitest.workspace.ts`
- **Commit:** d60824f

### A3 Assumption Confirmed (Expected deviation)

**A3 from 42-RESEARCH.md:** Vitest browser mode does NOT automatically serve Tailwind 4 CSS to the browser iframe. Without an explicit CSS import in setupFiles, `window.getComputedStyle()` returns browser defaults:
- `bg-black` → `rgba(0, 0, 0, 0)` (transparent, not black)
- `aspect-square` → `height: 0` (ratio not engaged)

**Resolution per plan instructions:** Added `tests/browser/setup.browser.ts` importing `src/app/globals.css`, and wired it via `setupFiles` in the browser project of `vitest.workspace.ts`. Both assertions now return correct Tailwind-resolved values.

This deviation was explicitly anticipated by the plan: "If it fails for that reason, add a CSS import for `src/app/globals.css` to a browser-project `setupFiles` entry in `vitest.workspace.ts` and document the addition in the SUMMARY."
