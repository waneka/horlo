NOTE on await semantics: The user-perceived behavior is "row commits successfully, page redirects fast." Per D-09 + RESEARCH §"Anthropic Vision Call Reference" the LLM round-trip is ~1 second text / ~3-5s vision. Awaiting inside `addWatch` adds that latency to the user's add-watch action. To match D-09 truly fire-and-forget posture, you can EITHER:
   - **Option A (simpler, lands now):** await synchronously inside addWatch. ~1-5s latency added to addWatch. Acceptable at v4.0 scale.
   - **Option B (true fire-and-forget):** detach via `void Promise.resolve().then(async () => { ... })` so the promise runs in the background and addWatch returns immediately.
