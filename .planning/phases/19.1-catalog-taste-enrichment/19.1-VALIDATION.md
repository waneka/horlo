## Sampling Rate

- **After every task commit:** Run the task's `<automated>` command (typically `npm test -- <module>` or `npx tsc --noEmit`)
- **After every plan wave:** Run `npm test` (full suite)
- **Before `/gsd-verify-work`:** Full suite must be green; D-22 + D-21 manual UAT items confirmed
- **Max feedback latency:** 60 seconds (full suite). Quick per-module: 5 seconds.

The following **new** test files are created INSIDE Wave 1 / Wave 2 tasks (they are deliverables of those tasks, not Wave 0 prerequisites):

- `tests/integration/add-watch-photo.test.ts` (Plan 05 Task 4)
- `tests/integration/backfill-taste.test.ts` (Plan 06 Task 3)

Live-DB integration tests (Plan 04 Task 3, Plan 05 Task 4, Plan 06 Task 3) require:
- `DATABASE_URL` in `.env.local` pointing at the local Supabase Postgres
- A seeded `auth.users` row whose UUID matches `TEST_USER_ID` in `tests/integration/add-watch-photo.test.ts` (otherwise the FK from `watches.user_id` fails)
- The Plan 01 BLOCKING checkpoint (Task 4 of Plan 01) must complete first — schema columns + CHECK constraints + bucket must be applied to local DB

These prerequisites are documented in each task's `<read_first>` and `<action>` sections. Tests use `describe.skipIf(!process.env.DATABASE_URL)` to gracefully skip when the DB is unavailable (e.g., in CI without a live DB).
