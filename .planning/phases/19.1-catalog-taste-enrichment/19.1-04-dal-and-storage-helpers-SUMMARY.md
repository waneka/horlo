## Final DAL Helper Signatures

### updateCatalogTaste

```typescript
export async function updateCatalogTaste(
  catalogId: string,
  taste: CatalogTasteAttributes,
  options?: { force?: boolean },
): Promise<{ updated: boolean }>
```

- Default mode: `WHERE id = $1 AND confidence IS NULL` â€” returns `{updated: false}` if row already has taste data (D-13 first-write-wins).
- Force mode (`options.force = true`): omits the predicate â€” used by backfill scripts (Plan 06).
- Writes `extracted_from_photo` from `taste.extractedFromPhoto` (D-22).
- Empty `designMotifs` array emits `'{}'::text[]` literal (not `()::text[]` which postgres.js rejects).

### applyUserUploadedPhoto

```typescript
export async function applyUserUploadedPhoto(
  catalogId: string,
  input: { imageUrl: string; imageSourceUrl: string },
): Promise<{ applied: boolean }>
```

- COALESCE first-non-null-wins for `image_url`, `image_source_url`, `image_source_quality` (D-21).
- `imageUrl` sanitized via `sanitizeHttpUrl` â€” rejects non-http/https (T-19.1-04-04).
- `imageSourceUrl` is a bucket path (not a URL) â€” length-capped to 512 chars, not URL-sanitized.
- Returns `{applied: false}` if all three columns already non-NULL.

## Storage Helper Signatures


## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed CHECK constraint test assertion â€” use `rejects.toSatisfy()` not `rejects.toThrow()`**
- **Found during:** Task 3 integration test run
- **Issue:** `db.execute()` in drizzle wraps postgres.js errors. The outer error message is "Failed query: ..." â€” the constraint violation text lives in `err.cause.message`. `.toThrow(/violates check constraint/i)` matches only the outer message and fails.
- **Fix:** Replaced `.rejects.toThrow(regex)` with `.rejects.toSatisfy(isCheckConstraintError)` where `isCheckConstraintError` inspects both `err.message` and `err.cause?.message`. Mirrors the pattern from `tests/integration/phase17-schema.test.ts` lines 176-180.
- **Files modified:** `tests/integration/catalog-taste-schema.test.ts`
- **Commit:** c18176a (same task commit)

**2. [Rule 1 - Bug] Fixed invalid UUID in unit test constant**
- **Found during:** Task 2 unit test run (1 test failed)
- **Issue:** `VALID_CATALOG_ID = 'b1ffcd00-0d1c-5fg9-cc7e-7cc0ce491b22'` contains `5fg9` which includes `g` â€” not valid hex. The UUID regex `[0-9a-f-]{36}` rejected it, causing the path builder to throw unexpectedly.
- **Fix:** Changed to `'b1ffcd00-0d1c-4f99-cc7e-7cc0ce491b22'` (all hex).
- **Files modified:** `src/lib/storage/catalogSourcePhotos.test.ts`
- **Commit:** 61013c5 (same task commit)
