**One-liner:** LLM taste enrichment service with closed vocabularies, Zod wire schema, Anthropic tool-use (claude-sonnet-4-6), and 20 unit tests with mocked SDK.

The `src/lib/taste/` module provides the complete server-side enrichment surface for Phase 19.1. It is pure code with no database dependency — Plans 04 (DAL) and 05 (server-action wiring) consume the public `enrichTasteAttributes()` entry point.

**4. [Rule 1 - Bug] TasteSchema uses z.string() instead of z.enum() for primary_archetype/era_signal**
- **Found during:** Task 2 enricher test (test 6: out-of-vocab archetype)
- **Issue:** Plan's `TasteSchema` used `z.enum(PRIMARY_ARCHETYPES)` which rejects unknown values at Zod parse time — making the post-parse vocab filter in `validateAndCleanTaste` unreachable for those fields. Test 6 asserts that an out-of-vocab archetype results in a non-null return with `primaryArchetype=null`, which requires the value to pass Zod and be handled by the vocab filter.
- **Fix:** Changed `primary_archetype` and `era_signal` in TasteSchema to `z.string()`. The `validateAndCleanTaste` function remains the vocab gate (D-05). The Anthropic `strict:true` tool schema handles upstream rejection. This matches the stated D-05 intent: "vocab violations are dropped + warned, not coerced."
- **Files modified:** `src/lib/taste/vocab.ts`
- **Commit:** `957d3d3`
