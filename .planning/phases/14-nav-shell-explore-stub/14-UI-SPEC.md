| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Compact element spacing, nav icon-to-label gap |

## Layout Geometry

### Mobile Bottom Nav (< 768px)

**Focal point:** The Wear circle (56×56, `bg-accent`, elevated ~20px above the bar top) is the primary visual anchor of the mobile bottom nav. Its accent fill and physical elevation above the bar plane draw the eye and reinforce the core action hierarchy.

```
┌─────────────────────────────────────────────┐
│                                   ○ Wear    │ ← Wear circle: 56×56, bg-accent, elevated ~20px above bar
│  Home    Explore   [WEAR]   Add   Profile   │
│  24×24   24×24    28×28    24×24  24×24     │
│  label   label    "Wear"   label  label     │
│──────────────────────────────────────────────│
│          env(safe-area-inset-bottom)         │
└─────────────────────────────────────────────┘
```

- Bar background: `bg-background/95 backdrop-blur border-t border-border`
- Bar content height: 60px *(hardware/Figma-constrained, not a design-grid token)*
- Total height: `60px + env(safe-area-inset-bottom)`
- Position: `fixed bottom-0 left-0 right-0 z-50`
- Layout: `flex justify-between items-end` (Wear column uses `flex flex-col items-center` with negative top offset to elevate circle)
- Wear circle shadow: `shadow-[0px_10px_15px_0px_rgba(0,0,0,0.1),0px_4px_6px_0px_rgba(0,0,0,0.1)]`
- Active state: filled icon variant + `text-accent` label
- Inactive state: outline icon + `text-muted-foreground` label
- Active detection: `usePathname()` in a Client Component

### Slim Mobile Top Nav (< 768px)

- Height: 48px
- Background: `bg-background/80 backdrop-blur border-b border-border sticky top-0 z-50`
- Left: Horlo wordmark (`font-serif text-xl`)
- Right (left-to-right): Search icon → NotificationBell → Settings cog
- Icon size: 20×20 (`h-5 w-5`)
- All icons use `text-foreground` in inactive state

### Desktop Top Nav (≥ 768px)

- Height: 64px (existing `h-16`)
- Background: `bg-background/80 backdrop-blur border-b border-border sticky top-0 z-50`
- Left-to-right: Horlo wordmark · Explore link · persistent search input (routes to `/search`)
- Right-to-left: UserMenu (profile dropdown) · NotificationBell · Add icon link (`/watch/new`) · NavWearButton (Wear CTA)
- ThemeToggle removed from top strip; consolidated into UserMenu dropdown

### Profile Dropdown (desktop)

1. Signed-in-as header (email, not interactive)
2. Profile link → `/u/[username]/collection`

## Interaction Contract

### Active state detection (NAV-04)

- `BottomNav` is a Client Component
- Reads `usePathname()` on every render
- Exact match for `/` (Home), `/explore`, `/watch/new` (Add), `/u/[username]` prefix (Profile)
- Wear button has no "active route" concept (it opens a dialog, not a route) — rendered in accent color always

### Wear CTA behavior (NAV-09)

- Tapping Wear in BottomNav calls the same `NavWearButton` trigger pattern
- Do NOT fork `WatchPickerDialog` or `NavWearButton` — extend via props if behavior diverges (Pitfall I-2)
- Opens `WatchPickerDialog` as a modal overlay

### Auth-route exclusion (NAV-05)

- `BottomNav` and `SlimTopNav` render `null` when `usePathname()` matches any entry in `PUBLIC_PATHS`
- `PUBLIC_PATHS` extracted from `src/proxy.ts` into `src/lib/constants/public-paths.ts`
- Routes excluded: `/login`, `/signup`, `/forgot-password`, `/reset-password`, `/auth/*`

### iOS safe-area (NAV-03)

- Root layout viewport meta must include `viewport-fit=cover`
- `<main>` receives `pb-[calc(4rem+env(safe-area-inset-bottom))]` className
- BottomNav total height accounts for `safe-area-inset-bottom`

### NotificationBell placement (D-23, D-24)

- Bell renders in both SlimTopNav (mobile) and DesktopTopNav
- Both placements pass the same `viewerId` prop
- Bell is its own `<Suspense>` leaf at both placements (Pitfall A-1/B-1)
- `cacheLife('30s')` + `cacheTag('notifications', \`viewer:${viewerId}\`)` — unchanged from Phase 13

### Insights tab owner-gate (D-13)

- `ProfileTabs` receives `isOwner: boolean` prop
- Insights tab renders only when `isOwner === true`
- Non-owners never see the tab label — no "locked" state, no existence leak
- Data reads for Insights tab are guarded at both tab render AND DAL call

### `/insights` route retirement (D-13)

- Route file kept; contents replaced with `redirect()` to `/u/[username]/insights-tab`
- Use `getCurrentUser()` inside the page to resolve target username
- Preserves bookmarks and shared URLs
