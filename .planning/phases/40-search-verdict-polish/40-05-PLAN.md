<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL share-link → Client | Pasted URL parameters flow into `useSearchState` initial state — sanitized by Zod on the next Server Action call. |
| Chip onClick → setter → URL | All chip-emitted values are closed enums (Movement, Size) or comma-joined strings from the cached vocab (Style) — no untrusted input enters via the UI surface. |
| Style vocab → client bundle | `styleVocab` is server-fetched from `getTopStyleTags(8)` (40-01 cached) and threaded as a prop — no client-side cache invalidation; refreshes hourly per cacheLife. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-14 | Tampering | Chip component emitting unexpected value | mitigate | MovementChips + CaseSizeChips emit values from a frozen `as const` array; StyleChips emits values from the server-supplied `vocab` prop. No user-supplied free-text entry. |
| T-40-15 | Information Disclosure | URL share with active facets | accept | Facet params expose only catalog filter state — public data. URL shareability is the intended D-03 contract. |
| T-40-16 | Tampering | XSS via styleVocab tag content | mitigate | Tags are stored in `watches_catalog.style_tags` (admin-only writes; LLM-extracted via Phase 19.1 closed vocab `src/lib/taste/vocab.ts`). React JSX auto-escapes the rendered tag label. No `dangerouslySetInnerHTML` introduced. |
| T-40-17 | Denial of Service | Large styleVocab array | accept | `getTopStyleTags(8)` is hard-capped at 8 by the DAL call. Server-side cache + client-side fixed render count keeps the surface stable. |
</threat_model>
