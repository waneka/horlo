<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Server Action | Client passes facet params via `searchWatchesAction({ q, movement, size, style })` — untrusted input crosses here. |
| Server Action → DAL | Validated Zod payload flows into `searchCatalogWatches({ filters })` — parameterized Drizzle predicates from here on. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-01 | Tampering | `searchCatalogWatches` predicates | mitigate | All facet values flow through Drizzle parameterized operators (`eq`, `between`, `arrayOverlaps`) — no string interpolation into SQL text. `sql\`...\`` template at `getTopStyleTags` interpolates only the `limit` integer via parameterized binding. |
| T-40-02 | Tampering / Elevation of Privilege | `searchSchema` (Server Action input) | mitigate | `.strict()` preserved on Zod schema — rejects unknown keys (mass-assignment guard). Three new fields are `.enum(...)` (closed allowlist) or `z.string().max(500)` (length-bounded). |
| T-40-03 | Denial of Service | `searchCatalogWatches` with degenerate facet input | mitigate | `style` capped at 500 chars before split (Zod); pre-LIMIT 50 candidate cap unchanged; `getTopStyleTags` is `'use cache'` with `cacheLife('hours')` so repeated requests do not hammer the DB. |
| T-40-04 | Information Disclosure | `getTopStyleTags` cached vocab | accept | Catalog metadata is public; cached top-N is identical across all viewers (no user-specific inputs). Low-value target. |
| T-40-05 | Tampering | `style` facet value injection (user supplies tags not in vocab) | mitigate | DAL splits comma string and passes as parameterized `text[]` to `arrayOverlaps` — never builds SQL text from user data. Unknown tags simply yield no rows; no SQL surface. |
</threat_model>
