<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| script -> Anthropic API | Outbound LLM call carrying brand/model strings; web_search fetches untrusted external content |
| `.env.local` -> script process | `ANTHROPIC_API_KEY` and `DATABASE_URL` (local) read into process env |
| web_search result -> enricher | Untrusted external page content returned as `web_search_result` blocks |
| backfill-taste.ts -> supabase/migrations/ | LLM-derived taste values written into a committed SQL data migration |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-44-01 | Information Disclosure | backfill-taste.ts / reenrich-taste.ts logging | mitigate | Per-row JSON logs include only `catalog_id`, `status`, `confidence`, `error.message`, `timestamp` — never `process.env.ANTHROPIC_API_KEY` or HTTP headers. Error field uses `err.message` only, not the full error object. |
| T-44-02 | Tampering | web_search result content fed into taste tool output | mitigate | Tool output is constrained by `TASTE_TOOL` `strict` schema and re-validated by existing `TasteSchema.safeParse` / `validateAndCleanTaste` — untrusted web text cannot widen the closed taste vocab. |
| T-44-03 | Denial of Service | Anthropic rate limiter during a ~100-row batch | mitigate | SDK `maxRetries: 3` (429-aware, Retry-After respecting) plus `INTER_ROW_DELAY_MS` inter-row pacing keeps sustained request rate ~1/sec, well under Anthropic limits. |
| T-44-04 | Information Disclosure | enricher.ts never-throws posture | accept | Errors are logged via `logError` with structured payloads; stack traces may appear in local stdout. Accepted — local-only CLI script, no network log sink, single operator. |
| T-44-05 | Tampering | SQL injection via LLM-derived taste values interpolated into the emitted phase44_taste_data.sql migration | mitigate | The taste values written to the migration are first constrained by the `TASTE_TOOL` strict schema and `validateAndCleanTaste` (closed enums for `primary_archetype`/`era_signal`, numeric ranges for the scores). The migration emitter emits numbers unquoted, single-quote-escapes enum/text values, and renders `design_motifs` as a typed `text[]` literal — no raw un-validated LLM string is interpolated. |
</threat_model>
