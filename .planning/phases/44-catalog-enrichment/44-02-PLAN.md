From src/lib/types.ts:
- `CatalogTasteAttributes` includes `extractedFromPhoto: boolean` and `confidence: number`

D-08 BLOCK RULE — reject a force write when ALL three hold:
  1. existing row `extracted_from_photo = true`
  2. existing row `confidence` is non-null AND `>= 0.7`
  3. incoming write `taste.extractedFromPhoto === false`
A vision-mode incoming write (`extractedFromPhoto === true`) is ALWAYS allowed.
A text-mode write over an existing row with `confidence < 0.7` is allowed.
On block, log `taste_downgrade_guard_blocked` and return `{ updated: false }`.
