## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Component → user | The component renders LOCKED D-15 copy strings to the user. No user-controlled input is rendered (the `message` prop, if exposed, is dev-only/debug). |
| Component → caller (callbacks) | `retryAction` / `manualAction` callbacks are invoked on click; the component does NOT construct URLs internally. The caller (25-04 wiring) owns the routing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-02-01 | I (Information disclosure) | optional `message` prop rendering raw error from /api/extract-watch | mitigate | Per orchestrator security note: "error copy is locked (D-15) and must NOT include raw stack traces, internal URLs, or LLM provider names visible to the client". The `message` prop is rendered only in `process.env.NODE_ENV !== 'production'` (dev-only debug detail), and the route handler in 25-04 will sanitize the response payload such that `error` field never includes raw stack traces or "Anthropic" / "claude" provider names. The locked D-15 body copy is the user-visible surface; `message` is dev-grade. |
| T-25-02-02 | T (open redirect via manualAction) | `manualAction` callback prop wiring | mitigate | The callback is invoked from a caller that uses `router.push('/watch/new?manual=1')` — a static internal route, no user input concatenated. Component does NOT construct URLs from props. The orchestrator security note ("`?manual=1` and `?status=wishlist` query params: open redirect — the params are consumed locally by AddWatchFlow; verify no URL-construction sinks") is satisfied because the component is a pure callback consumer, and the caller (25-04) hard-codes the destination. |
| T-25-02-03 | D (rerender storm) | `retryAction` / `manualAction` callback props | mitigate | Per orchestrator: "retryAction prop must be stable (useCallback) to avoid unbounded effect loops." The component itself does NOT depend on the callbacks in any `useEffect` / `useMemo`. Stability is the CALLER's responsibility (25-04 wraps in `useCallback`). Document this in a JSDoc on the component file: "Callbacks should be stable (useCallback) to avoid effect-loop in callers that observe them." |
| T-25-02-04 | I (XSS via heading/body) | static LOCKED copy rendering | accept | Headings and body strings are compile-time constants in the component file. No user input flows into them. React escapes text content by default. No XSS surface. |
