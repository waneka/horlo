## Warnings

### WR-01 — Add-to-Collection has no re-entrancy guard
**File:** src/components/search/WatchSearchRowsAccordion.tsx:103-105
**Issue:** handleAddToCollection calls router.push directly with no committing-state UX or re-entrancy guard. A double-click fires two navigations. Add-to-Wishlist is not disabled while a Collection click is in flight.
**Fix:** Set committingId and committingTarget for the Collection path too. Disable both buttons when either is committing.

### WR-02 — Wishlist failure rollback discards typed notes
**File:** src/components/watch/AddWatchFlow.tsx:248-269
**Issue:** When addWatch fails inside the wishlist-rationale flow, rollback returns to verdict-ready without preserving the user's typed rationale.
**Fix:** Cache notes in flow state during rollback so the user lands back on WishlistRationalePanel with their text intact.

### WR-03 — Spread after explicit keys can shadow them
**File:** src/app/api/extract-watch/route.ts:134-142
**Issue:** Response shape spreads ...result AFTER explicit success/catalogId/catalogIdError keys. If fetchAndExtract ever returns one of those names, the explicit values get silently shadowed.
**Fix:** Spread ...result first, then set explicit keys after.

### WR-04 — Auto-focus by string id couples to PasteSection internals
**File:** src/components/watch/AddWatchFlow.tsx:87-92
**Issue:** document.getElementById('paste-url') couples AddWatchFlow to a string-id detail inside PasteSection with no compile-time check.
**Fix:** Lift the input ref to AddWatchFlow (or use forwardRef) and call inputRef.current?.focus().

## Info

### IN-01 — Magic number 20 in SearchPageClient.tsx
"Showing top 20" footer hardcodes the cap. Pull into a named constant or share with the server-side cap.

### IN-02 — Inline IIFE inside Accordion.Panel JSX
WatchSearchRowsAccordion.tsx:135-138 wraps cache-lookup in an IIFE. Pull to a const above the JSX or hoist into a helper.

### IN-03 — Over-broad aria-live="polite" region in VerdictStep.tsx
The polite live region wraps buttons + spec preview + verdict together. SR users hear the whole subtree announced on any change. Narrow the live region to just the verdict text.

### IN-04 — data.data ?? {} cast in AddWatchFlow.tsx:134
Empty-object fallback gets cast to ExtractedWatchData. Hides regressions if the extract route shape changes. Consider a typed default or runtime guard.

### IN-05 — Synth rail-id 'synth:brand|model' is technically collidable
Real brand names virtually never contain '|'. Practical risk is near zero. Defensive fix: encodeURIComponent each part or hash brand+model.
