## Technical changes

**Original gap 5 fix (Tasks 1+2):**

- `src/components/search/WatchSearchRowsAccordion.tsx` — `Accordion.Panel` className changed from `data-[state=open]:`/`data-[state=closed]:` (which never matched base-ui's actual DOM) to `data-[open]:`/`data-[ending-style]:`. base-ui's panel exposes `data-open` (no value) per `node_modules/@base-ui/react/accordion/panel/AccordionPanelDataAttributes.js`.
- `src/components/search/WatchSearchRow.tsx` — replaced the `buttonVariants(...)`-styled `<span>Evaluate</span>` (the styled-as-button-but-not-a-button anti-pattern) with a plain chevron-led "Tap to evaluate"/"Hide" label.

**Extended UX fix (Task 4 — verification follow-up):**

- `WatchSearchRow.tsx` — refactored: row body is a `<Link>` to `/catalog/[catalogId]` (primary action: navigate to detail page). Chevron is owned by the parent via a `trigger?: React.ReactNode` slot prop. Removed the `isOpen` prop and the "Tap to evaluate"/"Hide" right-edge label.
- `WatchSearchRowsAccordion.tsx` — only the right-edge chevron is the `Accordion.Trigger` (aria-label = `"Toggle fit for {brand} {model}"`). Chevron rotates via `data-[panel-open]:[&>svg]:rotate-180`. Removed the "Hide" CTA button. Removed `handleValueChange` instrumentation.
- `AllTabResults.tsx` — swapped bare `<WatchSearchRow>` for `<WatchSearchRowsAccordion>` so the All tab presents the same affordances as the Watches tab (closes the inconsistency reported during verification).
- `SearchPageClient.tsx` — threaded `collectionRevision` to `<AllTabResults>`.

## Why it works

- **Selector match**: base-ui's `AccordionPanel` exposes `data-open` (no value) and `data-ending-style="closed"`. The new Tailwind selectors target both — animations now run on open and close.
- **Single click target per intent**: row link = navigate; chevron button = expand. No more whole-row trigger that fights the user's intent to follow a result through to detail.
- **Consistent UX across tabs**: All tab and Watches tab now render the exact same component for the watches list — no divergent click semantics.
