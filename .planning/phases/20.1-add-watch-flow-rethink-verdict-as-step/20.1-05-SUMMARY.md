## Decisions Made

- **Wishlist payload from /search uses minimal SPEC defaults** — `SearchCatalogWatchResult` (defined at `src/lib/searchTypes.ts`) only carries identity fields (catalogId, brand, model, reference, imageUrl) plus counts + viewerState. The plan's action snippet referenced `r.movement`, `r.complications`, etc. — these don't exist on the type. Resolution: build the addWatch payload from the available fields plus required defaults (movement='automatic', empty arrays for complications/styleTags/designTraits/roleTags). The /search inline path is the FAST commit; the catalog upserter (already wired in `addWatch`) + Phase 19.1 taste enricher (also wired) backfill SPEC fields on the catalog row asynchronously. The user's `watches.id` row has the basic identity; cross-user features read SPEC from the catalog row, which gets enriched.
- **aria-label = exact button copy** — Plan's action snippet specified `aria-label={\`Add ${r.brand} ${r.model} to Wishlist\`}` (interpolation for screen-reader disambiguation). But the Plan 01 RED test uses `getByRole('button', { name: 'Add to Wishlist' })` — exact match. Resolution: kept aria-label literal to match the button copy. The Accordion only opens one row at a time, so the visible row context provides screen-reader disambiguation organically.
- **strapType set to null on /catalog/[catalogId]** — `CatalogEntry` (per `src/lib/types.ts` lines 114-149) does NOT carry `strapType`. The catalog table doesn't store it (per-user field). Set literally to null in `actionsSpec` so the deep-link to /watch/new lets the user pick strap on commit.
- **D-05 it() block assertions changed from component-name match to props match** — Plan 01's RED scaffold asserted `JSON.stringify(result).toMatch(/CatalogPageActions/)`. JSON.stringify of a React JSX element drops the function `type` field, so a regex against the component name can never match. The fix is to match unique discriminator props that only CatalogPageActions receives at the page tree level: `catalogId` (UUID string), `spec` (object), and `framing":"cross-user"`. Same intent (verify CatalogPageActions IS rendered in the right framing); satisfiable contract. See Deviation §1 below.
