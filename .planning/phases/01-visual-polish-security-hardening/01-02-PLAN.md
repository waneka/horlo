<objective>
Close the SSRF hole on `POST /api/extract-watch` (SEC-01). Currently `fetchAndExtract()` calls bare `fetch(url)` with no DNS resolution check and Next.js follows redirects automatically. Replace this with a DNS-pinning `safeFetch` that validates every resolved IP against RFC1918/loopback/link-local/reserved ranges, rejects IPv6 equivalents, and follows redirects manually with per-hop re-validation.

Purpose: Prevents the app from being used to probe the internal network, read cloud metadata endpoints, or pivot via DNS rebinding / redirect chains.

Output: `src/lib/ssrf.ts` hardening module, integrated into the extractor pipeline, with exhaustive Vitest coverage.
</objective>

  beforeEach(() => lookup.mockReset())

  it('resolves public hostname to pinned IP', async () => {
    lookup.mockResolvedValueOnce([{ address: '93.184.216.34', family: 4 }])
    await expect(resolveAndValidate('example.com')).resolves.toBe('93.184.216.34')

  it('rejects when any resolved address is private', async () => {
    lookup.mockResolvedValueOnce([

    lookup.mockResolvedValueOnce([])

    lookup.mockReset()
    vi.restoreAllMocks()

      .mockResolvedValueOnce(new Response('ok', { status: 200 }))
    const res = await safeFetch('https://example.com/page')

  })

  it('throws on redirect with no Location header', async () => {
    lookup.mockResolvedValueOnce([{ address: '93.184.216.34', family: 4 }])
    vi.spyOn(globalThis, 'fetch').mockResolvedValueOnce(
      new Response(null, { status: 302 })
    )
    await expect(safeFetch('https://example.com/')).rejects.toThrow(/Location/)
  })

  it('caps redirect chain at maxRedirects', async () => {
    lookup.mockResolvedValue([{ address: '93.184.216.34', family: 4 }])
    const fetchSpy = vi.spyOn(globalThis, 'fetch')
    for (let i = 0; i < 10; i++) {
      fetchSpy.mockResolvedValueOnce(
        new Response(null, { status: 302, headers: { location: `https://example.com/hop${i}` } })
      )
    }
    await expect(safeFetch('https://example.com/', {}, 3)).rejects.toThrow(/Too many redirects/)
  })
})

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client browser -> `/api/extract-watch` | Attacker-supplied URL crosses the wire and is fetched server-side |
| Server -> arbitrary hostname | DNS resolution can return private IPs; redirects can pivot to internal network |
| DNS response -> fetch | Time-of-check-vs-time-of-use: second DNS lookup in fetch could differ from validation lookup |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-10 | Information Disclosure (SSRF) | `fetchAndExtract` -> bare fetch | mitigate | Task 1 creates `resolveAndValidate` which rejects RFC1918/loopback/link-local/CGNAT/multicast/reserved/0.0.0.0 ranges and IPv6 equivalents before the outbound request |
| T-01-11 | Information Disclosure (DNS rebinding) | hostname-based allow-list | mitigate | Use `dns.lookup({ all: true })` and validate every returned IP; hostname strings are never trusted |
| T-01-12 | Elevation of Privilege (AWS metadata) | 169.254.169.254 | mitigate | Explicit entry in `PRIVATE_RANGES_V4` (link-local 169.254.0.0/16); regression test in `tests/ssrf.test.ts` |
| T-01-13 | Information Disclosure (redirect pivot) | redirect following | mitigate | `redirect: 'manual'` + per-hop `resolveAndValidate`; regression test asserts blocked redirect to 10.x |
| T-01-14 | Denial of Service (infinite redirect loop) | redirect chain | mitigate | `maxRedirects = 5`; regression test asserts `SsrfError('Too many redirects')` |
| T-01-15 | Information Disclosure (error message leak) | route handler response | mitigate | 400 response uses the static UI-SPEC copy; no IP, no stack trace, no underlying message |
| T-01-16 | TOCTOU (DNS changes between validate and fetch) | dual DNS lookup | accept | Node `fetch` performs its own DNS lookup. Full mitigation (dial+pin socket) requires undici `dispatcher` + `connect` override and is scoped to v2 hardening. Practical impact: requires attacker-controlled authoritative DNS, brief race window, and still faces the redirect-hop validator. Low exploit probability given phase scope. |
| T-01-17 | Spoofing (IPv6 mapped IPv4) | IPv6 format | mitigate | `PRIVATE_IPV6_PREFIXES` catches `::ffff:` via the `net.isIPv6` branch; unknown IP format falls through to `true` (fail-closed) |
</threat_model>
