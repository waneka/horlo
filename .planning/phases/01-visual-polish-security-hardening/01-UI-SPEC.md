# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Visual Polish & Security Hardening.
> Generated by gsd-ui-researcher 2026-04-11, verified by gsd-ui-checker.

---

## Product Voice

Horlo is a taste-aware decision engine for personal watch collectors. The app should read as a **curated collector's journal**, not a marketplace, not a spreadsheet, not a SaaS dashboard. Every visual choice in this phase serves that positioning:

- **Editorial, not promotional.** No gradients, no shine, no "get started" energy.
- **Warm analog, not clinical.** Neutral surfaces lean paper-warm rather than pure cool gray. Accent color evokes brass, lume, and patina — horological warmth without being kitschy.
- **Confident whitespace.** The collection is small and opinionated. Let the watches breathe.
- **Restraint with color.** Color is used to signal, not to decorate. Status, tags, and CTAs are predominantly monochromatic; accent appears only where the user needs to act or where data needs to pop.

### FOUC prevention (non-negotiable)

`next-themes` must mount in `src/app/layout.tsx` via `<ThemeProvider attribute="class" defaultTheme="system" enableSystem storageKey="horlo-theme">`. `<html>` must carry `suppressHydrationWarning`. No custom inline script is needed — `next-themes` injects its own blocking script. Do NOT ship a hand-rolled `useEffect`-based toggle; it will flash.

## Spacing Scale

8-point scale; all layout values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, badge inner padding |
| sm | 8px | Spec-row gaps, tag cluster gaps |
| md | 16px | Default card padding, form field spacing |
| lg | 24px | Card outer padding, grid gaps on mobile |
| xl | 32px | Section padding, grid gaps ≥ tablet |
| 2xl | 48px | Major section breaks on detail + insights pages |
| 3xl | 64px | Page top padding on desktop, empty-state vertical rhythm |

**Exceptions:**
- Minimum touch target: **44px** for all interactive elements on mobile (theme toggle, status filter chips, nav links). Use padding — do not force 44px font sizes.
- Watch card image gutter: 0px (image is edge-to-edge inside the card).

## Typography

Four sizes, two weights. Declared here once — the executor may not introduce additional sizes during this phase.

| Role | Font | Size | Weight | Line height | Tracking |
|------|------|------|--------|-------------|----------|
| Display (watch detail H1, page titles) | Instrument Serif | 32px mobile / 40px ≥md | 400 (regular) | 1.1 | -0.01em |
| Heading (card title, section heads) | Geist Sans | 20px | 600 (semibold) | 1.25 | -0.005em |
| Body | Geist Sans | 16px | 400 (regular) | 1.5 | 0 |
| Label / caption / spec value | Geist Sans | 14px | 400 or 600 | 1.4 | 0 |
| Spec numeric (case size, ref no.) | Geist Mono | 14px | 400 | 1.4 | 0 |

Weights: **only 400 and 600** across the whole app. No 500, no 700. If a heading needs more presence, use the serif display style, not a heavier weight.

**Forbidden:** `text-xs` below 12px, any `font-light` / `font-bold` / `font-extrabold`, any serif outside the Display role.

### Accent reserved for (explicit list — not "all interactive elements")

The brass/lume accent color may ONLY appear on:

1. Primary CTA button inner state on hover/active (the resting state is `--primary`, not accent)
2. Active/selected filter chip background in FilterBar
3. "Good deal" indicator in wishlist (Phase 2 uses this token — declared here for consistency)
4. Chart primary series (`--chart-1`) on BalanceChart
5. Focus rings on all interactive elements (`--ring` points at accent)
6. Link hover color on inline text links (not buttons)

Accent may NOT appear on: body text, icon default color, card borders, badge backgrounds, status labels, or any decorative element.

### Status color (owned / wishlist / sold / grail)

The current implementation uses hardcoded Tailwind colors (`bg-green-100 text-green-800`, etc.) which break in dark mode. Phase 1 replaces these with **monochromatic badges** that rely on outline + label rather than color, plus a small colored dot:

| Status | Badge style | Dot color (5px) |
|--------|-------------|-----------------|
| owned | `variant="outline"` — border + foreground text | `oklch(0.55 0.15 145)` (muted green) |
| wishlist | `variant="outline"` | `oklch(0.55 0.12 240)` (muted blue) |
| sold | `variant="secondary"` — muted surface | `--muted-foreground` |
| grail | `variant="outline"` with `text-accent` label | `--accent` |

This keeps status visually distinguishable without carrying four full color families through both themes.

## Component Contracts

### WatchCard (VIS-04)

- **Aspect ratio:** image container is **4:5** (portrait). Watches are vertical objects; square crops waste visual weight on wrist shadow. 4:5 shows case + full strap taper. `aspect-[4/5]` in Tailwind.
- **Image:** `next/image` with `fill`, `sizes="(max-width: 640px) 100vw, (max-width: 1024px) 50vw, 25vw"`, `object-cover`. `imageUrl` passed through a `getSafeImageUrl()` helper that returns `null` if host is not allowlisted; null renders the placeholder SVG from lucide-react (`Watch` icon) inside a `bg-muted` div. (SEC-02)
- **Hierarchy on card body (top to bottom):**
  1. Brand name — `text-base font-semibold` (16px/600), `text-card-foreground`
  2. Model name — `text-sm text-muted-foreground` (14px/400) — directly under brand, no margin
  3. Spec row — `text-xs font-mono text-muted-foreground` (Geist Mono 12px) — case size · movement · water resistance, separated by middle dots (`·`)
  4. Style tags — max 2 visible + `+N` overflow badge, `variant="outline"`
- **Status badge:** top-right overlay on image, monochromatic per table above
- **Hover:** `shadow-md` → `shadow-lg` transition 200ms, NO image zoom, NO border color change
- **Padding:** `p-4` (md token, 16px) on `CardContent`
- **Radius:** inherits `--radius-lg` from card primitive (0.625rem)

### WatchDetail (VIS-04)

- **Layout:** two-column grid `lg:grid-cols-[2fr_1fr]`. Left = image gallery (square 1:1 for detail view — user is intentionally inspecting, square maximizes face presence). Right = metadata stack. Collapses to single column `<lg`.
- **Header:** watch brand as Display (Instrument Serif 40px/1.1) + model as Body (16px muted). Reference number in Geist Mono below.
- **Spec cards:** keep current four-card layout (Specifications / Pricing / Classification / Tracking) but:
  - Replace all `text-gray-500` → `text-muted-foreground`
  - Replace all `text-gray-900` → `text-foreground`
  - Replace all `text-gray-400` → `text-muted-foreground/70`
  - Replace `border-t` → `border-t border-border`
  - Spec values in Geist Mono where numeric
- **Image:** `next/image` via `getSafeImageUrl()` wrapper (same helper as card)
- **Note:** "days since last worn" affordance on this page is Phase 2 (VIS-05), NOT this phase. Do not add it in Phase 1. The current hardcoded parenthetical `(X days ago)` may remain or be removed — executor's choice.

### WatchGrid

- Gap: `gap-6` (24px) on mobile, `gap-8` (32px) `≥lg`
- Grid columns per breakpoint table above
- Empty state: Instrument Serif heading "No watches yet" (display 32px), body caption ("Add your first watch to start your collection."), lucide-react `Watch` icon at 64px, `text-muted-foreground`, `py-24` vertical padding

### BalanceChart (VIS-06) — REWRITE

Current implementation is CSS bars — replace with real Recharts chart wired through shadcn's `Chart` primitive.

- **Install:** `npx shadcn add chart` (pulls in the shadcn Chart wrapper and Recharts)
- **Chart type:** horizontal bar chart (not donut, not pie). Bars are easier to compare than angular wedges and work better with categorical labels like "dive", "field", "dress".
- **Props:** `title: string`, `data: { label: string; count: number; percentage: number }[]`, `emptyMessage?: string`
- **Colors:** use `var(--chart-1)` through `var(--chart-5)` — chart auto-themes with dark/light via the CSS variable system. Single-series (no stacking).
- **Labels:** category name on Y-axis (left), count + percentage on the bar end. Axis labels use `--muted-foreground`, bar fill uses `--chart-1`.
- **Accessibility:** chart must have a visible title (CardHeader) and an `aria-label` summarizing the distribution. Tabular fallback not required for Phase 1.
- **Three instances on /insights page:** Style distribution, Role distribution, Dial color distribution.

### Header (layout)

- Sticky top, `bg-background/80 backdrop-blur`, `border-b border-border`
- Logo "Horlo" in Instrument Serif 20px on left
- Nav links (Collection / Insights / Preferences) center (hidden `<md`, in drawer ≥mobile)
- **Theme toggle:** right side, lucide-react `Sun` / `Moon` / `Monitor` icons, dropdown via base-ui `Popover`. Three options: Light / Dark / System. Uses `useTheme()` from `next-themes`. 44px touch target.
- Mobile: hamburger opens a drawer with nav links + theme toggle

### Root layout

- Remove hardcoded `bg-gray-50` from `<body>` — use `bg-background`
- Add `ThemeProvider` wrapping `<Header />` + `<main>` inside `<body>`
- `<html suppressHydrationWarning>` — required for next-themes

## Accessibility Contract

- All interactive elements reachable via keyboard in logical order
- All images have `alt` text — for watch cards, `alt={\`${brand} ${model}\`}`
- Color contrast: body text on background ≥ 4.5:1 (WCAG AA). Validated: foreground `oklch(0.18)` on background `oklch(0.985)` → contrast ~14:1. Dark mode: `oklch(0.96)` on `oklch(0.14)` → ~14:1. Both pass AAA.
- Accent on primary contrast: brass accent is decorative — accessible text color on accent surfaces is `--accent-foreground` (near-black), validated ≥4.5:1
- `suppressHydrationWarning` only on `<html>` (next-themes requirement), NOT on data-bearing elements
- No content conveyed by color alone — status uses label + dot, not dot alone
- Chart has descriptive `aria-label` summarizing counts

## Security-Visible UI Contracts

### SEC-02: `next/image` with remotePatterns allowlist

- `next.config.ts` gains `images.remotePatterns` with explicit allowlist. Initial allowlist (executor may extend during planning): `hodinkee.com`, `watchuseek.com`, `rolex.com`, `omega-watches.com`, `tudorwatch.com`, `seikowatches.com`, `grand-seiko.com`, `cdn.shopify.com`, `images.squarespace-cdn.com`. Pattern: `{ protocol: 'https', hostname: '**.domain.tld' }`.
- Central helper `getSafeImageUrl(url: string | undefined): string | null` in `src/lib/images.ts`. Returns the URL if host matches allowlist, `null` otherwise. WatchCard and WatchDetail both consume this helper.
- When a watch's image host is not allowlisted, the UI silently falls back to the placeholder (no error message to the user — this is not a failure state, just a gracefully-unrendered image). The underlying `imageUrl` is preserved in the data so it can render once the host is added to the allowlist.
- All raw `<img>` tags in WatchCard, WatchDetail, WatchForm (preview), and WatchGrid empty state must be removed. The only `<img>` allowed post-phase is the shadcn/base-ui primitive ones (none currently).

### SEC-01: SSRF on `/api/extract-watch`

This is a server-side change; the only user-visible surface is the error copy when extraction is blocked by the SSRF filter. Copy is declared above: **"That URL points to a private address and can't be imported."** — shown inline below the URL input on the Add Watch page. No modal. No stack trace. No IP address leaked in the error.
