    Run all 5 read-only SELECT queries from D-07 against the live catalog (prefer prod since it's single-user per `project_db_wipeable_2026_05_09`; local mirror is acceptable if current). Use whichever access method is established for this project — typically `supabase` CLI for prod or `docker exec` into the local Supabase container for the local mirror. NEVER write or modify; the queries are SELECT only.

    For each query: (a) write the SQL as a fenced ` ```sql ` code block in 49-SPIKE.md so a reader can copy-paste and re-run; (b) immediately below, embed the result as a markdown table or short prose summary with raw counts.

    The 5 queries (D-07):

    1. **Agreement count** — for each of the 10 PRIMARY_ARCHETYPES values, count rows where `primary_archetype = X AND X = ANY(style_tags)`. Output: a small table `archetype | rows where style_tags contains archetype | total rows with that archetype | agreement %`. This answers "does style restate genre?"

    2. **Divergence count** — two counts:
       - rows where `primary_archetype IS NOT NULL AND (style_tags IS NULL OR style_tags = '{}')`
       - rows where `(primary_archetype IS NULL) AND style_tags IS NOT NULL AND array_length(style_tags, 1) > 0`
       Embed both as raw numbers with a one-sentence interpretation.

    3. **Per-archetype top-3 styleTags** — `SELECT primary_archetype, unnest(style_tags) AS tag, COUNT(*) FROM watches_catalog WHERE primary_archetype IS NOT NULL GROUP BY 1, 2 ORDER BY 1, 3 DESC` then for each archetype keep the top 3 tags. Output: a table `archetype | top tag #1 (count) | top tag #2 (count) | top tag #3 (count)`. Tells us whether each genre has a distinctive style fingerprint or whether the same tags appear across all 10 genres.

    4. **Null coverage** — three raw counts on `watches_catalog`:
       - total rows
       - rows with `primary_archetype IS NULL`
       - rows with `style_tags IS NULL OR style_tags = '{}'`
       Embed as a small table; add a one-line interpretation ("X% of catalog rows lack archetype; Y% lack style — these are the rows the enrichment hasn't reached").

    5. **Watch-level disagreement examples** — `SELECT brand, model, reference, primary_archetype, style_tags FROM watches_catalog WHERE (primary_archetype IS NOT NULL AND NOT (primary_archetype = ANY(style_tags))) LIMIT 5` plus a similar query for "one is empty and the other isn't." Pick 3-5 representative rows that illustrate the divergence and embed them as a small table. If any row identifies a known watch in a way that feels exposing, anonymize the reference (e.g. "Submariner-class dive watch (anonymized)") — though per `project_db_wipeable_2026_05_09` this is a single-user DB and anonymization is largely a stylistic choice.
