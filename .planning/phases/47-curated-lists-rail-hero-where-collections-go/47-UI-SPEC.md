## Spacing Scale

Declared values (multiples of 4). All tokens are the project-standard 8-point scale already in use across Phase 46 components.

| Token | Value | Usage in Phase 47 |
|-------|-------|-------------------|
| xs | 4px | Icon-to-label gaps, badge inner padding, number-badge size |
| sm | 8px | Card inner padding (compact), connector line gap to content, chip padding |
| md | 16px | Default element spacing, card padding, section-to-content gap |
| lg | 24px | Section-to-section gap (inside `space-y-4` / `gap-6` blocks) |
| xl | 32px | Module-level vertical spacing, hero bottom margin |
| 2xl | 48px | Hero full-bleed minimum height anchor (combined with aspect-ratio) |
| 3xl | 64px | Page-level vertical padding (existing `py-8` = 32px on page shell; hero sits at full-width outside container) |

Exceptions:
- Rail card width: `w-44` (176px) mobile / `w-52` (208px) desktop — matches existing `DiscoveryWatchCard` widths (source: DiscoveryWatchCard.tsx)
- Touch targets on rail cards and path watch nodes: minimum `min-h-[44px]` — matches archetype chip pattern (source: CollectorArchetypes.tsx)
- Hero full-bleed: `aspect-video` (16:9) container with `overflow-hidden`; minimum height 240px on mobile (see CSS Chain section)

## Typography

All sizes match the established type scale in use across Phase 46 components (CollectorArchetypes.tsx, BrowseModule.tsx, DiscoveryWatchCard.tsx). Source: verified from live component code.

| Role | Size | Weight | Line Height | Usage in Phase 47 |

## Interaction Contracts

### Rail scroll

- Horizontal scroll: `overflow-x-auto` with `scroll-smooth`, no custom scrollbar styling needed.
- No snap — free scroll; rail cards are `shrink-0`.
- No keyboard navigation beyond native browser scroll; cards are individually focusable `<Link>` elements.

### Hero tap-through

- Full hero surface is `<Link href="/explore/lists/[id]">`. No separate CTA button.
- `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2` on the Link.

### Path node tap

- Each watch in a path (seed + follow-ons) is wrapped in `<Link href="/catalog/[catalogId]">`.
- Tap area: full card including image, brand, model. Min touch target `min-h-[44px]` via `DiscoveryWatchCard`.

### Sort controls

- Immediate re-sort in local state on `<select>` `onChange`. No server round-trip. No loading state needed.

### Empty/hidden modules

- EXPL-02: `return null` — zero DOM output, no spacing gap in the flex/grid container.
- Hero: `return null` when no eligible content.
- Individual `/explore/paths` sections: omit section element entirely when that path type has zero entries.

## CSS Chain Assertions

Explicitly declared per `feedback_ui_spec_css_chain_blind_spot` memory note.

### Hero full-bleed image chain

Container: `relative w-full aspect-video overflow-hidden rounded-xl min-h-[200px]`
Image: `absolute inset-0 w-full h-full object-cover`
Gradient overlay: `absolute inset-0 bg-gradient-to-t from-black/60 via-black/20 to-transparent`
Text: `absolute bottom-0 left-0 right-0 p-4`

Failure mode: if container lacks `overflow-hidden` OR image lacks `absolute inset-0`, image will escape or not fill. If container lacks explicit height anchor (`aspect-video` or `min-h-*`), height collapses to 0.

### Rail card cover image chain

Container: `aspect-square rounded-md bg-muted overflow-hidden`
Image: `w-full h-full object-cover`

Same pattern as `DiscoveryWatchCard` — verified working (source: DiscoveryWatchCard.tsx lines 35-41).

### Path node watch image chain

Reuses `DiscoveryWatchCard` unmodified → inherits `aspect-square rounded-md bg-muted overflow-hidden` + `w-full h-full object-cover`. No additional assertion needed.

### Editorial row image chain (list detail)

Container: `w-full md:w-40 aspect-square rounded-md bg-muted overflow-hidden`
Image: `w-full h-full object-cover`

Mobile: container expands to full column width. Desktop: container constrained to `w-40` (160px). `aspect-square` maintains 1:1 ratio at all sizes.

### Connector line (mobile path stack)

Connector: `w-px flex-1 bg-border` inside a `flex flex-col items-center` wrapper alongside the number badge. The `flex-1` on the line div makes it grow to fill the space between the number badge and the bottom of the watch content row. Last node: connector line omitted.

## Discretion Decisions (Claude's Discretion items resolved)

| Item | Decision | Rationale |
|------|----------|-----------|
| "New" badge recency window | 7 days | Standard "new" window; balances discovery signal vs. badge fatigue |
| Relative-timestamp format | `Intl.RelativeTimeFormat('en', { numeric: 'auto' })` — "today", "3 days ago", "last week", "2 months ago" | Built-in V8; no dependency; natural phrasing (RESEARCH.md Code Examples) |
| Week-index derivation | `Math.floor(Date.now() / (7 * 24 * 60 * 60 * 1000))` — epoch-days ÷ 7 | Simpler than ISO week; avoids year-boundary edge cases; single shared utility (RESEARCH.md Pattern 3) |
| Cache tags for rail/paths | `cacheTag('explore', 'explore:lists')` for CuratedListsRail; `cacheTag('explore', 'explore:paths')` for WhereCollectionsGo | Consistent with `explore:archetypes` and `explore:browse` naming from Phase 46 |
| `cacheLife` windows | `cacheLife('hours')` for all three modules | Matches Phase 46 modules; hero invalidation is event-driven via `explore:hero` tag, not TTL |
| `/explore/lists` sort state | Local component state (not URL params) | Avoids full page re-navigation on sort change (RESEARCH.md Pitfall 7) |
| Grid columns — `/explore/lists` | `grid-cols-2` mobile, `grid-cols-3` md, `grid-cols-4` lg | Consistent with DiscoveryWatchCard rail card width; 4 cols at lg gives ~4 × 208px ≈ fills max-6xl container |
| `/explore/paths` section ordering | `PATH_TYPES` declaration order: Going Deeper → Branching Out → Trading Up → Filling a Gap | Canonical order from `src/app/actions/cms/collectionPaths.ts:33`; least to most change |
| `PATH_TYPES` extraction | Extract to `src/lib/pathTypes.ts` | Avoids importing a `'use server'` file from a Server Component page; three consumers need the constant (RESEARCH.md open question 2) |
| Hero grid span | `HeroModule` renders with internal `w-full` and `aspect-video`; the explore page shell must give it `md:col-span-2` | Hero is full-bleed; other modules are single-column |
