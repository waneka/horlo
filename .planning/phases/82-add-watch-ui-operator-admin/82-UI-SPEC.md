## Color

| Role | Value (light / dark) | Usage |
|------|----------------------|-------|
| Dominant (60%) | `oklch(0.985 0.003 75)` / `oklch(0.14 0.005 75)` | Page background (`--background`); admin layout body |
| Secondary (30%) | `oklch(1 0 0)` / `oklch(0.19 0.005 75)` | Queue row cards (`--card`); dialog backgrounds (`--popover`); combobox popup |
| Accent (10%) | `oklch(0.76 0.12 75)` / `oklch(0.78 0.13 75)` | See reserved-for list below |
| Muted | `oklch(0.95 0.005 75)` / `oklch(0.24 0.005 75)` | Read-only chip background (`bg-muted`); affordance button hover; empty state text |
| Destructive | `oklch(0.55 0.22 27)` / `oklch(0.65 0.2 27)` | Merge action only (no direct data destructive in this phase beyond merge which moves rows) |

**Accent reserved for (exhaustive list):**
1. `Combobox.Item` highlighted state: `data-[highlighted]:bg-accent data-[highlighted]:text-accent-foreground` — source: SearchEntry.tsx L262
2. Merge pre-flight radiogroup selected radio: `bg-accent text-accent-foreground dark:bg-accent dark:text-accent-foreground` — per `[[accent-is-active-token]]` memory
3. AdminSubNav active link: active-state underline class applied; no `bg-accent` fill (ghost button with text-foreground underline — existing pattern)
4. Deep-link scroll-to highlight flash on brand row: `data-[highlighted=true]:bg-accent/30 dark:data-[highlighted=true]:bg-accent/20` with `transition-colors duration-300`

**GUARDRAIL — `bg-primary` forbidden for selected states:** Use `bg-accent text-accent-foreground` (with matching `dark:` pair) for any selected radio, highlighted combobox item, or active toggle. `bg-primary` in dark mode resolves to `oklch(0.96 0.005 75)` — near-white — and produces an illegible blinding fill. Source: `[[accent-is-active-token]]` memory.

**GUARDRAIL — outline Button dark override pairing:** Any explicit `bg-*` override on `variant="outline"` Button must be paired with a matching `dark:bg-*`. The Button CVA already injects `dark:bg-input/30 dark:hover:bg-input/50` — un-prefixed `bg-X` wins in light but loses in dark. Source: `[[button-outline-dark-override]]` memory.

Source: `src/app/globals.css` `:root` + `.dark` blocks, `src/components/watch/SearchEntry.tsx` L262, RESEARCH.md §Anti-Patterns

## Interaction Contracts

### BrandPicker (UI-01 + UI-02)

**API surface:**
```
props:
  brands: { id: string; name: string }[]
  value: { id: string; name: string } | null
  onChange: (next: { id: string; name: string }) => void
  onCouldntFind?: (typed: string) => void   // optional — merge dialog omits
```

**Controlled-open pattern (verbatim from SearchEntry.tsx L207–232):**
- `inputValue` controlled; `onInputValueChange` guard: ignore reasons !== `'input-change'`
- `filter={null}` disables base-ui internal string-match; `filteredItems={filteredBrands}` passes client-filtered list
- `open` controlled boolean; `onOpenChange` re-syncs on Escape/outside-click

**Client filtering:**
- `brands.filter(b => b.name.toLowerCase().includes(inputValue.toLowerCase().trim()))`
- No server round-trips — full list SSR-fetched once at page load (D-82-02)

**Item highlighting:** `data-[highlighted]:bg-accent data-[highlighted]:text-accent-foreground` on `Combobox.Item` — exact pattern from SearchEntry.tsx L262

**UI-02 affordance placement:**
- Gate: `filteredBrands.length === 0 && inputValue.trim().length > 0 && !!onCouldntFind`
- Element: `<button type="button">` rendered as SIBLING of `Combobox.List` inside `Combobox.Popup` — NOT inside the List
- SRCH-03 lesson: listbox-internal placement swallows clicks in real browsers; sibling placement preserves native click semantics. Source: SearchEntry.tsx L325–334 comment, RESEARCH.md §Pitfall 1
- Copy: `Couldn't find that brand — add as "{typed}"`  (verbatim — D-82-05, ROADMAP.md language)
- Classes: `mt-1 flex w-full items-center gap-2 rounded-md px-3 py-3 text-sm text-muted-foreground hover:bg-muted hover:text-foreground focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring min-h-[44px]`
- On click: calls `onCouldntFind(inputValue.trim())`, sets `open(false)` — panel locks typed string, `value.id = null`. No toast, no hint (D-82-04 silent)
- `data-testid="brand-picker-couldnt-find"` on the affordance button

**Accessibility:**
- `role="combobox"` on Combobox.Input (base-ui default)
- `aria-label="Search brands"` on Combobox.Input
- Popup list: `role="listbox"` (base-ui default)

**Test assertions required (per `[[assert-disappearance-too]]`):**
- Affordance click must assert BOTH: (a) popup closes (`queryByRole('listbox')` absent) AND (b) onCouldntFind called with typed string

### WatchForm Read-Only Chips (UI-03)

**Gate:** `watch.catalogId != null`

**Read-only chip element (exact class from WatchForm.tsx L376–378):**
```jsx
<div
  aria-readonly="true"
  className="flex h-9 items-center rounded-md border border-input bg-muted px-3 text-sm capitalize"
>
  {canonicalBrand ?? watch.brand}
</div>
```

**Display source:** `canonicalBrand` (from `brands.name` LEFT JOIN projection on `getWatchById`) with fallback to `watch.brand`. Rationale: RESEARCH.md Pitfall 4 / CONTEXT.md: "display should not depend on Phase 81's write-time overwrite having landed." Source: RESEARCH.md Open Question 1, recommendation Option B.

**Admin link cluster:**
- Gate: `mode === 'edit' && watch.catalogId != null && viewerIsAdmin === true`
- Container: `<div className="flex gap-2 mt-1">`
- Two ghost-variant chip buttons rendered as Next.js `<Link>`:
  - `<Button variant="ghost" size="sm" render={<Link href={`/admin/brands#brand-{watch.brandId}`} />}>Edit brand</Button>`
  - `<Button variant="ghost" size="sm" render={<Link href={`/admin/families?brandId={watch.brandId}`} />}>Edit family</Button>`
- Non-admin owners see read-only chips without the link cluster (no disclosure)

**Legacy fallback:** `watch.catalogId === null` → keep existing `<Input>` fields unchanged

### Admin Queue Rows (OPS-01 + OPS-02)

**Row structure (exact from ListIndexClient.tsx L152–215):**
```jsx
<Card key={brand.id} id={`brand-${brand.id}`} data-testid={`brand-row-${brand.id}`}>
  <CardContent className="pt-4">
    <div className="flex items-start gap-3">
      {/* Metadata block */}
      <div className="flex-1 min-w-0">
        <div className="flex flex-wrap items-center gap-2 mb-1">
          <span className="font-semibold truncate">{brand.name}</span>
          {brand.needsReview && (
            <Badge variant="secondary">needs review</Badge>
          )}
        </div>
        {/* Supporting info: country, family count, alias chips */}
      </div>
      {/* Actions block */}
      <div className="flex gap-2 shrink-0">
        <Button size="sm" variant="outline" onClick={handleConfirm}>Confirm as new</Button>
        <Button size="sm" variant="outline" onClick={() => setRenameTarget(brand)}>Rename</Button>
        <Button size="sm" variant="destructive" onClick={() => setMergeTarget(brand)}>Merge into…</Button>
      </div>
    </div>
  </CardContent>
</Card>
```

**Sort order:** `ORDER BY needs_review DESC, name ASC` — `needs_review = true` rows float to top

**`needs_review` pill:** `<Badge variant="secondary">needs review</Badge>` — lowercase, no capitalization

**Alias chips (OPS-02 family rows):** Existing aliases as inline `<Badge variant="secondary">` chips in the row metadata block

**`data-testid` convention:** `brand-row-{brand.id}` on each Card root; `family-row-{family.id}` on each family Card — used by deep-link scroll and test selectors

### Dialogs

**Rename dialog:**
- Title: `Rename {name}` (e.g. "Rename Hamilton Watch")
- Field: `<Input>` pre-filled with current name, `<Label>Name</Label>`
- Footer: `<Button variant="outline">Cancel</Button>` + `<Button variant="default">Save</Button>`
- `showCloseButton={false}` per ListIndexClient.tsx L225 pattern

**Merge dialog (OPS-01):**
- Title: `Merge {sourceName} into…`
- Content: `<BrandPicker brands={allBrandsExceptSource} value={target} onChange={setTarget} />` (no `onCouldntFind` — merge dialog omits it)
- Pre-flight guard (fires when source has ≥1 families): "Source brand has {N} families. Merging will move all families to target. Continue?"
  - WAI-ARIA radiogroup: `role="radiogroup"` on wrapper, `role="radio"` on options, roving tabindex, arrow-key navigation, first radio focused on Dialog open (mirrors Phase 68 D-03 pattern)
  - Option 1 (default): "Move all {N} families to target" — `bg-accent text-accent-foreground dark:bg-accent dark:text-accent-foreground` for selected state
  - Option 2: "Cancel — resolve families first"
  - Source: CONTEXT.md D-82-12, RESEARCH.md §Pattern 3 (critical transaction ordering note)
- When source has 0 families: skip pre-flight, render BrandPicker + confirm button directly
- Footer: `<Button variant="outline">Cancel</Button>` + `<Button variant="destructive">Merge</Button>`
- Success: `toast.success('Brand merged.')`, `revalidatePath('/admin/brands')` + `revalidatePath('/admin/families')`
- Error: `toast.error("Couldn't merge brand. Try again.")`

**Add-Alias dialog (OPS-02):**
- Title: `Aliases for {familyName}`
- Content (top): existing aliases as removable chips — `flex flex-wrap gap-2` parent (NOT `space-y-*`) with `<Badge variant="secondary">{alias}<Button size="icon-sm" variant="ghost" aria-label={`Remove alias ${alias}`}><X /></Button></Badge>` per alias
- Content (bottom): `<Label>New alias</Label><Input placeholder="e.g., submariner" />` + `<Button variant="default" size="sm">Add alias</Button>`
- Normalization: `alias.trim().toLowerCase()` before storage — must match resolver's `lower(trim($1))` SQL (RESEARCH.md Pitfall 3)
- `space-y-*` guardrail: chip list needs `flex flex-wrap gap-2` on parent, NOT `space-y-*`. Source: `[[space-y-inline-block-siblings]]` memory
- Alias chip: `<Badge variant="secondary">` (inline-block); remove button `<Button size="icon-sm" variant="ghost">`
- Success: silent row refresh via `router.refresh()` (no toast for add-alias — minor non-destructive action)
- Remove-alias: `toast.success('Alias removed.')` on success; `toast.error("Couldn't remove alias. Try again.")` on error

### Deep-Link Scroll + Highlight (`/admin/brands#brand-{brandId}`)

**Trigger:** WatchForm "Edit brand" link with hash fragment; BrandsQueue `useEffect` fires on mount.

**Implementation:**
```
useEffect(() => {
  const hash = window.location.hash   // '#brand-{id}'
  if (!hash.startsWith('#brand-')) return
  const id = hash.slice('#brand-'.length)
  const el = document.getElementById(`brand-${id}`)
  if (el) {
    el.scrollIntoView({ behavior: 'smooth', block: 'center' })
    el.dataset.highlighted = 'true'
    const timer = setTimeout(() => { delete el.dataset.highlighted }, 1000)
    return () => clearTimeout(timer)
  }
}, [])  // fires once on mount; hash is stable
```

**Tailwind classes on Card row:** `data-[highlighted=true]:bg-accent/30 dark:data-[highlighted=true]:bg-accent/20 transition-colors duration-300`

**`[[router-cache-stale-instance]]` note:** Effect runs on mount only. The operator workflow (WatchForm "Edit brand" → new navigation) always triggers a fresh mount. Acceptable.
