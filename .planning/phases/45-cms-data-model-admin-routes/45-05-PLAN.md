| client navigation → /admin pages | A non-owner must not reach admin pages |
| admin form input → Server Action | All mutations cross into owner-gated actions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-45-18 | Elevation of Privilege | non-owner navigating to /admin/lists or /admin/paths | mitigate | `src/app/admin/layout.tsx` server-component `assertOwner()` + `redirect('/')` on initial segment load |
| T-45-19 | Elevation of Privilege | non-owner reaching an /admin child page via client navigation (Partial Rendering — layout does not re-run) | mitigate | Defense-in-depth: every Server Action the page can invoke independently calls `assertOwner()` (D-06); the layout guard is UX-layer only |
| T-45-20 | Tampering | XSS via the markdown preview pane | accept | Owner-authored content only; `react-markdown` blocks `javascript:` URLs by default; preview pane is not a `prose`-injected raw-HTML sink |
| T-45-21 | Tampering | unauthorized cover image upload | mitigate | `uploadCmsCover` writes to the `cms-covers` bucket; storage RLS gates INSERT on the `is_admin` EXISTS predicate (Plan 02) |
</threat_model>
