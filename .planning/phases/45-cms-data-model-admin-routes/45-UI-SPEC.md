## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, badge padding, inline chip gap |
| sm | 8px | Input-to-label gap, compact list row spacing, button gap |
| md | 16px | Form field vertical rhythm, card padding, section internal spacing |
| lg | 24px | Section separator, card-to-card gap in list index |
| xl | 32px | Page content top padding, editor section breaks |
| 2xl | 48px | Major section breaks (e.g. between List Metadata and Items sections) |
| 3xl | 64px | Page-level bottom safe area |

Exceptions:
- Up/down reorder buttons: 32px touch target minimum (icon-sm = 28px + 4px padding offset). Do NOT go below 32px for the hit area.
- Cover image upload drop zone: 128px tall minimum (fixed height, not a spacing token).
- Search picker dropdown: max-height 240px with scroll, not constrained by scale.

Source: standard 8-point scale; exceptions derived from CONTEXT.md D-12 (up/down buttons) and D-14/D-15 (cover image).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Form field values, list item titles, search results, commentary textarea |
| Label | 14px (text-sm) | 500 (medium) | 1.4 | Form field labels, section headings within editor, chip text |

- `text-xs` (12px) is used for status badges, item count indicators, and muted metadata only — not for primary content.

Source: pattern-matched from `src/components/watch/WatchCard.tsx`, `src/components/ui/input.tsx`, `src/components/ui/button.tsx`.

## Layout

### Admin pages are utilitarian, not themed differently from the main app.

- Reuse the existing global `<Header>` and `<BottomNav>` (owner sees them on all pages).
- `/admin/layout.tsx` wraps children in a plain `<main className="mx-auto max-w-2xl px-4 py-8">` container — same max-width as the watch form pages.
- No admin-specific sidebar. No dedicated admin navigation widget.
- Sub-navigation between `/admin/lists` and `/admin/paths` is handled by two `Button` links at the top of each page (or within a shared admin sub-nav component), `variant="ghost"`, with an underline-style active state matching the app's existing pill pattern.

### Editor page layout

- Single-column, top-to-bottom form on mobile.
- On desktop (≥768px): no change — stays single-column at max-width 672px.
- Cover image: 16:9 aspect ratio container (`aspect-video`) above the title fields.
- Markdown editor: Tabs component ("Edit" | "Preview") above the textarea. Preview tab renders react-markdown output in a bordered `div` of the same height as the textarea.
- List items section: bordered card list, each item showing watch thumbnail (40×40px, `object-cover`, `rounded-sm`) + brand/model text + commentary textarea + up/down/remove controls in a `flex items-start gap-3` row.
- Path nodes section: same card-per-node pattern; node 0 is the seed watch (no remove button); nodes 1–3 are follow-ons (each has a remove button to clear that slot).

## Interaction Contracts

### Search-as-you-type watch picker (D-11)

- Minimum query length: 2 characters (enforced in `searchCatalogWatches`).
- Debounce: 200ms. No debounce needed for ≤100 catalog rows but add 200ms anyway to avoid SA call-on-every-keystroke.
- Dropdown: appears below the input, `z-50`, `bg-popover`, `border border-border`, `rounded-lg`, `shadow-md`, max-height 240px with `overflow-y-auto`.
- Each result row: `flex items-center gap-2 px-3 py-2 text-sm hover:bg-muted cursor-pointer`. Shows brand + model + reference in a single line.
- "No results" state: single row with `text-muted-foreground` copy "No matches".
- Selecting a result: closes the dropdown, adds the watch to the list/path, clears the search input.
- Already-added watches: show a checkmark icon in the result row and are not re-addable (disabled row).

### Up/down reorder (D-12)

- Up button: disabled when item is first in list.
- Down button: disabled when item is last in list.
- Disabled state: `disabled:opacity-50 disabled:pointer-events-none` (inherited from Button variants).
- No animation on reorder — list re-renders synchronously after Server Action resolves.
- Optimistic update: not required. Show a loading spinner on the button during the SA call using the `useFormStatus` or a local `isPending` flag.

### Markdown editor (D-13)

- Default tab: "Edit".
- Preview tab: renders react-markdown. No live-update on keystroke — updates only when switching to the Preview tab.
- Preview container: `min-h-[120px] p-3 rounded-lg border border-input bg-transparent text-sm leading-relaxed`.

### Cover image upload (D-14, D-15)

- Input type: `<input type="file" accept="image/jpeg,image/png,image/webp">` hidden behind a styled upload area.
- Upload area: 16:9 aspect ratio (`aspect-video`), `bg-muted`, `rounded-lg`, `border-2 border-dashed border-border`, centered `ImageIcon` with "Upload cover image" label below in `text-sm text-muted-foreground`.
- After upload: image renders in the same container with `object-cover w-full h-full`. A "Remove" button (`Button size="xs" variant="ghost"`) appears in the top-right corner of the container.
- File size guard: 4 MB. If exceeded, show toast: "Image is too large. Maximum size is 4 MB."
- Processing state: replace the upload area with a `Skeleton` of the same dimensions during EXIF strip + canvas re-encode + upload.
- CSS chain assertion: `aspect-video` produces the 16:9 ratio. `object-cover` on the `<Image>` fills it without stretching. Verify both classes are present on the container and img respectively — do NOT rely on intrinsic image dimensions.

### Path-type chip selection (D-16)

- Four chips rendered as `Button size="sm"` in a `flex flex-wrap gap-2` row.
- Exactly one must be selected before the path can be published.
- Visual state: `variant="secondary"` for selected; `variant="outline"` for unselected.
- Selection is an uncontrolled local state (`useState`) until form submission.
- Chips render the exact strings from CONTEXT.md D-16: "Going Deeper", "Branching Out", "Trading Up", "Filling a Gap".

### Draft / Publish controls (CMS-06)

- "Publish" button is disabled (not hidden) when the list has zero items. Add `disabled` attribute and tooltip via `Tooltip` primitive: "Add at least one watch to publish."
- "Unpublish" shows only when `status === 'published'`.
- "Publish" shows only when `status === 'draft'`.
- These two states are never shown simultaneously.

### Hero pin control (CMS-08)

- Located at the top of the list editor when `status === 'published'`.
- "Pin as Hero" button: `Button variant="outline" size="sm"`. Opens a `Dialog` asking for optional expiry date (`<input type="date">`), with "Pin" and "Cancel" buttons.
- When a pin is active: show "Pinned as Hero" `Badge variant="default"` and a "Clear Pin" `Button variant="ghost" size="sm"` next to it.
- No expiry required — the date input is optional.

## CSS Chain Assertions

Per memory note `feedback_ui_spec_css_chain_blind_spot`: the checker validates declared tokens, not whether the CSS chain produces the visual contract. Assert the following chains explicitly:

1. **Cover image 16:9 container:** `<div className="aspect-video ...">` must produce a 16:9 container. The child `<Image>` or `<img>` must have `className="object-cover w-full h-full"` (or `fill` prop for Next.js `<Image>`). Assert both classes are co-present.
2. **List item thumbnail:** 40×40px container `className="size-10 rounded-sm overflow-hidden bg-muted"` with `<Image className="object-cover size-full">`. No `aspect-*` class needed — `size-10` makes it square.
3. **Markdown preview pane:** `className="min-h-[120px] p-3 rounded-lg border border-input bg-transparent text-sm leading-relaxed"` — no overflow-hidden that would clip preview content. Use `overflow-y-auto` instead if height is capped.
