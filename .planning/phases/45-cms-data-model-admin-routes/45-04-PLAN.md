<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP POST → CMS Server Action | Server Actions are HTTP-callable; bypass layout guards |
| picker query → catalog search | Untrusted search string crosses into a DB query |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-45-13 | Elevation of Privilege | direct HTTP POST to a path/settings/picker action by a non-owner | mitigate | `assertOwner()` is the first statement in every action (D-06) |
| T-45-14 | Tampering | invalid path_type value bypassing the DB CHECK | mitigate | zod `z.enum([...])` on `pathType` rejects out-of-vocabulary values before the DB call; DB CHECK is the second layer |
| T-45-15 | Tampering | SQL injection via the catalog picker query string | mitigate | `searchCatalogWatches` uses Drizzle parameterized binds (verified in src/data/catalog.ts) |
| T-45-16 | Tampering | mass assignment via extra keys in action payload | mitigate | All zod schemas use `.strict()` |
| T-45-17 | Spoofing | pinning a draft (unpublished) list as hero | mitigate | `setPinnedHero` zod validates a uuid; Phase 47 hero render quality-gates on `status='published'` — and a list can only become hero-eligible via `publishCuratedList` |
</threat_model>
