## Option B Decision Rationale (RESEARCH §10 Q1)

SC#2 in the ROADMAP was ambiguously worded ("drops the status filter or is simplified accordingly — both are correct once the flip is gone"). However, SC#3 and SC#6 are the binding constraints: wishlist/grail/sold rows must NOT trigger the redirect. Option A (dropping the `status='owned'` filter) would break this invariant — the redirect would fire on wishlist watches, reintroducing BUG-01 in a worse form (redirect rather than framing error). Option B keeps the filter and simplifies the SELECT projection, producing cleaner code while satisfying all constraints. The WR-01 predicate-AST guard test continues to pass unchanged, confirming the filter is preserved.

## Threat Flags

None. The redirect URL is constructed from `viewerOwnedRow.id` — a Drizzle-fetched UUID from the `watches` table, not from request input. The `/watch/` prefix is a static template literal. Open-redirect class is structurally impossible (T-50.1-01). Cross-user leak is blocked by the `userId AND catalogId AND status='owned'` triple-predicate scope (T-50.1-02). The redirect lives at the page layer (not `proxy.ts`), avoiding Router Cache poisoning (T-50.1-03).
