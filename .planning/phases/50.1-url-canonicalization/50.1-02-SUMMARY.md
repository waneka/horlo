# Phase 50.1 Plan 02: Proxy Regression Guard Summary

**One-liner:** Static-grep vitest assertion in proxy.test.ts fails CI if any future commit adds `/catalog/` redirect logic to `src/proxy.ts`.

## What Was Built

A single new `describe` block was appended to `tests/proxy.test.ts` (lines 115-125):

```
describe('proxy.ts — SC#4 ARCH-02 regression guard (no /catalog/ redirect logic)', () => {
  it('no catalog redirect — src/proxy.ts must not contain /catalog/ handling ...', async () => {
    const fs = await import('node:fs')
    const content = fs.readFileSync('src/proxy.ts', 'utf8')
    // rationale comment ...
    expect(content).not.toContain('/catalog/')
  })
})
```

This is a static-grep assertion — no mocking, no runtime behavior, no code changes to `src/proxy.ts`.

## Rationale

The memory `feedback_proxy_router_cache_poisoning` documents that a `NextResponse.redirect` (any 3xx) issued from `proxy.ts` on an RSC prefetch poisons Next 16's in-memory Router Cache, causing 404s on subsequent soft-nav clicks. The canonical example is the `/u/*` route — see `.planning/debug/profile-page-404-top-nav.md`.

Phase 50 spike chose Variant B (page-layer `redirect()`) for the ARCH-02 catalog URL canonicalization precisely because `proxy.ts` MUST NOT contain `/catalog/` redirect logic. The current `src/proxy.ts` is clean (verified: `grep -c "/catalog/" src/proxy.ts` returns `0`). This test locks that invariant so any future commit that accidentally introduces catalog redirect logic into the proxy fails CI immediately.

## Idiom Replication

The implementation exactly replicates the canonical `fs.readFileSync` + `expect(...).not.toContain(...)` pattern from `tests/app/insights-retirement.test.tsx:47-52`:

```typescript
// Canonical analog (insights-retirement.test.tsx:48-51)
const fs = await import('node:fs')
const content = fs.readFileSync('src/app/insights/page.tsx', 'utf8')
expect(content).not.toContain("'use cache'")
```

Dynamic import of `node:fs` inside the test body (not a top-level import) matches the established idiom.
