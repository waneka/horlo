## Interaction Spec

### Search input — debounce + AbortController (D-04 LOCKED)

```
Keystroke → setQuery(value) (fires immediately, updates Combobox.Input value)
       └─→ useEffect debounces → 250ms timeout → setDebouncedQuery(value)
              └─→ useEffect on debouncedQuery:
                    1. Check length: < 2 → setResults([]); return
                    2. Check cache: hit → setResults(cached); return
                    3. AbortController.signal guards stale results
                    4. searchCatalogForAddFlow({ q }) Server Action call
                    5. On success → cache.set + setResults(res.data)
                    6. On stale (controller.signal.aborted) → return early
                    7. On unmount → controller.abort() cleanup
```

**Debounce value: 250ms** — mirrors `useSearchState.ts:130-133` byte-for-byte
per D-04. SC#1 verbatim: "fires results after ~250 ms debounce."

**No fetch fires for `query.length < 2`** — this is the SRCH-18 floor.

### Combobox keyboard navigation

Handled automatically by `@base-ui/react/combobox` 1.3.0 (per D-01). The
contract executor depends on:

| Key | Behavior | Source |
|-----|----------|--------|
| `ArrowDown` | Open popup if closed; move active option down; wrap at end | base-ui Combobox WAI-ARIA implementation |
| `ArrowUp` | Move active option up; wrap at top | base-ui |
| `Home` | First option | base-ui |
| `End` | Last option | base-ui |
| `Enter` | Pick active option → fires `onValueChange(picked)` → SearchEntry calls `onPick(picked)` | base-ui |
| `Escape` | Close popup; clear active option (does NOT clear input value) | base-ui |
| `Tab` | Move focus to next focusable element (does NOT pick active option) | Standard browser |

**Scroll-into-view on arrow nav:** base-ui Combobox handles automatically per the
WAI-ARIA Combobox pattern. The active option scrolls into view when keyboard
navigation moves beyond the popup viewport. Verified during executor implementation.

## Accessibility Contract

Every ARIA attribute below must be present in the rendered DOM. The
`@base-ui/react/combobox` 1.3.0 primitive provides the full WAI-ARIA Combobox
contract automatically (D-01 research-confirmed). This contract pins what
SearchEntry must explicitly add on top.

### SearchEntry — Combobox roles (provided by base-ui)

| Attribute | Element | Value | Source |
|-----------|---------|-------|--------|
| `role="combobox"` | Combobox.Input internal | static | base-ui Combobox |
| `aria-expanded` | Combobox.Input | dynamic boolean | base-ui (auto-tracks popup state) |
| `aria-controls` | Combobox.Input | id of Combobox.Popup | base-ui |
| `aria-activedescendant` | Combobox.Input | id of active option | base-ui (auto-tracks keyboard focus) |
| `role="listbox"` | Combobox.List | static | base-ui (renders via internal slot) |
| `role="option"` | Combobox.Item | static | base-ui |
| `aria-selected` | Combobox.Item | dynamic boolean | base-ui (tracks active option) |
