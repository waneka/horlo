## Spacing Scale

Declared values (multiples of 4 only — 8-point grid).

| Token | Value | Tailwind Class | Usage in Phase 68 |
|-------|-------|----------------|-------------------|
| xs | 4px | `gap-1` | — |
| sm | 8px | `gap-2` | CTA button-row gap on mobile (`gap-2`); radiogroup gap between buttons |
| md | 16px | `p-4` | CardContent horizontal padding (via `px-4` from Card primitive) |
| lg | 24px | `gap-6` | Root `space-y-6` section rhythm |
| xl | 32px | — | Not used in this component |
| 2xl | 48px | — | Not used in this component |
| 3xl | 64px | — | Not used in this component |

**Exceptions:**
- `gap-1.5` (6px): icon-to-text gap inside the grail button span — matches the Button primitive's `gap-1.5` default (button.tsx line 24). Justified: icon + text share a flex span, not a button-level gap.
- `gap-3` (12px): reference + year 2-col grid gap — lighter than WatchForm's `gap-6` (appropriate for inline-edit vs full form context); CTA button-row desktop gap (`sm:gap-3`).
- `gap-4`: CardContent flex gap between cover photo and identity text block (matches VerdictStep.tsx:63 `gap-4`).
- `min-h-[44px]` (44px): radiogroup buttons MUST meet touch-target minimum per WCAG 2.5.5. The default Button `h-8` (32px) is below the tap-target floor. Override with `min-h-[44px]` on the radiogroup Button instances. **CSS chain assertion:** `min-h-[44px]` overrides `h-8` from the Button CVA; Tailwind 4 processes utility classes in declaration order — `min-h` takes precedence over `h` when both are applied.

**Accent reserved for (10% rule):**
- `border-primary` — active/selected radiogroup button border
- `bg-primary/10` — active/selected radiogroup button fill overlay
- `text-primary-foreground` — primary CTA button text (via `default` variant)
- `bg-primary` — primary CTA button background (via `default` variant)

**Dark mode compliance:** `bg-primary/10` uses CSS variable `--primary` which resolves to `oklch(0.96 0.005 75)` in dark mode — a near-white at 10% opacity produces a legible, theme-aware selected state on dark card backgrounds. Verified via globals.css `.dark` block.

**Selected-state radiogroup token chain (assertable):**
```
Button[aria-checked=true] 
  → base: "border-border bg-background hover:bg-muted"
  → override via cn(): "border-primary bg-primary/10"
  → CSS chain: border-color resolves to var(--primary); background resolves to color-mix(var(--primary) 10% transparent)
  → light: border ~oklch(0.22) deep charcoal; fill ~oklch(0.22/10%) ≈ very light warm tint
  → dark: border ~oklch(0.96) near-white; fill ~oklch(0.96/10%) ≈ very subtle warm light on dark card
```

## Interaction Spec

### Status picker → label flip cascade

When `status` changes (via `onStatusChange` callback from picker):

1. **Price label** flips: `isOwned = status === 'owned'` → Label reads "Price paid" (owned) or "Target price" (wishlist, grail).
2. **CTA label** flips: `CTA_LABELS[status]` → "Add to Collection" (owned) / "Add to Wishlist" (wishlist) / "Save as Grail" (grail).
3. **Selected radiogroup button** gains `border-primary bg-primary/10`; previously selected button reverts to `border-border bg-background`.
4. **`aria-live="polite"`** on the root div announces both the CTA label change and price label change to screen readers without interrupting in-progress speech.

**CSS chain assertion — label flip:**
```
status === 'owned' → isOwned = true → <Label>Price paid</Label>
status !== 'owned' → isOwned = false → <Label>Target price</Label>
No DOM node is added/removed — the text node updates in place. aria-live="polite" triggers an announcement.
```

### Pending state

When `pending = true`:
- Primary CTA: `disabled` attribute set → `pointer-events-none opacity-50` (from Button CVA line 7); inner content swaps to `<Loader2 className="size-4 mr-2 animate-spin" aria-hidden="true" /> Saving...`
- "Edit details" button: `disabled` attribute set → same opacity/pointer-events suppression.
- "Start over" button: `disabled` attribute set → same.
- Radiogroup buttons: no `disabled` in pending state (status change during a pending commit is harmless — the parent ignores new `onStatusChange` calls while the Server Action is in flight). Phase 70 is responsible for disabling the picker if needed.

**CSS chain assertion — pending CTA:**
```
pending=true → Button[disabled] → .disabled:pointer-events-none.disabled:opacity-50 (button.tsx:7)
               → inner: <Loader2 className="size-4 mr-2 animate-spin">
               → CSS: animation: spin 1s linear infinite (tw-animate-css, imported in globals.css line 2)
```

### Keyboard navigation — radiogroup

Implementation of the WAI-ARIA 1.2 Radio Group design pattern. The `onKeyDown` handler lives on the `<div role="radiogroup">` wrapper:

```
ArrowRight / ArrowDown → select next option (wraps: grail → owned); call onStatusChange; move DOM focus
ArrowLeft / ArrowUp   → select previous option (wraps: owned → grail); call onStatusChange; move DOM focus
Home                  → select first option (owned); call onStatusChange; move DOM focus
End                   → last option (grail); call onStatusChange; move DOM focus
Space / Enter         → activate focused option (already selected; no-op if current)
Tab                   → moves PAST the entire group (only tabIndex=0 option is in tab order — roving tabindex)
```

The executor MUST implement `onKeyDown` in Phase 68. The `aria-checked` + `tabIndex` attributes are non-negotiable; the keyboard handler is the polish layer CONTEXT D-04 marks as planner-discretion, but this spec locks it IN for Phase 68 so the component ships complete.

**Why:** The radiogroup ships dormant (unmounted until Phase 70). Shipping arrow-key nav in Phase 68 avoids a Phase 70 follow-up task, and the keyboard handler is 10 lines of inline logic.

### Reference input — inline edit

```
<Input id="confirm-reference" value={reference ?? ''} onChange={(e) => onReferenceChange(e.target.value)} />
```

- Fires `onReferenceChange(string)` on every keystroke.
- D-08 invariant: on catalog-bound rows, Phase 67 D-10 overrides this value server-side. The input stays enabled — the user CAN type; the Server Action ignores it for identity fields when `catalogId` is supplied. Document in JSDoc on the prop (CONTEXT D-08 requirement).

### Year input — blank-to-undefined

```
<Input id="confirm-year" type="number" value={productionYear ?? ''} 
       onChange={(e) => onProductionYearChange(e.target.value ? Number(e.target.value) : undefined)} />
```

- Blank field fires `onProductionYearChange(undefined)`.
- Typed number fires `onProductionYearChange(Number(value))`.
- No min/max — Zod on Server Action is single source of truth (CONTEXT D-07).

### Price input — blank-to-undefined

```
<Input id="confirm-price" type="number" value={price ?? ''} 
       onChange={(e) => onPriceChange(e.target.value ? Number(e.target.value) : undefined)}
       placeholder="$" />
```

Mirrors WatchForm.tsx:408-413 verbatim.

## Accessibility Contract

Every ARIA attribute must be present in the rendered DOM. This is independently assertable by test case (o) and a11y scanners.

### Root wrapper

```
<div className="space-y-6" aria-live="polite">
```

- `aria-live="polite"` — announces CTA label changes and price label changes without interrupting speech. Mirrors VerdictStep.tsx:60.

### Cover photo block

```
<Image src={coverUrl} alt={brandModel} ... />
```

- `alt` must NEVER be empty. Guard: `const brandModel = [brand, model].filter(Boolean).join(' ') || 'Watch'`. The `|| 'Watch'` fallback prevents `alt=""` when both brand and model are empty strings. Mirrors VerdictStep.tsx:57.

```
<WatchIcon className="h-16 w-16 text-muted-foreground/40" aria-hidden />
```

- `aria-hidden` on the placeholder — it is purely decorative when neither image URL is available. The Card's adjacent identity text already describes the watch; the icon adds no information.

### Status radiogroup

```
<div role="radiogroup" aria-label="Watch status" id="confirm-status-group" className="flex gap-2">
  <Button role="radio" aria-checked={status === 'owned'}   tabIndex={status === 'owned'   ? 0 : -1} ...>Owned</Button>
  <Button role="radio" aria-checked={status === 'wishlist'} tabIndex={status === 'wishlist' ? 0 : -1} ...>Wishlist</Button>
  <Button role="radio" aria-checked={status === 'grail'}   tabIndex={status === 'grail'   ? 0 : -1} ...>
    <span className="inline-flex items-center gap-1.5">
      <Star className="size-4" aria-hidden />
      Grail
    </span>
  </Button>
</div>
```

| Attribute | Where | Value | Rule |
|-----------|-------|-------|------|
| `role="radiogroup"` | wrapper div | static | Marks the group as a radio group |
| `aria-label="Watch status"` | wrapper div | static | Labels the group for screen readers |
| `id="confirm-status-group"` | wrapper div | static | Collision-safe ID (avoids WatchForm.tsx `id="status"`) |
| `role="radio"` | each Button | static | Overrides button semantics with radio semantics |
| `aria-checked={status === value}` | each Button | dynamic boolean | Explicitly `true` or `false` — never `undefined` |
| `tabIndex={status === value ? 0 : -1}` | each Button | dynamic | Roving tabindex — only one stop per Tab key press |
| `type="button"` | each Button | static | Prevents accidental form submit if ConfirmStep ever wraps in a form |
| `aria-hidden` | `<Star>` icon | static | Decorative marker — the "Grail" text carries the meaning |

**Roving tabindex rule:** At any moment, exactly one Button in the radiogroup has `tabIndex={0}`. The other two have `tabIndex={-1}`. Arrow keys move both the selection AND the DOM focus. This is the WAI-ARIA 1.2 Radio Group pattern — non-negotiable.

### Field labels

```
<Label htmlFor="confirm-reference">Reference</Label>   → <Input id="confirm-reference" ... />
<Label htmlFor="confirm-year">Year</Label>              → <Input id="confirm-year" ... />
<Label htmlFor="confirm-price">Price paid / Target price</Label> → <Input id="confirm-price" ... />
```

- All `htmlFor` ↔ `id` pairs must match exactly. IDs use `confirm-` prefix to avoid WatchForm collisions (CONTEXT Claude's Discretion).

### Action buttons

```
<Button type="button" variant="ghost" onClick={onEditDetails} disabled={pending}>Edit details</Button>
<Button type="button" variant="ghost" onClick={onStartOver}   disabled={pending}>Start over</Button>
<Button type="button"                 onClick={onPrimary}     disabled={pending}>...</Button>
```

- `disabled={pending}` propagates `pointer-events-none opacity-50` via Button CVA.

### Loader2 spinner


- `aria-hidden="true"` (string, not boolean) — matches VerdictStep.tsx:112 exact form. The adjacent "Saving..." text provides the accessible label.

## Responsive Behavior

All breakpoints use the Tailwind default `sm:` = 640px. Mobile is the primary surface.

| Section | Mobile (<640px) | Desktop (≥640px) |
|---------|-----------------|------------------|
| Root wrapper | `space-y-6` vertical stack | Same — no layout shift |
| Cover photo + identity | Row (`flex items-start gap-4`) | Same — horizontal is fine at all widths |
| Reference + year inputs | Single column (`grid-cols-1`) | 2 columns (`sm:grid-cols-2`) |
| Status radiogroup | Row of 3 buttons (`flex gap-2`), wraps if needed | Same — 3 buttons fit at all widths |
| Price field | Full-width | Full-width |
| Ghost button row | Stacked column (`flex-col gap-2`) | Side by side (`sm:flex-row sm:gap-3`), each `sm:flex-1` |
| Primary CTA | Full-width (`w-full`) | Full-width (`w-full`) — intentional, not `sm:flex-1` |

**CSS chain assertion — ghost button row responsive:**
```
<div className="flex flex-col gap-2 sm:flex-row sm:gap-3">
  <Button className="w-full sm:flex-1">Edit details</Button>
  <Button className="w-full sm:flex-1">Start over</Button>
</div>
```
On mobile: `flex-col` → buttons stack; `gap-2` → 8px vertical gap; `w-full` → each button 100% width.
On desktop: `sm:flex-row` → buttons side by side; `sm:gap-3` → 12px horizontal gap; `sm:flex-1` → equal width distribution.

**CSS chain assertion — primary CTA:**
```
<Button className="w-full">...</Button>
```
Always 100% width. No `sm:flex-1` — the primary action should be full-width on desktop too for visual emphasis.

**CSS chain assertion — input grid:**
```
<div className="grid grid-cols-1 gap-3 sm:grid-cols-2">
```
On mobile: single column. On desktop: 2 columns, each input takes 50% minus gap. Gap is `gap-3` (12px) — lighter than WatchForm's `gap-6` (24px) appropriate for inline-edit context.

**Touch target compliance:** Radiogroup buttons have `min-h-[44px]` — this overrides the Button default `h-8` (32px). On mobile, users must be able to tap each option. `min-h` is additive with padding; the button won't collapse below 44px vertical.

### A1 — Selected radiogroup button

```

Light mode: --primary = oklch(0.22 0.005 75) → deep charcoal border; bg = charcoal at 10% opacity
Verification: render with status='owned', assert .border-primary on the owned button; assert non-owned buttons lack .border-primary
```

