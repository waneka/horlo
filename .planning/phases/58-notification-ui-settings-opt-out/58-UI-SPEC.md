## Component Inventory

All components below are EXISTING — Phase 58 extends them; it does not create new ones.

### NotificationRow (extended — primary edit target)

**Current state:** Renders `follow` and `watch_overlap` only. Returns `null` for all other types (B-8 guard, line 52).

**Phase 58 contract:** Extend to render four new types. Visual layout is identical to existing rows.

**Row structure (all types):**
```
[AvatarDisplay 40px] [flex-1 min-w-0 text-sm text-foreground]
                       {copy ReactNode} · {timeLabel muted}
                       {commentPreview — comment types only, see below}
```

**Outer row classes (unchanged):**
```
group relative flex items-center gap-3 min-h-12 bg-card px-4 py-2
transition-colors hover:bg-muted/40 focus-within:bg-muted/40
focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring
cursor-pointer
```
Plus `border-l-2 border-l-accent` when `isUnread`.

**actorClass pattern (unchanged):**
- Unread: `font-semibold text-foreground`
- Read: `font-normal text-foreground`

**Render strings (D-01, locked):**

| Type | Single actor | Grouped (actorCount > 1) |
|------|-------------|--------------------------|
| `watch_like` | `{actor} liked your {model}` | `{actor} + {N-1} others liked your {model}` |
| `wear_like` | `{actor} liked your {model} wear` | `{actor} + {N-1} others liked your {model} wear` |
| `watch_comment` | `{actor} commented on your {model}` | (never grouped — D-05) |
| `wear_comment` | `{actor} commented on your {model} wear` | (never grouped — D-05) |

Notes:
- `{actor}` = `actorDisplayName ?? actorUsername ?? 'Someone'` — identical to existing `follow`/`watch_overlap` rows
- `{model}` = `payload.watch_model` (field name is load-bearing; matches `types.ts` payload shapes)
- Actor name is wrapped in `<span className={actorClass}>` — bold when unread, normal when read
- `{model}` is always wrapped in `<span className="font-semibold text-foreground">` regardless of read state
- Grouped phrasing uses `+` NOT `and` — matches the live `watch_overlap` row at `NotificationRow.tsx:162`

**Comment preview second line (D-02):**

Applies to `watch_comment` and `wear_comment` only. Like rows (`watch_like`, `wear_like`) have NO second line.

```tsx
{commentPreview && (
  <p className="text-xs text-muted-foreground line-clamp-2 mt-0.5">{commentPreview}</p>
)}
```

Where `commentPreview = (row.payload as { comment_preview?: string })?.comment_preview ?? null`.

The `<p>` is a sibling of the `{copy}` node inside the same `<div className="flex-1 min-w-0 text-sm text-foreground">` container.

`line-clamp-2` is native in Tailwind 4 (no plugin required).

**Deep-links (D-07):**

| Type | href |
|------|------|
| `watch_like` | `/watch/{payload.watch_id}` |
| `wear_like` | `/wear/{payload.wear_event_id}` |
| `watch_comment` | `/watch/{payload.watch_id}` |
| `wear_comment` | `/wear/{payload.wear_event_id}` |

No scroll-anchor suffix. `/watch/[id]` and `/wear/[wearEventId]` routes confirmed to exist.

**Interaction (unchanged pattern):**
- Whole row is tappable: `onClick={activate}`, `onKeyDown` handles Enter/Space
- Tapping fires `markNotificationRead` optimistically, then `router.push(href)`
- `role="link"`, `tabIndex={0}`, `aria-label="{actorName} notification"`

**B-8 guard (D-08):**
- Guard extends to allow the four new types through the render path
- Unknown future types still return `null`
- Guard is placed after all hooks (React Rules of Hooks preserved)

### NotificationsInbox (extended — collapse logic)

**Current state:** `collapseWatchOverlaps` groups `watch_overlap` rows per (brand_normalized, model_normalized, UTC-day). All other types pass through to `nonOverlap` uncollapsed.

**Phase 58 contract:** Extend the collapse to also group `watch_like` and `wear_like` rows.

**Like grouping key format:**
```
`${row.type}|${targetId}|${toUtcDayKey(row.createdAt)}`
// watch_like: targetId = payload.watch_id
// wear_like:  targetId = payload.wear_event_id
```

Including `row.type` in the key prevents `watch_like` rows from merging with `watch_overlap` rows that share the same `watch_id`.

**Grouping behavior (D-04, identical to watch_overlap):**
- Most-recent actor wins (rows arrive newest-first from DAL; first in group array is newest)
- `actorCount` = group size
- Newest-first re-sort after merge (existing `collapsed.sort(...)` at lines 103-108 is reused)
- Group's displayed row `id` is what `markNotificationRead` targets on tap (D-06)

**Comments (D-05):** `watch_comment` and `wear_comment` pass through to `nonOverlap` unchanged — never grouped.

**`bucketByDay` (unchanged):** Today / Yesterday / Earlier bucketing is unmodified. No new bucket labels.

### NotificationsSection (extended — settings toggles)

**Current state:** Title "Email notifications", two `PrivacyToggleRow` instances: New Followers + Watch Overlaps.

**Phase 58 contract:**

Title rename: `"Email notifications"` → `"Notifications"` (D-09)

Four `PrivacyToggleRow` instances in order:

| Order | Label | Description | field |
|-------|-------|-------------|-------|
| 1 | New Followers | "Get notified when someone starts following you." | `notifyOnFollow` |
| 2 | Watch Overlaps | "Get notified when another collector owns a watch you own." | `notifyOnWatchOverlap` |
| 3 | Likes | "Get notified when someone likes your watches or wear posts." | `notifyOnLike` |
| 4 | Comments | "Get notified when someone comments on your watches or wear posts." | `notifyOnComment` |

Copy for rows 1-2 is unchanged from existing. Copy for rows 3-4 is verbatim from D-10.

**Props widening (D-11):** `Pick<ProfileSettings, 'notifyOnFollow' | 'notifyOnWatchOverlap'>` → also include `'notifyOnLike' | 'notifyOnComment'`.

**Layout (unchanged):** `<div className="divide-y divide-border">` wrapper around all four toggle rows.

### PrivacyToggleRow (unchanged — no modifications)
