<objective>
Build the read + write DAL for surfaced wear pics: a NEW `getPublicWearPicsForWatch(watchId)` that unions public, not-hidden wear pics into the carousel (D-01/D-02), and `hideWearPic`/`unhideWearPic` DAL writes plus their `.strict()`-validated, ownership-re-checking server actions (D-08/D-11/D-12).

The hide/unhide actions give the owner the per-pic control without touching `visibility` (D-11).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → server action | hide/unhide request from the browser; untrusted `{wearEventId, watchId}` payload |
| server action → DAL → DB | service-role Drizzle write; ownership must be enforced here, not by RLS |
| DAL union → any viewer | getPublicWearPicsForWatch result is sent to owner AND non-owner clients |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-62-04 | Tampering / IDOR (HIGH) | hideWearPicAction | mitigate | Server action re-checks ownership via `watchDAL.getWatchById(user.id, watchId)` AND the DAL UPDATE WHERE scopes `watch_id IN (SELECT id FROM watches WHERE user_id = ?)` — a non-owner cannot hide another user's wear pic. Returns generic 'Watch not found' for cross-user IDs. |
| T-62-05 | Tampering / Mass-assignment (MEDIUM) | hide/unhide Zod schema | mitigate | `.strict()` schema with two UUID fields rejects any extra key (analog deleteWatchPhotoAction). No arbitrary column writable. |
| T-62-06 | Information Disclosure / Visibility leak (HIGH) | getPublicWearPicsForWatch WHERE clause | mitigate | Service-role Drizzle query filters `visibility='public' AND hidden_from_detail=false` at read time (WPIC-05/D-18) — RLS-subquery-caller is NOT relied on (Phase 53 lesson). Client never receives a non-public or hidden row. |
| T-62-07 | Information Disclosure / Visibility coupling (HIGH) | hide DAL .set() | mitigate | hide/unhide write ONLY `hidden_from_detail`; never `visibility` (D-11 constraint a). "Hide from detail" ≠ "make private"; the Wears tab + Home rail are untouched. Asserted by the no-`.set(visibility)` acceptance check. |
| T-62-08 | Information Disclosure / Rail regression (HIGH) | getWearRailForViewer | mitigate | New functions are separate; rail guardrail test (wearRail.test.ts) re-run in this plan stays green (WPIC-04 / D-17). |
