## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anon role → new tables | Supabase auto-grants direct table privileges to `anon` at CREATE TABLE; untrusted unauthenticated access crosses here. |
| authenticated client → comments RLS | Any authenticated user can attempt to read/write any comment row; the RLS gate decides what they actually see/insert. |
| watch/wear delete → interaction rows | Parent deletion must propagate to child interaction rows or orphans accumulate. |

|-----------|----------|-----------|-------------|-----------------|
| T-53-01 | Information Disclosure | anon SELECT on watch_likes / wear_likes / comments | mitigate | `REVOKE ALL ON <table> FROM anon` + `FROM public`; all RLS policies `TO authenticated`; in-migration `NOT has_table_privilege('anon', ...)` assertion (SEC-01). Authored in Task 2. |
