<objective>
Polish the two shared comment leaf components — `CommentCompose` (SC-2) and `CommentItem`
(SC-3 + D-04) — so they match the approved 57.1 UI-SPEC. These are presentational/layout-only
edits to already-shipped components; no props change, no data flow changes, no shared-state
touch. SC-2 polishes the CMNT-01 compose surface; SC-3 polishes the CMNT-08 comment-row surface.

Purpose: SC-2 puts the Post button inline with the textarea so the compose row reads as a single
unit. SC-3 + D-04 move the per-comment edit/delete controls (and the delete confirm) onto the
comment's meta line so they no longer "hang below" the body — the exact UAT complaint this phase
fixes. Because both components are shared, the change propagates automatically to the wears-lane
bottom-sheet, `/wear/[wearEventId]`, and `/watch/[id]` — no per-host wiring here.

Output: Updated `CommentCompose.tsx` and `CommentItem.tsx` with the UI-SPEC layout classes.
</objective>
