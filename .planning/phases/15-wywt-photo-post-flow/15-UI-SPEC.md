## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, character-counter gap, inline label padding |
| sm | 8px | Watch-card internal padding, button gaps, segmented control item gaps |
| md | 16px | Default element spacing, dialog section padding, form field spacing |
| lg | 24px | Dialog header/body separation, section spacing |
| xl | 32px | Step 2 photo zone height padding |
| 2xl | 48px | Hero image min-height baseline on detail page |
| 3xl | 64px | Detail page desktop max-width column padding |

Exceptions:
- Touch targets (camera capture button, remove-photo X) minimum 44px per iOS HIG — enforced with `min-h-11` / `size-11` classes matching existing nav pattern.
- Camera overlay SVG is unitless percentage-based within the `viewBox` — not driven by the spacing scale.
- Bottom-nav clearance `pb-[calc(4rem+env(safe-area-inset-bottom))]` is carried from Phase 14 layout; `/wear/[wearEventId]` inherits it from the root layout.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Watch list items, note textarea content, metadata text on detail page |
| Label | 12px (text-xs) | 600 (semibold) | 1.333 | Character counter (0/200), segmented control button labels, "Change" link, bottom-nav Wear label |
| Heading | 16px (text-base) | 600 (semibold) | 1.25 | Dialog title ("Log a wear"), Step 2 form section labels, detail-page watch brand |
| Display | 20px (text-xl) | 400 (regular — Instrument Serif) | 1.2 | Collector username on wear detail page hero (matches profile page convention) |

Source: IBM Plex Sans is the established sans font. All existing dialog components (WatchPickerDialog) use text-sm at weight 400 for list copy and font-semibold for emphasis. Instrument Serif is reserved for the display role on high-presence surfaces (collector name on wear detail page).

Character counter: `0/200` — renders at 12px / weight 400 in `text-muted-foreground` until 200 chars; at 200 chars switches to `text-destructive` at weight 600. Blocked at 200 via `maxLength` attribute — counter never shows 201+.

## Color

All values are CSS custom properties resolved from `globals.css`. Both light and dark modes are defined. Executor MUST use the token name, not a raw hex/oklch value.

| Role | CSS Token | Light value | Dark value | Usage |
|------|-----------|-------------|------------|-------|
| Dominant (60%) | --background | oklch(0.985 0.003 75) | oklch(0.14 0.005 75) | Dialog backdrop, page background, photo zone background |
| Secondary (30%) | --card / --muted | oklch(1 0 0) / oklch(0.95 0.005 75) | oklch(0.19 0.005 75) / oklch(0.24 0.005 75) | Step 2 compact watch card, disabled-watch rows in picker, photo zone dashed border fill |
| Accent (10%) | --accent | oklch(0.76 0.12 75) | oklch(0.78 0.13 75) | Reserved — see accent list below |
| Destructive | --destructive | oklch(0.55 0.22 27) | oklch(0.65 0.2 27) | Character counter at 200, inline error alerts (role="alert"), disabled-watch "Worn today" label |

Accent (`--accent` / `text-accent`) reserved ONLY for:
1. Active segmented control button fill (visibility selector — Private / Followers / Public active state)
2. "Change" link in Step 2 dialog header (watch swap affordance)
3. BottomNav Wear circle fill (`bg-accent`) and its label (`text-accent`) — inherited from Phase 14; not modified in Phase 15
4. Active tab indicator in BottomNav — inherited from Phase 14; not modified in Phase 15

Accent is NOT used for: the photo zone dashed border, the note textarea, the submit button (which uses `bg-primary`), the "Logging…" spinner, or the camera overlay SVG stroke (which uses `rgba(255,255,255,0.85)` for contrast against any viewfinder content).

Photo zone dashed border: `border-2 border-dashed border-border` — uses the neutral `--border` token, not accent.

Worn-today disabled watch rows in picker: `opacity-50` + `aria-disabled="true"` + "Worn today" micro-label in `text-muted-foreground`. No destructive color — disabled is informational, not an error.

## Interaction Contracts

### WywtPostDialog — two-step state machine

```
Step 1: PICKER
  - Renders WatchPickerDialog with onWatchSelected + wornTodayIds props
  - Disabled watches show opacity-50 + "Worn today" micro-label + aria-disabled="true"
  - Selecting a watch advances to COMPOSE; no markAsWorn call fires
  - Cancel closes dialog completely, resets all state

Step 2: COMPOSE
  - Header: compact watch thumbnail (watch.imageUrl or muted placeholder) + brand/model text
    + "Change" link (text-accent, underline) → returns to PICKER, preserves note/visibility/photo blob
  - Photo section (pre-capture): dashed border zone, two side-by-side buttons:
      "Take wrist shot" (Camera icon) — expands zone inline to video preview; no sub-dialog
      "Upload photo" (ImageIcon) — triggers native file picker
  - Photo section (during camera): video element fills the zone; WristOverlaySvg overlay absolute-positioned
    over video; "Capture" button below (primary, 44px min tap target)
  - Photo section (after capture or upload): preview image fills zone at full width; top-right X button
    (44px tap target) removes photo and returns to pre-capture state; text link below reads
    "Retake" (camera path) or "Choose another" (upload path)
  - Note: Textarea, maxLength=200, placeholder="Add a note…"; character counter bottom-right "0/200"
  - Visibility: VisibilitySegmentedControl (see below)
  - Submit: "Log wear" button (primary) / pending state "Logging…" with inline spinner; disabled during submission
  - Cancel and Change buttons disabled during submission
  - Error: role="alert" inline banner, destructive color, below the form fields
  - Success: close dialog, fire Sonner toast "Wear logged", no forced navigation
```

### VisibilitySegmentedControl

```
Three-button inline toggle group (iOS-style segmented control):
  Private | Followers | Public
Default: Public (locked per STATE.md D-1 — user override of researcher recommendation)
Active button: bg-accent text-accent-foreground, rounded within the group container
Inactive button: bg-transparent text-foreground
Icons (lucide): Lock (Private) / Users (Followers) / Globe2 (Public) — 14px, inline-start of label

Sub-label row below the control (mandatory — Followers tier is new in v3.0):
  Private selected   → "Only you"
  Followers selected → "Followers — people who follow you"
  Public selected    → "Anyone on Horlo"

Sub-label typography: 12px / weight 400 / text-muted-foreground
```

### CameraCaptureView + WristOverlaySvg

```
- Zone expands inline inside ComposeStep; no route change, no modal within modal
- video element: object-cover, rounded-md, width 100%
- WristOverlaySvg: absolute inset-0, pointer-events-none, preserveAspectRatio="xMidYMid meet"
  Geometry (from CONTEXT.md D-08 + specifics):
    - viewBox: "0 0 100 100" (percentage-based)
    - Arm lines: horizontal strokes at y=38 and y=62, x1=0 x2=100
    - Outer circle: cx=50 cy=50 r=22
    - Inner circle: cx=50 cy=50 r=17
    - Hour hand: line from center (50,50) to ~(38,27) — 10 o'clock, -60° from top
    - Minute hand: line from center (50,50) to ~(62,27) — 2 o'clock, +60° from top
    - Crown: small rect at (72,49) width=4 height=3 (3 o'clock on outer circle edge)
    - Nothing else — no hour markers, no lugs, no strap
  Stroke: strokeWidth="1.5" stroke="rgba(255,255,255,0.85)" fill="none" strokeLinecap="round"
- Capture button: "Capture" label, primary variant, min-h-11, below the video
- getUserMedia must be the FIRST await on iOS gesture — no awaits before it (Pitfall D-1)
- MediaStream tracks stopped on component unmount (Pitfall D-2)
- Permission denied: inline error "Camera access denied — use Upload photo instead" with
  the destructive color; camera zone collapses back to pre-capture chooser state (Pitfall D-3)
```

### EXIF strip + resize pipeline

```
- Applies on BOTH camera and upload paths (Pitfall E-4)
- Shared helper: src/lib/exif/strip.ts
- Output: canvas.toBlob('image/jpeg', 0.85) — max 1080px on longest dimension, target < 500KB
- HEIC: heic2any lazy-loaded in Web Worker (src/lib/exif/heic-worker.ts); non-HEIC skips worker
- EXIF orientation: use createImageBitmap() (auto-corrects EXIF orientation on iOS Safari 15+)
  — researcher resolution to STATE.md open flag; exifr not required
```

### /wear/[wearEventId] detail page

```
Layout — mobile-first:
  Hero: full viewport-width image, aspect-ratio 4/5 (portrait — matches wrist-shot composition)
    - photo_url present: signed URL per-request (Pitfall F-2; never inside 'use cache')
    - photo_url null + watch.imageUrl present: watch.imageUrl (no signed URL needed — same CDN pattern)
    - both null: muted bg-muted div with brand/model centered in text-muted-foreground at 14px/weight 600
  Below hero (stacked, gap-4, px-4 pb-6):
    - Collector row: Avatar (32px) + username linked to /u/[username] at 14px/semibold + relative time at 12px/muted
    - Watch row: watch thumbnail (40×40, rounded-md) + brand (14px/semibold) + model (14px/regular/muted) — display only, no link
    - Note: 14px/regular, text-foreground, renders only when note is non-null
  Desktop: image caps at max-w-[600px], centered column, same metadata stack below

Gating: uniform notFound() for missing-or-private — no response differential (Pitfall from Phase 8 precedent)
Signed URL TTL: 60 minutes (Claude's Discretion default)
```

### Sonner ThemedToaster mount

```
- Mounted in src/app/layout.tsx ABOVE the Suspense wrapper that wraps Header + children (Pitfall H-1)
- Bound to Horlo's custom ThemeProvider — NOT npx shadcn add sonner scaffold (Pitfall H-3)
- toast.success('Wear logged') fired from WywtPostDialog client handler after Server Action returns success
- toast() never called from Server Actions directly (Pitfall H-2)
```

## Accessibility Contract

- `aria-disabled="true"` on worn-today watch rows in picker (not just visual opacity)
- `role="alert"` on all inline error states (matches PreferencesClient DEBT-01 pattern)
- `aria-label="Remove photo"` on the X button in photo preview
- `aria-label="Capture photo"` on the camera capture button
- `<video>` element has `aria-label="Camera preview"` (screen readers should know what the element is)
- WristOverlaySvg has `aria-hidden="true"` — it is decorative
- `<img>` on detail page has `alt="{username} wearing {brand} {model}"` (or "Watch on wrist" when username/watch unknown)
- Segmented control: use `role="group"` container with `aria-label="Post visibility"`, each button has descriptive `aria-label` ("Private — only you", "Followers only", "Public — anyone on Horlo")
- `aria-pressed` on active segmented control button
