<objective>
Sign owner-photo cover storage paths at every grid/rail RSC so that a watch's cover photo (and any drag-reorder cover change from Plan 02) actually renders as the card thumbnail across the app — completing PHOTO-05's "reordering sets the cover/thumbnail across grids."

Purpose: Phase 60 made `Watch.imageUrl` return a RAW Supabase storage path whenever an owner photo exists (DAL kept admin-client-free by design). Every grid/rail card consumes `watch.imageUrl` through `getSafeImageUrl()`, which returns null for non-https paths — so without signing, every owner-photo watch shows a blank/placeholder thumbnail and SC3 ("drag-to-first makes it the card thumbnail across grids") is unobservable. Plan 02 proved the signing pattern on the detail page; this plan centralizes it into a reusable batch helper and applies it at the four cover-rendering RSCs. This is in-scope (it is load-bearing for a phase success criterion), not deferrable.
Output: new `signCoverUrls` helper + unit test; signing wired into home, profile tab, profile-shell-resolver, and search RSCs.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RSC → signed URL | Grid/rail RSCs sign storage paths into time-boxed read URLs before passing to client card components |
| server helper → Supabase Storage | `signCoverUrls` uses the server client to mint signed URLs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-61-16 | Information Disclosure | a viewer's grid signs and renders another user's cover storage path | mitigate | RSCs only sign covers from watch lists the viewer is already authorized to see (the DAL/search already scope by viewer); signing converts a path the viewer can legitimately view into a 60-min read URL — no new exposure beyond what the grid already shows. Folder scoping `{userId}/...` is RLS-enforced at write. |
| T-61-17 | Information Disclosure | raw storage path string leaks into client markup if signing skipped | mitigate | `getSafeImageUrl` returns null for raw paths, so an unsigned path renders as a placeholder (no path string in `src`). Signing replaces it with an https url; either way the raw path does not appear in client HTML. |
| T-61-18 | Information Disclosure | signed cover URL TTL exposure | accept | 60-min TTL matches the detail-page + wear-photos precedent; routes are dynamic (no ISR past TTL). Acceptable for a personal auth-gated app (consistent with T-61-07). |
| T-61-19 | Spoofing | client bundle gaining the service-role storage client via signCoverUrls import | mitigate | `signCoverUrls` is `server-only`; importing it into a client component fails the build. |
</threat_model>
