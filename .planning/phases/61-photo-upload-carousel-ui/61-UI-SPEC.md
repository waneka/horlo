## Spacing Scale

All spacing uses multiples of 4. Existing project uses Tailwind 4 utility classes inline (no CSS modules).

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Cover badge padding (`px-1 py-0.5`), × badge offset from thumbnail corner |
| sm | 8px | Filmstrip gap between thumbnails (`gap-2`), carousel arrow button inset |
| md | 16px | Carousel container horizontal padding, dropzone inner padding |
| lg | 24px | Spacing between carousel and filmstrip, spacing between filmstrip and upload action row |
| xl | 32px | Top-level section gap matching existing `space-y-8` (32px) in WatchDetail |
| 2xl | 48px | Add-watch photo step vertical padding (full-screen step) |
| 3xl | 64px | Minimum dropzone height in add-watch flow |

Exceptions:
- Filmstrip thumbnails: 64px × 64px (`w-16 h-16`) — deliberately non-standard to fit ≤10 thumbs in a horizontal strip
- Touch targets for all interactive controls: minimum 44px (`min-h-[44px]`) — matching existing WatchDetail pattern (e.g., line 170 and line 198 in WatchDetail.tsx)
- Carousel arrow buttons: 36px × 36px (`size-9`) minimum, elevated to 44px touch target via padding on mobile
- Delete × badge on thumbnail: 20px × 20px (`size-5`) — centered over thumbnail corner; hit target expanded with padding to 32px

- "Skip for now": `text-sm` (14px / 400) — muted-foreground, no underline — visually secondary to the primary CTA

## State Inventory

Every distinct visual state that the executor must implement:

### Carousel States

| State | Trigger | Visual Contract |
|-------|---------|-----------------|
| **Empty / Catalog fallback** | Zero owner uploads, catalog `imageUrl` exists | Single slide showing catalog image via signed URL from `Watch.imageUrl` fallback. No filmstrip (0 owner photos). No edit controls in filmstrip (nothing to manage). Owner sees "+Add photos" affordance below slide. |
| **Empty / No image** | Zero owner uploads AND no catalog `imageUrl` | Single placeholder slide: `bg-muted` with centered `<WatchIcon className="h-16 w-16 text-muted-foreground/40">`. Same placeholder as current WatchDetail fallback. No arrows (single slide). |
| **Single owner photo** | Exactly 1 owner photo | Single slide, signed URL. No prev/next arrows (canScrollPrev = false, canScrollNext = false — arrows hidden or `disabled` + `opacity-50`). Position indicator hidden (1 / 1 is redundant). Filmstrip shows 1 thumbnail with Cover badge. |
| **Multiple owner photos** | 2–10 owner photos | Swipeable + arrow-navigable. Position indicator: `"{current} / {total}"` in `text-sm text-muted-foreground`. Prev arrow disabled at index 0; Next arrow disabled at last index. Filmstrip shows all thumbnails. |
| **Upload in progress** | Owner adding photos | Active carousel slides show existing photos normally. New photos appear in filmstrip as they complete (embla auto-reinits via MutationObserver). Per-file progress shown in filmstrip/step (see Upload States). |

### Filmstrip States

| State | Visual Contract |
|-------|-----------------|
| **View mode (default, all users)** | Horizontal scrollable strip of 64×64px thumbnails. Tap/click a thumbnail → `emblaApi.scrollTo(index)`. First thumbnail always has "Cover" badge (`bg-background/80` text overlay, `text-xs font-medium`). No × badges, no drag handles, no +Add tile. |
| **Edit mode (owner only, toggle ON)** | Each thumbnail adds: delete × badge (top-right, `bg-destructive`, 20×20px), drag handle hint (cursor changes to `grab`). +Add tile appended at end (dashed border, `bg-muted`, Camera icon centered). Embla `draggable: false` (no carousel swipe conflict). Cover badge persists on position [0] and moves with drag. |
| **Drag active** | Source thumbnail: `opacity-30`. Drop indicator: vertical gap-line between target positions (left/right). DragOverlay renders a full-opacity clone at pointer. `touchAction: 'manipulation'` on each `SortablePhotoThumb`. |
| **At cap (10 photos)** | Edit mode: +Add tile shows `opacity-50 cursor-not-allowed`. View mode: no +Add tile visible (tile is edit-mode-only). Cap message appears below filmstrip: "10 photos — at the limit." (`text-sm text-muted-foreground`) |

### Upload States (per file)

Each selected file goes through these states sequentially:

| State | Visual Contract |
|-------|-----------------|
| **Queued** | Thumbnail placeholder in filmstrip: `bg-muted`, animated pulse (`animate-pulse`). `text-xs text-muted-foreground` below: "Processing…" |
| **Uploading** | Thumbnail placeholder with a thin progress ring or indeterminate spinner overlaid (auto-decided: indeterminate spinner centered, `size-4`, `text-muted-foreground`). No percentage shown (upload is fast for ≤1080px JPEG). |
| **Complete** | Thumbnail slot fills with the actual image. "Processing…" label gone. Slide added to carousel (embla auto-reinits). |
| **Failed** | Thumbnail shows destructive red overlay with `X` icon (`text-destructive-foreground`). `text-xs text-destructive` below: "Failed — tap to retry." Retry by re-tapping the failed thumbnail (re-processes same file). |
| **Batch overflow (cap exceeded)** | Files beyond the remaining cap are rejected immediately before processing. Toast: "Added {n} photo{s}. {m} skipped — you've reached the 10-photo limit." (`toast.warning`, not `toast.error`). |

### Delete State

Per-photo delete uses **immediate delete with undo toast** (lighter than whole-watch Dialog confirm — per Claude's Discretion):
- Owner taps × badge → optimistic: remove thumbnail from filmstrip, remove slide from carousel (embla reinits), show `toast` with undo: `"Photo deleted — Undo"` (5-second window).
- Undo tapped → restore thumbnail + slide optimistically, cancel server action if possible (or re-add).
- If undo window expires → `deleteWatchPhotoAction` fires. On server error → revert + `toast.error("Couldn't delete photo.")`.
- No Dialog. No confirmation prompt.

### Add-Watch Photo Step States (in AddWatchFlow)

| State | Visual Contract |
|-------|-----------------|
| **Entry (no photos yet)** | Full-width dropzone card (desktop + mobile). Heading: `text-lg font-semibold` "Add your photos". Subheading: `text-sm text-muted-foreground` "Show how it looks in person." Dropzone body: Camera icon + `text-sm` "Drop photos here or tap to choose". Primary CTA: `<Button variant="default">Add photos</Button>`. Secondary: `<button className="text-sm text-muted-foreground mt-2">Skip for now</button>` — no Button component, plain `<button>` styled as text link. |
| **Photos selected / uploading** | Per-file progress thumbnails in a 3-column mini-grid below the dropzone. Primary CTA changes to `<Button variant="default">Continue</Button>` once ≥1 upload completes. "Skip for now" remains visible (smaller). |
| **All uploaded** | Dropzone collapses to a smaller "Add more" link (if under cap). Primary CTA: `<Button variant="default">Continue</Button>`. |
| **At cap in add-watch step** | Dropzone hidden. "Add more" link hidden. Cap message: `text-sm text-muted-foreground` "10 photos — at the limit." Primary CTA: `<Button variant="default">Continue</Button>`. |

## Interaction Contract

### Carousel (PHOTO-03)

- **Library:** `embla-carousel-react` v8.6.0. Options: `{ loop: false, dragFree: false, draggable: !editMode }`.
- **Swipe:** embla handles natively on mobile. Disabled when edit mode is ON (`draggable: false`).
- **Arrows:** `ChevronLeft` + `ChevronRight` icons from lucide-react. Positioned absolutely inside the carousel viewport: left-center and right-center. Hidden (not just `disabled`) when there is only 1 slide. Disabled + `opacity-50` when at first/last slide (`canScrollPrev` / `canScrollNext`).
- **Filmstrip tap-to-jump:** `emblaApi?.scrollTo(index)` — instant, no animation delay. Syncs `selectedIndex` via `api.on('select', cb)`.
- **Position indicator:** `"{selectedIndex + 1} / {slides.length}"` — visible only when slides.length > 1. Centered below carousel, above filmstrip.
- **Keyboard:** Left/right arrow keys on a focused carousel element → `goToPrev()` / `goToNext()`. Managed via `onKeyDown` on the carousel wrapper div (role="region", aria-label="Watch photos").
- **Auto-reinit on DOM change:** embla v8 auto-watches via MutationObserver (`slideChanges: true` default). No manual `reInit` call on upload/delete/reorder.
- **Stale instance reset:** Edit toggle state reset on `onPointerDown` (interaction), not mount — per `project_router_cache_stale_instance` MEMORY. Carousel index managed by embla itself (resets to 0 on reInit).

### Filmstrip Drag-Reorder (PHOTO-05)

- **Library:** `@dnd-kit/core` + `@dnd-kit/sortable`. Strategy: `horizontalListSortingStrategy` (single horizontal row).
- **Sensors:** `MouseSensor({ activationConstraint: { delay: 150, tolerance: 5 } })` + `TouchSensor({ activationConstraint: { delay: 250, tolerance: 8 } })`. Mirrors Phase 27 pattern verbatim.
- **Drag handle:** `GripVertical` icon (lucide) attached via `{...attributes} {...listeners}` — not the entire thumbnail. Prevents conflict between × tap and drag start (Pitfall 3 from RESEARCH.md).
- **`touchAction: 'manipulation'`** on each `SortablePhotoThumb` outer div — required for iOS Safari.
- **Optimistic update:** `useOptimistic` + `useTransition`. On drag end → `setOptimistic(newOrder)` + call `reorderWatchPhotos` action. On failure → revert optimistic + `toast.error("Couldn't save new order.")`.
- **Cover badge moves with drag:** The "Cover" badge renders on the thumbnail at array index [0] of `optimisticIds`. As optimistic state updates, the badge moves instantly to the new first position.
- **Toast on success:** `toast.success("Order updated")` — fires after the transition completes and action succeeds.
- **SetMismatchError:** `toast.error("Photos changed in another tab. Refresh and try again.")` — optimistic state auto-reverts (no `revalidatePath` in action on error path).

### Upload Input (PHOTO-02)

- **File input:** `<input type="file" multiple accept="image/*,.heic,.heif" className="sr-only">`. Triggered programmatically from user-gesture handler (button/tile click). Value reset to `""` after each batch to allow re-selecting the same file.
- **Desktop drop zone:** `onDragOver` / `onDragLeave` / `onDrop` handlers on the dropzone container div. Drag-over adds `ring-2 ring-ring` to the container. Files extracted from `e.dataTransfer.files`.
- **Processing order:** Sequential (not parallel) to avoid `sort_order` race conditions in `addWatchPhoto`.
- **Cap check:** Before processing, compute `remaining = 10 - currentPhotoCount`. Accept `files.slice(0, remaining)`, reject `files.slice(remaining)`. Show batch-overflow toast immediately for rejected files. Do not silently drop.
- **HEIC→JPEG:** Existing `convertHeic` worker per file. One worker per file (acceptable at ≤10 scale).
- **`stripAndResize`:** Existing `src/lib/exif/strip.ts`. Reuse verbatim (EXIF strip + ≤1080px JPEG @0.85).
- **`uploadWatchPhoto`:** Client-direct to Supabase Storage (`src/lib/storage/watchPhotos.ts`).
- **`addWatchPhotoAction`:** Server action records the row. Called after successful upload.

### Delete (PHOTO-06)

- Immediate optimistic delete + undo toast pattern (see Delete State above).
- × badge uses `onClick` (not drag listener). No gesture conflict since dnd-kit listeners attach only to the drag handle icon.
- `deleteWatchPhotoAction` server action fires after undo window expires (or immediately if no undo UX — undo is the auto-decided lighter pattern).

### Edit Toggle

- `<button>` or `<Button variant="outline" size="sm">Edit photos</Button>` — positioned to the right of the filmstrip, below the carousel.
- Toggling to OFF: dnd-kit context disengages (filmstrip re-renders without `DndContext`), × badges hidden, +Add tile hidden. Embla `draggable: true` restored via `emblaApi?.reInit({ draggable: true })`.
- Toggling to ON: embla `draggable: false` via `emblaApi?.reInit({ draggable: false })`.
- Only rendered when `viewerCanEdit === true`.

## Accessibility Contract

| Concern | Implementation |
|---------|---------------|
| Carousel landmark | `<div role="region" aria-label="Watch photos">` wraps the embla viewport |
| Slide alt text | `alt="Photo {index + 1} of {total}"` for owner photos. Catalog fallback: `alt="{brand} {model}"`. Placeholder: `aria-hidden` + `role="img" aria-label="No photo yet"` |
| Arrow buttons | `aria-label="Previous photo"` / `aria-label="Next photo"`. `aria-disabled` when at boundary (not just CSS disabled). |
| Position indicator | `aria-live="polite"` — announces slide change to screen readers without interrupting |
| Filmstrip | `role="list"`. Each thumbnail: `role="listitem"`. Focusable thumbnails in view mode: `tabIndex={0}`, `onKeyDown` for Enter/Space → `scrollTo`. |
| Edit toggle | `aria-pressed={editMode}` — toggles state semantics. |
| Delete × badge | `aria-label="Delete photo {index + 1}"`. Not `aria-hidden`. |
| "Cover" badge | `aria-label="Cover photo"` on the span. |
| Drag-reorder | `aria-roledescription="Photo, grab to reorder"` on each `SortablePhotoThumb`. Instructions via `aria-describedby` pointing to a visually-hidden instructions element: "Press space to start dragging, arrow keys to move, space or enter to drop." |
| Drop zone | `role="button" tabIndex={0}` on the drop zone container. `aria-label="Upload photos — drop files here or press Enter to browse"`. `aria-disabled` when at cap. |
| Focus management | On delete: focus moves to previous thumbnail (or +Add tile if last deleted). On upload complete: no forced focus move (toast handles feedback). |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` — disable `animate-pulse` on queued thumbnails. Embla snap animation instant. Optimistic reorder no transition (`transition: none`). |
| Touch targets | All interactive controls: `min-h-[44px]` or `min-w-[44px]`. × badge expands via padding: `p-2` around the 20px icon inside a larger hit area. |
