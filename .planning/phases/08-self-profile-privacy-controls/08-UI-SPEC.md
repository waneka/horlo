## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, pill gap |
| sm | 8px | Compact element spacing, tag pill margins |
| md | 16px | Card padding, form field spacing |
| lg | 24px | Section padding, tab content top margin |
| xl | 32px | Layout horizontal padding (desktop) |
| 2xl | 48px | Profile header vertical padding |
| 3xl | 64px | Page-level top padding |

Exceptions:
- Avatar circle: 64px diameter (16px multiples; touch target safe)
- Stat card min-height: 120px (Stats tab 2-col grid)
- Calendar day cell: 48px min-height (touch target minimum per WCAG 2.5.5; 48 = 4 × 12, exceeds the 44px floor)
- Settings toggle row: 48px min-height (touch target minimum per WCAG 2.5.5; 48 = 4 × 12)

## Typography

| Role | Size | Weight | Line Height | Letter Spacing | Usage |
|------|------|--------|-------------|----------------|-------|
| Body | 14px | 400 | 1.5 | normal | Note text, descriptions, secondary copy, form labels, bio paragraph |
| Label | 12px | 400 | 1.2 | tracking-wide | Filter chip labels, badge text, section meta — `text-xs uppercase tracking-wide` |
| Heading | 20px | 600 | 1.2 | normal | Profile username, section headings, card titles |
| Display | 28px | 600 | 1.1 | normal | Stats numbers (e.g. wear count large), page-level headings |

All sizes via Tailwind: `text-xs` (12px), `text-sm` (14px), `text-xl` (20px), `text-3xl` (28px).

**Two weights only:** `font-normal` (400) for Body and Label; `font-semibold` (600) for Heading and Display. Label is differentiated from Body via size (12px vs 14px) and `uppercase tracking-wide`, not weight.

**Bio paragraph:** Use `text-sm font-normal leading-relaxed` (14px, weight 400, line-height 1.625).

## Color

The existing CSS custom properties in globals.css define the live token system. Map Figma values to those tokens.

| Role | CSS Token | Approx Hex | Usage |
|------|-----------|------------|-------|
| Dominant (60%) | `--background` | `#fefdfb` (light) / dark equiv | Page background, profile header background |
| Secondary (30%) | `--card` | `oklch(1 0 0)` (light) / `oklch(0.19...)` (dark) | Watch cards, stats cards, settings section backgrounds |
| Accent (10%) | `--accent` | `oklch(0.76 0.12 75)` ~`#8b6f47` warm brown | Reserved list below |
| Destructive | `--destructive` | `oklch(0.55 0.22 27)` ~`#c44536` | Delete Account action only |
| Muted text | `--muted-foreground` | `oklch(0.48 0.01 75)` ~`#6b5d4f` | Timestamps, secondary copy, stat labels |
| Brand text | `--foreground` | `oklch(0.18 0.01 75)` ~`#2b1f14` | Username, headings |
| Card border | `--border` | `oklch(0.9 0.005 75)` | Watch card rings; override opacity to `rgba(accent, 0.15)` for profile-specific cards |

**Accent (`--accent`) reserved for:**
- Active privacy toggle (on-state fill)
- Active filter chip (background)
- Taste tag pill background
- "Worn today" status badge on watch card
- Timeline/Calendar toggle active state
- "Log Today's Wear" CTA button
- Per-note "Public" visibility pill

**Never apply accent to:** general body text, inactive states, borders, icons, placeholder text.

**Inactive toggle / secondary pill color:** `--muted` background with `--muted-foreground` text (maps to Figma's `#d4b896` inactive tone).

Both light and dark modes are handled by the existing CSS variable system. All new components MUST use `var(--token)` references through Tailwind classes (e.g. `bg-accent`, `text-muted-foreground`) — never hardcode hex values.

## Interaction Contracts

### Profile Header

**Primary focal point:** the avatar + username block in the ProfileHeader is the dominant visual anchor; all other elements are subordinate.

- Desktop: avatar left, username + bio + stats row + taste tags right
- Mobile: avatar centered top, username centered, bio left-aligned, stats row below, taste tags below stats
- Edit mode: clicking "Edit Profile" replaces header content with `ProfileEditForm` inline (no modal)
- Avatar: circular, 64px diameter, falls back to initials on colored bg if URL fails or empty
- Follower/following counts: clickable (taps open list — Phase 9 implements the list; this phase renders the count as non-linked text)
- "+ Follow" button: rendered as disabled Button with `coming soon` tooltip — non-functional this phase per D-14
- Taste tags: up to 3 pills, read-only, no interaction

### Profile Tabs

- Use existing `TabsList` + `TabsTrigger` with `variant="line"` (underline style, not pill)
- Tab order: Collection · Wishlist · Worn · Notes · Stats
- URL-driven active state: `/u/[username]/collection`, `/u/[username]/wishlist`, etc. — NOT controlled by React state (use Next.js Link-based navigation)
- Mobile: tabs scroll horizontally if they don't fit; no truncation

### Collection Tab

- Grid: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4` (matches existing WatchGrid pattern)
- Profile watch card: extends existing WatchCard — adds "Last worn X ago" text below model name; adds "Worn today" brown accent pill or "Not worn recently" white outline pill as overlay top-left of image
- Filter chips row: horizontal scroll on mobile; chips derived dynamically from `roleTags` in collection (e.g. Sport, Dress, Dive, Pilot) — do not hardcode labels
- Search input: rightmost in filter row; `text-sm`, placeholder "Search watches..."
- "+ Add Watch" card: only rendered when viewer is owner; dashed border card at end of grid; clicking routes to existing add-watch flow

### Wishlist Tab

- Reuse ProfileWatchCard layout exactly
- Add "Target: $X,XXX" in card body when `targetPrice` is set
- Add "Notes:" truncated 2-line text when `notes` is non-empty
- No additional filter chips beyond those already on Collection tab

### Worn Tab

- View toggle pill: "Timeline" | "Calendar" — custom `ViewTogglePill` using two buttons styled as a segmented control (rounded pill container, active segment fills with `bg-accent text-accent-foreground`)
- Per-watch filter: `<select>` styled as existing Select component; "All watches" default
- "+ Log Today's Wear" CTA: `bg-accent text-accent-foreground` button, full-width on mobile, right-aligned on desktop, always visible above the content area

**Calendar view:**
- Week header row: Sun Mon Tue Wed Thu Fri Sat, `text-xs text-muted-foreground`
- Day cells: 48px min-height (WCAG 2.5.5 touch target; 48 = 4 × 12); if worn: show watch image (24px circle thumbnail) centered; if not worn: empty; if multiple watches: show first image + "+N" overlay
- Month navigation: `<` `>` chevron buttons, current month label centered, `text-sm font-medium`; previous-month button must have `aria-label="Previous month"`; next-month button must have `aria-label="Next month"`
- Current day highlight: `ring-1 ring-accent`

**Timeline view:**
- Group by date: date heading `text-sm font-medium text-foreground` (e.g. "Tuesday, April 15")
- Each event row: 40px watch thumbnail + brand/model + "worn for X hours" (if data) + time
- Empty day groups not shown

### Notes Tab

- List layout (not grid); each row full-width
- Row contents: 48px watch thumbnail left, brand + model heading, full note text (no truncation), per-note visibility pill, "X days ago" right-aligned, 3-dot menu (DropdownMenu)
- 3-dot menu trigger: icon-only button with `aria-label="Note options"`
- Visibility pill: `NoteVisibilityPill` — clicking it toggles `notes_public` in DB; "Public" = `bg-accent text-accent-foreground`; "Private" = `bg-muted text-muted-foreground`
- 3-dot menu items: "Edit Note", "Make Public" / "Make Private" (mirror pill state), separator, "Remove Note" (destructive)
- Remove Note confirmation: clicking "Remove Note" opens an AlertDialog — title "Remove this note?", body "The note for {Brand} {Model} will be deleted. The watch itself stays in your collection.", buttons "Keep Note" (outline, dismiss) + "Remove Note" (destructive variant, confirm)
- Empty state: no watches have notes yet

### Stats Tab

- 2-col grid desktop (`grid-cols-2 gap-6`), 1-col mobile
- Most Worn card: top 3 watches stacked; each row = 40px thumbnail + brand/model + wear count right-aligned
- Least Worn card: same layout as Most Worn
- Style Distribution card: `HorizontalBarChart` — each row is label left + bar + percentage right; bar fill = `bg-accent`, track = `bg-muted`; bar height = 8px; rounded full
- Role Distribution card: same as Style Distribution
- Collection Observations: full-width card below the 2-col grid; bulleted list of insight strings; no icons on bullets; `text-sm text-foreground`

### Private Profile (Locked State)

- Shown when `profile_public = false` AND viewer is not owner
- Avatar + username + bio + follower/following counts visible
- Tabs area replaced by centered block: Lucide `Lock` icon (24px, `text-muted-foreground`) + "This profile is private" (`text-sm text-muted-foreground`) + non-functional "Follow" button below
- Watch count and tab labels hidden entirely

### Settings Page (`/settings`)

- Layout: max-width 640px centered on desktop; full-width on mobile; `py-8 px-4` or `py-12 px-8`
- Section grouping via `SettingsSection` (card with section title heading + rows)
- Section order: Privacy Controls · Appearance · Notifications · Data Preferences · Account
- Section title: `text-xs font-normal text-muted-foreground uppercase tracking-wide` above each card (Label role — 12px, weight 400, uppercase, tracking-wide)

**Privacy Controls section (functional this phase):**
- 4 toggle rows: Profile Visibility, Collection, Wishlist, Worn History
- "New Note Visibility" dropdown (Select component): "Public" | "Private" default
- Toggle row anatomy: label left (`text-sm font-normal`), description below label (`text-xs text-muted-foreground`), toggle switch right (48px touch target; 48 = 4 × 12, exceeds WCAG 2.5.5 minimum)
- Toggle on-state: `bg-accent`; off-state: `bg-muted`; transition: 150ms

**Other settings sections (non-functional this phase):**
- Render section structure and row labels as designed
- Each non-functional toggle/button: rendered with `opacity-60 cursor-not-allowed` OR labeled with `Badge` variant "outline" containing "Coming soon"
- Do NOT add tooltip or hover state to "Coming soon" badges — static label only

**Account section — Delete Account:**
- Red text button: `text-destructive text-sm font-normal`
- Clicking opens a `Dialog` confirmation: "Delete account?" heading, "This will permanently delete your profile, collection, and all data. This cannot be undone." body, "Keep Account" (outline) + "Delete Account" (destructive variant) buttons
- This phase: Delete Account dialog is non-functional (button disabled after confirmation, no actual deletion implemented); add "Coming soon" note inside dialog

## Implementation Notes for Executor

1. **Routing:** Use Next.js App Router dynamic segments `src/app/u/[username]/[tab]/page.tsx`. Default tab redirect at `src/app/u/[username]/page.tsx` → redirect to `/u/[username]/collection`.

2. **Tab navigation:** Tabs are URL-driven, not React state. Use `next/link` inside `TabsTrigger` or a custom trigger that wraps a Link. Active state derived from `usePathname()` comparing current segment.

3. **Calendar:** Build with native Date. Week starts Sunday. Current month state is local React state (useState). No external date library unless `date-fns` is already in `package.json` (check before adding).

4. **Taste tags:** Computed server-side in a Server Component; passed as props. Not stored in DB for now — derived fresh on each profile page render from the user's watches query result.

5. **Privacy toggles:** Each toggle calls a server action that updates `profile_settings` table. Optimistic update on client using `useOptimistic` (React 19 pattern). On error, revert and show error toast (use existing toast pattern or shadcn sonner if present — check before adding).

6. **Per-note visibility:** `NoteVisibilityPill` calls a server action patching `watches.notes_public` for that watch ID. Optimistic update pattern same as privacy toggles.

7. **Font:** Use `font-sans` Tailwind class (maps to Geist Sans via `--font-geist-sans`). Do NOT import IBM Plex Sans.

8. **Dark mode:** All components use `bg-background`, `text-foreground`, `bg-card`, `text-muted-foreground` etc. — the existing CSS variable system handles dark mode automatically. Never write `dark:` overrides for base color values; only use `dark:` for structural differences (e.g. `dark:ring-accent`).

9. **Avatar fallback:** If `avatarUrl` is empty or fails to load, render initials (first letter of display name) centered in a circle with `bg-accent text-accent-foreground`.

10. **Stats — HorizontalBarChart:** Do not import recharts or chart.js. Build with a simple `div` bar: outer `div` full-width `bg-muted rounded-full h-2`, inner `div` width set as inline style `width: {pct}%` with `bg-accent rounded-full`. Check if existing `chart.tsx` in `/ui/` supports this pattern before building new.
