<task type="auto">
  <name>Task 1: Create watches DAL with row-to-domain mapping</name>
  <files>src/data/watches.ts</files>
  <action>
Create `src/data/watches.ts` with `import 'server-only'` as the first import (per D-06).

Import `db` from `@/db`, `watches` from `@/db/schema`, `eq` and `and` from `drizzle-orm`, and `Watch` type from `@/lib/types`.

**mapRowToWatch function (per D-09):**
Write a private `mapRowToWatch` function that converts a Drizzle `watches.$inferSelect` row to the domain `Watch` type. Key mapping rules:
- All nullable DB columns map to `undefined` in the domain type (use `?? undefined` for nullable fields)
- Array columns (`complications`, `styleTags`, `designTraits`, `roleTags`) are already non-null in DB schema, pass through directly
- Omit `userId`, `createdAt`, `updatedAt` from the domain type — these are DB-only fields
- Boolean fields `isFlaggedDeal` and `isChronometer` default to `false` if null: `row.isFlaggedDeal ?? false` (since the domain type is `boolean | undefined` but the Watch interface uses `isFlaggedDeal?: boolean`)

**Note on Drizzle column naming:** When the schema uses `text('snake_case')`, Drizzle's `$inferSelect` type uses the JavaScript property name from the column definition. Since the schema defines columns like `caseSizeMm: real('case_size_mm')`, the JS property is already `caseSizeMm`. No snake_case-to-camelCase conversion needed — Drizzle handles this via the property name.

**Export these functions (per D-07, D-08, D-10):**

1. `getWatchesByUser(userId: string): Promise<Watch[]>` — SELECT all watches WHERE userId matches, map each row.

2. `getWatchById(userId: string, watchId: string): Promise<Watch | null>` — SELECT single watch WHERE userId AND id match. Return null if not found (not an error — D-08 says throw for unexpected failures, not-found is expected). Use `and(eq(watches.userId, userId), eq(watches.id, watchId))`.

3. `createWatch(userId: string, data: Omit<Watch, 'id'>): Promise<Watch>` — INSERT into watches table. Map domain fields to DB columns. Set `userId` from parameter. Return the created watch mapped to domain type. Use `.returning()` to get the inserted row.

4. `updateWatch(userId: string, watchId: string, data: Partial<Watch>): Promise<Watch>` — UPDATE watches SET ... WHERE userId AND id match. Use `.returning()`. Throw if no row returned (watch not found or wrong user). Set `updatedAt` to `new Date()` on every update.

5. `deleteWatch(userId: string, watchId: string): Promise<void>` — DELETE FROM watches WHERE userId AND id match. Throw if no row deleted (`.returning()` returns empty array).

For `createWatch` and `updateWatch`, write a `mapDomainToRow` helper that converts Watch fields to DB column format. Since Drizzle column property names match JS property names, this is mostly passing through values — but convert `undefined` to `null` for nullable fields. Do NOT include `id`, `createdAt`, `updatedAt` in the insert/update mapping.

Create `src/data/preferences.ts` with `import 'server-only'` as the first import (per D-06).

Import `db` from `@/db`, `userPreferences` from `@/db/schema`, `eq` from `drizzle-orm`, and `UserPreferences` type from `@/lib/types`.

**mapRowToPreferences function (per D-09):**
Convert a `userPreferences.$inferSelect` row to the domain `UserPreferences` type:
- Array columns pass through directly (non-null in schema)
- `preferredCaseSizeRange`: read from jsonb column — cast to `{ min: number; max: number } | undefined`. If the DB value is null, return undefined.
- `overlapTolerance`: pass through (non-null in schema, defaults to 'medium')
- `collectionGoal`: `?? undefined`
- `notes`: `?? undefined`
- Omit `id`, `userId`, `createdAt`, `updatedAt`

**Export these functions:**

1. `getPreferencesByUser(userId: string): Promise<UserPreferences>` — SELECT from userPreferences WHERE userId matches. If no row exists, return a default UserPreferences object (matching `defaultPreferences` from `src/store/preferencesStore.ts`):
```typescript
const defaults: UserPreferences = {
  preferredStyles: [], dislikedStyles: [],
  preferredDesignTraits: [], dislikedDesignTraits: [],
  preferredComplications: [], complicationExceptions: [],
  preferredDialColors: [], dislikedDialColors: [],
  preferredCaseSizeRange: undefined,
  overlapTolerance: 'medium',
  collectionGoal: undefined,
  notes: undefined,
}
```
This avoids requiring a row to exist before preferences can be read.

2. `upsertPreferences(userId: string, data: Partial<UserPreferences>): Promise<UserPreferences>` — Use Drizzle's `onConflictDoUpdate` (upsert) pattern. INSERT a new row with the provided fields (plus defaults for missing ones). ON CONFLICT on `userId` (unique constraint), UPDATE only the provided fields. Use `.returning()` to get the result. Map and return.

The upsert needs to handle `preferredCaseSizeRange` specially: if the domain value is `undefined`, store `null` in the jsonb column. If it is `{ min, max }`, store the object as-is.

Set `updatedAt` to `new Date()` on the conflict update path.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DAL function params -> DB query | userId must always appear in WHERE clause |
| DB row types -> Domain types | Mapping must not leak internal fields |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Information Disclosure | DAL returns DB-internal fields | mitigate | mapRowToWatch/mapRowToPreferences strip userId, createdAt, updatedAt before returning |
| T-03-05 | Elevation of Privilege | DAL imported from client code | mitigate | `import 'server-only'` at top of every DAL file causes build-time error if client-imported |
| T-03-06 | Tampering | Query without userId scope | mitigate | Every exported function includes `eq(*.userId, userId)` in WHERE; no function accepts a bare watchId without userId |
</threat_model>
