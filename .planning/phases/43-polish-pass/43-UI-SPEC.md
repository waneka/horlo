## Spacing Scale

8-point scale (existing project convention, no exceptions declared for this phase):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, drag-handle internal spacing |
| sm | 8px | Compact element spacing (badge margins, filter-chip gaps) |
| md | 16px | Default element spacing (card padding, form field gaps) |
| lg | 24px | Section padding (FilterDrawer content sections) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- **Touch targets**: Add-watch button minimum touch target 44px tall ‚Äî `size="sm"` Button in shadcn is 36px; add `min-h-[44px]` on mobile breakpoints.
- **Drawer drag handle**: 8px tall √ó 40px wide, centered ‚Äî `h-2 w-10` (8 is a multiple of 4; updated from the non-conforming 6px/`h-1.5` that existed in the pattern source).
- **Avatar crop modal**: Cropper UI occupies a container of at least 300px √ó 300px to allow drag interaction ‚Äî no fixed pixel token, but executor must ensure the modal panel is at least 320px wide.

**Interaction contract:**
- `swipeDirection="down"` ‚Äî swipe gesture provided natively; no custom touch handlers

- ESC key fires `onOpenChange(false)` unconditionally
- `Drawer.Popup` transition: `data-starting-style` / `data-ending-style` only (NOT a blanket `transition` class) to avoid conflicting with `--drawer-swipe-movement-y` CSS variable animation during swipe
- CSS variables `--drawer-swipe-movement-y`, `--drawer-swipe-progress` are available but not required for PLSH-02 ‚Äî do not add custom drag-follow animation in Phase 43

**Props interface:** Identical to existing `WatchFacetSheetProps` ‚Äî same shape, drop-in replacement at the call site.

**Import path:** `import { Drawer } from '@base-ui/react/drawer'`

**Image aspect ratio:** `aspect-[3/4]` ‚Äî "roughly 4/5, maybe a bit shorter" (CONTEXT.md D-04 Claude's Discretion). 3/4 = 0.75 ratio vs 4/5 = 0.80. Executor note: both are acceptable; use `aspect-[3/4]` as the default. If the existing `aspect-[4/5]` grid passes visual review at equal height, either is acceptable ‚Äî but choose one and apply consistently.

**Equal-height contract (D-05, PLSH-04):**
- Outer `<Card>` must have `h-full flex flex-col` ‚Äî NOT `height: auto`
- Grid parent in `CollectionTabContent` and `WishlistTabContent` must use CSS grid with `items-stretch` (default grid behavior) ‚Äî existing `grid grid-cols-2 gap-4` already works; verify items-stretch is not overridden
- Text block uses `flex-1` (not a fixed `min-h`) so it expands to fill remaining height uniformly across all cards in a row
- `aspect-[3/4]` must be on the image container `<div>`, NOT on the outer `<Card>` ‚Äî placing aspect on Card breaks equal-height layout (RESEARCH.md Pitfall 2)

**Wear suppression (D-12, PLSH-03):**
```
isWishlistLike = watch.status === 'wishlist' || watch.status === 'grail'

Wear badge renders only when:  !isWishlistLike && (isWornToday || isStale)

Last-worn line renders only when:  !isWishlistLike
```
Both suppressed for wishlist/grail cards. The text block `flex-1` absorbs the vertical space freed by the missing wear line ‚Äî no visible gap or height change.

**CSS-chain assertion (project memory blind-spot flag):**
The executor MUST verify after implementation:
1. `aspect-[3/4]` is on the image `<div>`, not on `<Card>` or `<Link>`
2. `object-cover` is on `<Image>` (not omitted)
3. Cards in the same grid row are identical height at 375px viewport with mixed-metadata fixture
