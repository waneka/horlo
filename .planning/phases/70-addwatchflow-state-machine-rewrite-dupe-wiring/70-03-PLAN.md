<objective>
Ship the **`moveWishlistToCollection`** Server Action that DUPE-03 uses to flip a wishlist watch to owned (UPDATE, not INSERT). The action is a confirmed gap-closer: `editWatch` does NOT fire `logActivity` or `logNotification` (RESEARCH verified at watches.ts:358-512), so a bare `editWatch({status:'owned'})` for the wishlist→collection move would silently skip the Network Activity feed entry and cross-user overlap notifications. This action fires them.

Purpose: DUPE-03's "Move to Collection" affordance in DupeBanner (Plan 02 ships the UI) needs this Server Action to commit the UPDATE + fire side-effects identical to addWatch. Plan 05's orchestrator calls it from DupeBanner.onMoveToCollection.

Output: 1 new named export in `src/app/actions/watches.ts` (additive — addWatch + editWatch + removeWatch all UNCHANGED) + 1 new co-located test file under `src/app/actions/__tests__/` with 6 cases.
</objective>

Internal sequence (D-10 verbatim — mirrors addWatch.ts patterns at the cited line ranges):
1. Auth-first gate (BEFORE Zod parse — matches addWatch:84): `let user; try { user = await getCurrentUser() } catch { return { success: false, error: 'Not authenticated' } }`
2. Zod parse (Claude's Discretion schema): `z.object({ watchId: z.string().uuid(), pricePaid: z.number().int().min(0).optional(), notes: z.string().max(2000).optional() })`
3. DAL ownership check: `const priorRow = await watchDAL.getWatchById(user.id, watchId)` — null → `'Watch not found'`
4. Status whitelist: priorRow.status must be `'wishlist'`; if `'owned'` → idempotent `{ success: true, data: priorRow }`; if `'sold'`/`'grail'` → `'Cannot move {status} watch to collection'`
5. updateWatch with `{status: 'owned', pricePaid: opts?.pricePaid ?? null, notes: opts?.notes ?? priorRow.notes}` — strip sortOrder (WR-01 server-truth; mirrors editWatch)
6. Activity log (mirrors addWatch:247-266 'watch_added' branch): `logActivity(user.id, 'watch_added', updatedWatch.id, { brand, model, imageUrl: imageUrl ?? null })` — NO `source` field (RESEARCH Pitfall 3 — WatchAddedMetadata type has no `source` field; use console.warn per Claude's Discretion)
7. Overlap notifications (mirrors addWatch:272-318): only fires when updatedWatch.status === 'owned' (which it is by definition here); resolves actor profile via getProfileById; payload includes brandNormalized + modelNormalized + actor_username + actor_display_name + watch_id + watch_brand + watch_model
8. Cache invalidation (mirrors addWatch:319-341): `revalidatePath('/')`, `revalidatePath('/u/[username]', 'layout')`, `revalidateTag(\`profile:\${username}\`, 'max')` (only when ownerProfile?.username), `revalidateTag('explore', 'max')`
9. Return `{ success: true, data: updatedWatch }`
10. Outer try/catch: `} catch (err) { console.error('[moveWishlistToCollection] unexpected error:', err); return { success: false, error: 'Failed to move watch to collection' } }`
