## Interaction Spec

### DupeBanner interactions

#### Owned context (DUPE-02)

```
State: confirming with dupeContext.existingStatus === 'owned'

"View existing" button click (when existingReference is non-null):
  → router.push(`/w/${encodeURIComponent(dupeContext.existingReference)}`)
  → Navigates away from add flow entirely
  → No commit; no photos step

"Add another copy" button click:
  → setState(prev => ({...prev, dupeContext: null}))  (clears dupeContext only)
  → DupeBanner unmounts; ConfirmStep stays mounted with same status/price state
  → User proceeds to ConfirmStep primary CTA for a second watch row (addWatch with same catalogId)
  → D-08: two rows for same (userId, catalogId) — permitted (no UNIQUE constraint verified by RESEARCH)

null-reference owned context (D-06 fallback):
  → "View existing" button hidden
  → Only "Add another copy" renders
  → Headline: "Already in your collection"
  → No subtext line (no reference to display)
```

#### Wishlist context (DUPE-03)

```
State: confirming with dupeContext.existingStatus === 'wishlist'

"Move to Collection" button click:
  → pending = true → DupeBanner shows Loader2 spinner + "Moving…"; buttons disabled
  → moveWishlistToCollection(dupeContext.existingWatchId) Server Action call
  → On success:
      toast.success('Moved to collection', { action: { label: 'View', onClick: () => router.push(dest) } })
      router.push(initialReturnTo ?? defaultDestinationForStatus('owned', viewerUsername))
      [photos step skipped per D-13 — existing watch row, not a fresh add]
  → On failure:
      toast.error(result.error)
      pending = false

"Add another copy" button click:
  → Same as owned context: clears dupeContext, user proceeds to add second wishlist entry
  → D-11: user explicitly bypasses the banner; second wishlist row is a deliberate choice

Pending state:
  → Both buttons disabled (pending=true)
  → "Move to Collection" button shows Loader2 + "Moving…" copy
  → "Add another copy" button: disabled styling (opacity-50 pointer-events-none via Button CVA)
  → ConfirmStep below also receives pending=true → its primary CTA is disabled
```

#### DupeBanner absent (no dupeContext)

```
When confirming.dupeContext === null:
  → DupeBanner not rendered at all
  → ConfirmStep renders solo (standard fresh-add or DUPE-02 "Add another copy" bypass)
  → ConfirmStep primary CTA is the only commit affordance
```

### extracting-url branch interactions

```
State: extracting-url (url: string payload in FlowState)

"← Back to search" click:
  → setState({ kind: 'search-idle' })
  → No history push; URL stays at /watch/new
  → isExtracting resets (component unmounts)

Input onChange:
  → setUrl(value) → local state update
  → "Find specs" button enables when url.trim() is non-empty AND !isExtracting

Input onKeyDown Enter:
  → If !url.trim() || isExtracting → no-op
  → Else → handleUrlBackup()

"Find specs" button click → handleUrlBackup():
  → Check useUrlExtractCache(viewerUserId).get(url.trim()) — cache hit short-circuits fetch
  → Cache miss: isExtracting = true → Loader2 spinner + "Finding specs…"
  → POST /api/extract-watch { mode: 'url', url: url.trim() }
  → On success:
      urlCache.set(url.trim(), { catalogId, extracted, catalogIdError })
      findViewerWatchByCatalogId(viewerUserId, catalogId, ['owned', 'wishlist']) → dupeContext or null
      setState({ kind: 'confirming', catalogId, extracted, pickedResult: null, dupeContext, pending: false })
  → On failure:
      setState({ kind: 'extraction-failed', partial: null, reason, category, mode: 'url' })
  → On network throw:
      setState({ kind: 'extraction-failed', partial: null, reason: err.message, category: 'generic-network', mode: 'url' })

isExtracting=true:
  → "Find specs" button disabled + spinner
  → URL input disabled (prevents edits during flight)
  → "← Back to search" link NOT disabled (escape always available during loading)
```

**CSS chain assertion — "Find specs" pending:**
```
Button disabled={isExtracting || !url.trim()}
  → Button[disabled] → .disabled:pointer-events-none.disabled:opacity-50 (button.tsx CVA line 7)
  → Inner content: <Loader2 className="size-4 mr-2 animate-spin" aria-hidden="true" /> + "Finding specs…"
  → animate-spin: tw-animate-css (globals.css:2); animation: spin 1s linear infinite
```

### Search-idle branch + CLNP-06

```
State: search-idle

SearchEntry renders → handles its own search/results/panel sub-states internally (D-01)
  → onPick(result) fires when user selects a result:
       if result.viewerState === 'owned' && result.reference:
```
