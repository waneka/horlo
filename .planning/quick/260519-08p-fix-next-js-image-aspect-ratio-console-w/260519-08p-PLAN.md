<objective>
Eliminate the Next.js `<Image>` aspect-ratio console warning ("Image with src '...' has either
width or height modified, but not the other...") on watch product images.

Purpose: The warning is noise that pollutes the dev console and signals a real layout-correctness
risk (one-axis CSS resize on an intrinsic-ratio image). Three components trigger it; three others
that render watch images are already correct.

Output: Three components converted from the explicit-`width`/`height` + CSS-resize pattern to the
`fill` + ratio-constrained-parent pattern that the codebase already uses correctly in WatchCard,
ProfileWatchCard, and WatchDetail.
</objective>
