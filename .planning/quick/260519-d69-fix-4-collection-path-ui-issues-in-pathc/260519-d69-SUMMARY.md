# Quick Task 260519-d69: Fix 4 Collection Path UI Issues in PathCard Summary

**One-liner:** Responsive desktop path nodes (flex-1 min-w-0 max-w-[208px]), seed watch carrying path rationale, and group-hover "Watch Details" overlay on desktop watch images.

## What Was Built

Four UI defects in `src/components/explore/PathCard.tsx` fixed in a single file change:

1. **Desktop overflow fix (Task 1):** Node wrapper changed from `w-44` to `flex-1 min-w-0 max-w-[208px]`. A 4-node path now shares the available row width equally and never overflows its container. `min-w-0` allows nodes to shrink below intrinsic content width so text truncation engages.

2. **Consistent image heights (Task 1, derived):** Because every node wrapper has equal `flex-1` width, every `aspect-square` image container resolves to the same computed size within a card. The existing CSS chain (`aspect-square rounded-md bg-muted overflow-hidden`) is preserved unchanged.

3. **Seed watch rationale (Task 2):** `seedNode.rationale` changed from hardcoded `null` to `pathWithNodes.rationale ?? null`. The seed watch now displays the path-level curator copy beneath it on both mobile and desktop. Null/empty rationale renders nothing — the existing `{node.rationale && (...)}` guard handles it. WR-01 escaped-plain-text contract unchanged.

4. **Hover CTA overlay (Task 3):** Desktop image container gains `group relative`. A bottom-anchored overlay div (`absolute inset-x-0 bottom-0 ... opacity-0 group-hover:opacity-100`) contains a `Button` rendered as a `Link` to `/catalog/${node.catalogId}`, copy "Watch Details", `variant="secondary"`, `min-h-[44px]`. Pure CSS group-hover — PathCard remains a Server Component with no `'use client'`.
