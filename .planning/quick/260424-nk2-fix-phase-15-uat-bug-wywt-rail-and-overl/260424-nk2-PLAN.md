<objective>
Phase 15 UAT bug fix — the WYWT rail tiles and overlay slides currently render the watch catalog image (`watches.imageUrl`) instead of the user's actual wrist-shot wear photo (`wear_events.photo_url`). The wear detail page (`/wear/[wearEventId]`) already implements the correct pattern (mint signed URL inline, render with fallback). This plan ports that pattern to the rail/overlay path.

Purpose: Restore the WYWT-as-Reels UX promise — when a user posts a wrist shot, their followers see the actual photo on the home rail and in the overlay slide, not the generic catalog image. Falls back to catalog imageUrl when photo_url is null (no-photo posts and pre-Phase-15 backfilled events).

Output: 3 atomic commits — (1) DAL/type changes + DAL test, (2) Server Component signing + UI fallback render + UI tests, (3) verification run.
</objective>
