- `globals.css` currently has no `scroll-behavior` rule and no `prefers-reduced-motion` block. The fix adds both.

In `src/app/globals.css`, inside the existing `@layer base` block, add `scroll-behavior: smooth;` to the existing `html` rule (the one currently containing `@apply font-sans;`). Then add a `prefers-reduced-motion` override inside the same `@layer base` block so motion-sensitive users get an instant jump: a `@media (prefers-reduced-motion: reduce)` block containing `html { scroll-behavior: auto; }`.

The `html` element is the document scroll container (confirmed in diagnosis — neither `body` nor any inner `main` sets `overflow`), so this is where native hash navigation (`href="#letter-X"`) reads `scroll-behavior` from.

The class never took effect (`<main>` is not a scroll container) — leaving it is misleading dead markup. Do NOT touch the `scroll-mt-28 md:scroll-mt-32` classes on the `<section>` elements; `scroll-margin-top` is correct and required for the sticky-nav offset.
