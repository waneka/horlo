## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-260513-m31-01 | Information disclosure | `OtherOwnersRoster` count label | accept | `totalCount` is the result of a `count(DISTINCT)` query with the SAME WHERE clause (profilePublic = true AND collectionPublic = true AND status IN (owned/wishlist/grail) AND self-exclusion) as the chip rows. Exposing a small integer that was already implicitly observable by counting the chip row (when ≤ 5) does not leak additional information; for `totalCount > 5` the label was already shipped. No PII, no user IDs in the label. |
| T-260513-m31-02 | Tampering | `WornCalendar` per-cell `onClick`/`onKeyDown` | accept | `setSelectedDate(key)` only updates local component state. `key` is a deterministic `YYYY-MM-DD` produced by `dateKey()` from the rendered grid — not user-supplied input. Cannot reach the network or persistence layer. |
| T-260513-m31-03 | XSS | New copy strings in both components | mitigate | React text-node auto-escape applies to both `{totalCount} collectors own this` and the unchanged `formatDateLabel(...)` output. No `dangerouslySetInnerHTML` introduced. `aria-label={`View wear events for ${key}`}` uses a sanitized `YYYY-MM-DD` literal. |
