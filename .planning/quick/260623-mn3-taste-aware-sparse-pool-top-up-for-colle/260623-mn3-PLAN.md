<objective>
Replace the alphabetical-by-brand fallback in `topUpFromCatalogPopularity` with a taste-aware ranking against the viewer's top brand and dominant style, and project the catalog row's real `styleTags` onto the synthetic Watch shape so the existing rationale loop produces meaningful messages on top-up rows.

Purpose: `owners_count = 0` for all 161 local catalog rows means the current top-up sort (`owners_count DESC, brand ASC LIMIT 20`) is effectively alphabetical. In a low-density product this code path dominates rendering of the "From Collectors Like You" home rail and the result is generic. After this change the rail brand-matches and style-matches what the viewer already owns even when the peer-collector pool is sparse, AND the existing rule-based rationale templates fire on top-up cards instead of always falling through to "Popular in the community."

Output: A small, surgical rewrite of one DAL helper + two exports on a pure helper module + one extended test case. No UI changes, no schema changes, no migrations, no new caches.
</objective>
